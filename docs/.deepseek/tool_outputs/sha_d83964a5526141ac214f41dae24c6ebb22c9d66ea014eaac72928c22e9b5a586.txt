   214│ ### 如果服务器是集成在 MCP 客户端中（比如 Claude Desktop）
   215│ 你根本不需要在终端手动启动它。这种情况下，配置是写在客户端的 JSON 配置文件里的，修改方法如下：
   216│ 
   217│ 1. 关闭 Claude Desktop 等客户端。
   218│ 2. 编辑配置文件（例如 `claude_desktop_config.json`），修改 `args` 数组中的参数：
   219│    ```json
   220│    "args": [
   221│        "--index", "https://pypi.tuna.tsinghua.edu.cn/simple",
   222│        "duckduckgo-mcp-server",
   223│        "--safe-search", "strict",
   224│        "--region", "us-en"
   225│    ]
   226│    ```
   227│ 3. 保存文件，重新启动客户端——它会自动用新参数启动 DuckDuckGo 服务器。
   228│ 
   229│ ---
   230│ 
   231│ ### 关于你问的“只在运行的终端进行修改可以吗？”
   232│ - 如果你指的是在**正在运行的终端里直接输入新命令或修改环境变量** → **不可以**。已经启动的进程不会感知这些变化。
   233│ - 唯一有效的“在终端里修改”就是：**先停掉它，再重新运行**。
   234│ 
   235│ 所以，记住这个简单规则：**修改配置 = 重启服务器**。希望这样解释能让你清楚该如何操作。
