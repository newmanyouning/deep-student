   231│ 在 Ubuntu 22.04 系统中进入 Docker 容器，最标准、最推荐的方法是使用 `docker exec` 命令。这会为你打开一个容器的交互式 Shell（命令行终端）[citation:1][citation:5][citation:7]。
   232│ 
   233│ 你可以按照以下步骤操作：
   234│ 
   235│ ### 🚀 核心步骤：进入正在运行的容器
   236│ 
   237│ 1.  **第一步：查看正在运行的容器**
   238│     首先，你需要找到你想要进入的容器的**名字**或**ID**。在终端中输入以下命令：
   239│     ```bash
   240│     sudo docker ps
   241│     ```
   242│     这个命令会列出所有当前正在运行的容器。找到你之前创建的 PaddleOCR 容器，记下它的 `CONTAINER ID` 或 `NAMES` [citation:1][citation:5]。
   243│ 
   244│ 2.  **第二步：使用 `docker exec` 命令进入容器**
   245│     拿到容器名或ID后，就可以使用下面的命令进入容器了。请将 `<容器名或ID>` 替换为你实际的值。
   246│     ```bash
   247│     sudo docker exec -it <容器名或ID> /bin/bash
   248│     ```
   249│     *   **命令解析** [citation:5][citation:6][citation:10]：
   250│         *   `sudo`：因为 Docker 守护进程通常以 root 权限运行，所以很多 Docker 命令前需要加上 `sudo` [citation:2]。
   251│         *   `docker exec`：核心命令，用于在运行的容器中执行新命令。
   252│         *   `-it`：这是两个参数的组合。`-i` 保证交互式操作，`-t` 分配一个伪终端，两者结合才能获得一个可以输入命令的交互式 Shell 环境。
   253│         *   `/bin/bash`：这是你要在容器内启动的程序，即 Bash Shell。如果容器内没有安装 bash，可以尝试 `/bin/sh` [citation:2][citation:5]。
   254│ 
   255│     执行成功后，你的终端提示符会发生变化，例如变成 `root@<容器ID>:/#`，这表明你现在已经在**容器内部**了。在这里，你就可以运行 `paddleocr` 命令或你的 Python 脚本了 [citation:1][citation:3]。
   256│ 
   257│ ### 🤔 常见问题与解决
   258│ 
   259│ 如果执行命令后遇到问题，可以参考以下情况：
   260│ 
   261│ *   **情况一：提示 `docker: command not found`**
   262│     *   **原因**：Docker 没有安装或当前用户权限不足。
   263│     *   **解决**：请确保你已经安装了 Docker。如果已安装，在命令前加上 `sudo` 执行，或者将你的用户加入 `docker` 用户组 [citation:2][citation:9]。
   264│ 
   265│ *   **情况二：提示 `Error: No such container`**
   266│     *   **原因**：你输入的容器名或ID有误，或者容器没有在运行。
   267│     *   **解决**：
   268│         1.  再次运行 `sudo docker ps`，仔细核对名称和ID。
   269│         2.  如果容器没有在运行，先用 `sudo docker ps -a` 查看所有容器（包括已停止的），找到你的容器，然后用 `sudo docker start <容器名或ID>` 启动它 [citation:1][citation:5][citation:6]。
   270│ 
   271│ *   **情况三：提示 `exec: "/bin/bash": stat /bin/bash: no such file or directory`**
   272│     *   **原因**：你尝试进入的容器是基于一个非常精简的镜像（如 Alpine Linux），它默认没有安装 `bash`。
   273│     *   **解决**：将命令中的 `/bin/bash` 替换为 `/bin/sh` 即可 [citation:2][citation:5]。
   274│         ```bash
   275│         sudo docker exec -it <容器名或ID> /bin/sh
   276│         ```
   277│ 
   278│ ### 🚪 如何退出容器
   279│ 
   280│ 当你完成容器内部的操作后，想回到宿主机（你的 Ubuntu 22.04 系统）时，只需在容器内的终端中输入 `exit` 命令并按回车即可 [citation:1][citation:3][citation:9]。这会让你的 Shell 会话结束，但容器本身会**继续在后台运行**，不会受到影响。
   281│
