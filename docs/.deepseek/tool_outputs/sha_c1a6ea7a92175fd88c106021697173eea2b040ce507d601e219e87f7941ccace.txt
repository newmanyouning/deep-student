    68│ ## ❓ unknown
    69│ 
    70│ 问题出在 PowerShell 中分号 `;` 是命令分隔符，而不是参数的一部分。在 PowerShell 中运行 PyInstaller 时，需要将包含分号的参数用引号括起来。
    71│ 
    72│ 这是修复后的命令：
    73│ 
    74│ ```powershell
    75│ python -m PyInstaller --onefile --windowed --name ExcelAnalyzer `
    76│   --add-data "src;src" `
    77│   --add-data "config.json;." `
    78│   --add-data "template.xls;." `
    79│   --add-data "test_image.png;." `
    80│   --hidden-import PyQt5 `
    81│   --hidden-import PyQt5.QtCore `
    82│   --hidden-import PyQt5.QtGui `
    83│   --hidden-import PyQt5.QtWidgets `
    84│   --hidden-import openpyxl `
    85│   --hidden-import pandas `

    87│   --hidden-import xlwings `

   117│   main.py

   119│ 
   120│ **关键修改：**
   121│ 1. 使用 PowerShell 的行继续符 `` ` `` 将命令分成多行（更易读）
   122│ 2. 所有 `--add-data` 和 `--add-binary` 的参数都用**双引号**括起来
   123│ 3. 分号在引号内会被当作普通字符处理
   124│ 
   125│ **或者，如果不想使用多行，可以写成一行：**
   126│ 
   127│ ```powershell
   128│ python -m PyInstaller --onefile --windowed --name ExcelAnalyzer --add-data "src;src" --add-data "config.json;." --add-data "template.xls;." --add-data "test_image.png;." --hidden-import PyQt5 --hidden-import PyQt5.QtCore --hidden-import PyQt5.QtGui --hidden-import PyQt5.QtWidgets --hidden-import openpyxl --hidden-import pandas --hidden-import numpy --hidden-import xlwings --hidden-import xlwings.utils --hidden-import xlwings._xlwindows --hidden-import xlrd --hidden-import PIL --hidden-import PIL.Image --hidden-import pywintypes --hidden-import win32api --hidden-import win32com --hidden-import win32com.gen_py --hidden-import win32com.client --hidden-import win32com.server --hidden-import win32timezone --hidden-import pythoncom --hidden-import networkx --hidden-import networkx.algorithms --hidden-import json --hidden-import logging --hidden-import threading --hidden-import tempfile --collect-all PyQt5 --collect-all PIL --collect-all xlwings --collect-all win32com --exclude-module matplotlib --exclude-module scipy --runtime-tmpdir . --add-data "xlwings_runtime_hook.py;." --add-binary "$env:LOCALAPPDATA\Programs\Python\Python313\Lib\site-packages\pywin32_system32\pywintypes313.dll;pywin32_system32" --add-binary "$env:LOCALAPPDATA\Programs\Python\Python313\Lib\site-packages\pywin32_system32\pythoncom313.dll;pywin32_system32" main.py
   129│ ```
   130│ 
   131│ **额外建议：**
   132│ 如果这个命令仍然因为路径长度问题出错，可以考虑创建一个 `.spec` 文件来管理这些复杂的配置，会更加清晰和易于维护。
