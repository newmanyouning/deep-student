    27│ ### 可能的原因
    28│ 1. 项目依赖未安装（如 `requirements.txt` 缺失包）。
    29│ 2. Python 版本与项目要求不符（如需要 3.10+，系统默认是 2.7）。
    30│ 3. 未使用虚拟环境导致全局权限或依赖冲突。

    32│ ### 解决步骤
    33│ 1. 检查 Python 版本：`python3 --version`（无需 sudo，若未安装则用 `brew install python@3`）。
    34│ 2. 在项目根目录创建虚拟环境：`python3 -m venv venv`（无需 sudo）。
    35│ 3. 激活虚拟环境：`source venv/bin/activate`（无需 sudo）。
    36│ 4. 安装项目依赖：`pip install -r requirements.txt`（若没有该文件，先 `pip install <主包名>`）。
    37│ 5. 运行项目主脚本：`python main.py`（根据实际入口文件调整）。

    40│ 确保激活虚拟环境后再安装依赖和运行，避免全局污染和权限报错（如 `Permission denied` 优先用用户级路径）。
