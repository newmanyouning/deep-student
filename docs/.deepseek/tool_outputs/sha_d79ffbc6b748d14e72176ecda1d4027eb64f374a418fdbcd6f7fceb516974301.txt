  1653│ 
  1654│ **用法：** `[驱动器编号][选项] r[Clover修订版本号(可选)，需要同时使用C选项]`  
  1655│ **示例：** `1B C r5092`
  1656│ 
  1657│ **选项的优先级顺序为：B > F > E > U > G**（即如果同时指定多个，优先级高的生效）
  1658│ 
  1659│ - **B** = 仅将引导管理器安装到驱动器的第一个分区（不写入恢复映像）。
  1660│ - **F** = 跳过格式化磁盘 – 将引导管理器安装到第一个分区，并用 `dd` 命令将恢复映像写入第二个分区（适用于已手动分好区的情况）。
  1661│ - **E** = 将驱动器第一个分区的类型设置为 EFI 系统分区。
  1662│ - **U** = 类似于 E，但将分区类型设置为“基本数据”（便于在 Windows 下编辑分区内容）。
  1663│ - **G** = 格式化为 GPT 分区表（默认是 MBR 分区表）。
  1664│ - **C** = 使用 Clover 引导程序代替默认的 OpenCore。
  1665│ - **L** = 为引导管理器提供本地存档文件（例如下载好的 Clover 压缩包），如果使用 Clover 则必须同时加上 `C` 选项。
  1666│ - **D** = 在不指定驱动器编号时使用，切换显示所有磁盘（当前此选项已禁用）。
  1667│

  1699│ ## ❓ unknown
  1700│ 
  1701│ 我们注意到用户尝试用 diskpart 手动分区时遇到了错误。错误信息显示磁盘在 clean 后无法转换为 GPT，提示“指定的磁盘不是 MBR 格式”和“指定的磁盘不可转换”。这通常是因为磁盘本身已经是 GPT 格式，但 clean 之后磁盘处于未初始化状态，Windows 可能错误地认为它不可转换。也可能是磁盘类型被识别为可移动设备，导致某些操作受限。
  1702│ 
  1703│ 我们需要指导用户正确手动分区。关键在于 clean 后需要创建分区，或者用其他命令初始化。但鉴于用户之前多次尝试脚本失败，建议使用 Rufus 方案更为稳妥。然而用户似乎希望继续手动尝试，我们需要提供准确的 diskpart 步骤。
  1704│ 
  1705│ 实际上，clean 后磁盘应该处于“没有媒体”或未初始化的状态。convert gpt 命令应该可以工作，但这里报错可能是因为磁盘是“可移动”设备，convert gpt 在某些情况下可能受限。一种方法是不使用 convert gpt，而是在 clean 后直接创建分区，因为 Windows 会自动处理分区表类型？但如果是 MBR 的磁盘，创建分区默认会创建 MBR 分区表。我们需要明确：对于可移动磁盘，Windows 可能不允许转换为 GPT？这是可能存在的限制。
  1706│ 
  1707│ 另一种方法是使用 diskpart 中的其它命令，比如先创建分区，然后使用“gpt attributes”等，但比较复杂。更简单的做法是使用第三方工具如 Rufus 或 balenaEtcher，它们可以处理可移动磁盘的 GPT 分区。
  1708│ 
  1709│ 用户当前的错误提示似乎矛盾：先提示“指定的磁盘不是 MBR 格式”，然后再次尝试时提示“指定的磁盘不可转换”。可能因为第一次 convert gpt 失败后，磁盘状态变化了。我们需要告诉用户忽略这些，直接用 Rufus。
  1710│ 
  1711│ 但考虑到用户坚持用 diskpart，我们可以提供详细步骤：在 clean 后，不要执行 convert gpt，而是直接使用“create partition primary”创建分区，然后格式化。但这样创建的是 MBR 分区表，而 M1 Mac 需要 GPT 才能引导。然而对于可移动 USB 设备，Mac 也能从 MBR 引导吗？一般来说，Mac 的 EFI 固件要求 GPT 分区表才能从 USB 引导。但有些 UEFI 固件也支持从 MBR 引导，但 M1 Mac 可能要求严格。所以还是需要 GPT。
  1712│ 
  1713│ 也许我们可以指导用户使用 diskpart 的“clean”后，再用“convert gpt”命令，但需要确保磁盘是“固定”而不是“可移动”。有些 USB 设备被系统识别为“可移动”，无法转换。这可能是根本问题。因此，建议使用 Rufus 来绕过限制。
  1714│ 
  1715│ 最终回答应该引导用户使用 Rufus 方案，并提供提取 DMG 和用 Rufus 写入的步骤。同时可以解释为什么 diskpart 可能失败，并安抚用户情绪。
  1716│

  1728│ ### ✅ 改用 Rufus 方案（成功率 99%）
  1729│ 
  1730│ #### 第一步：提取系统映像文件（DMG）
  1731│ 1. 用 **7-Zip** 打开你下载的 `InstallAssistant.pkg`（如果没装 7-Zip，先安装）。
  1732│ 2. 在 7-Zip 窗口中，进入 `InstallAssistant.pkg\Contents\SharedSupport\` 文件夹。
  1733│ 3. 找到 **`InstallESD.dmg`** 文件（大小约 12-15 GB），将它**解压或直接复制**到一个空间较大的地方，例如 `D:\macOS26.dmg`。
  1734│ 
  1735│ #### 第二步：用 Rufus 写入 U 盘
  1736│ 1. 下载 **Rufus**（[官网 rufus.ie](https://rufus.ie/zh/)，选择“Rufus 便携版”即可）。
  1737│ 2. 插入你的 U 盘（Kingston 31GB），打开 Rufus。
  1738│ 3. 在 Rufus 窗口中：
  1739│    - **设备**：选择你的 U 盘（确认是 Kingston 31GB）。
  1740│    - **引导类型选择**：点击“选择”，找到刚才提取的 `InstallESD.dmg` 文件。
  1741│ 4. **关键一步**：Rufus 会弹出一个提示框——“该镜像使用了 DD 镜像格式，是否确定以 DD 模式写入？” 此时**必须选择“是”**（否则无法引导 M1 Mac）。
  1742│ 5. 点击“开始”，等待进度条完成（约 10-30 分钟，取决于 U 盘速度）。
  1743│ 6. 完成后点击“关闭”，安全弹出 U 盘。
