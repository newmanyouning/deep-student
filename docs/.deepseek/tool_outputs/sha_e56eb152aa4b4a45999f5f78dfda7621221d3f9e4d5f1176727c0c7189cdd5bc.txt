### 提取的知识点

- **[nvidia-smi — NVIDIA GPU 监控的核心命令]** — 在云服务器终端查看 GPU 占用情况最通用、最核心的命令。
  - 来源文件：云服务器GPU占用查看命令.md
  - 核心解释：`nvidia-smi`（NVIDIA System Management Interface）是 NVIDIA 官方提供的命令行工具，安装驱动后自带。执行该命令可查看 GPU 的实时状态，包括驱动版本、显存使用率（Memory-Usage）、GPU 计算单元利用率（GPU-Util）、温度、功耗等关键信息。直接终端输入 `nvidia-smi` 即可使用，适用于 Linux、Windows 等各平台。

- **[GPU-Util 与 Memory-Usage — 两个核心监控指标]** — 解读 nvidia-smi 输出中最关键的两项数据。
  - 来源文件：云服务器GPU占用查看命令.md
  - 核心解释：`GPU-Util` 表示 GPU 计算核心的利用率，接近 100% 说明计算单元正在全力工作；`Memory-Usage` 表示显存使用情况（已用/总量），用于判断显存是否充足或已耗尽。这两项指标是判断 GPU 工作负载和资源瓶颈的核心依据。

- **[nvidia-smi -l — 实时刷新监控]** — 实现 GPU 占用情况的持续动态监控。
  - 来源文件：云服务器GPU占用查看命令.md
  - 核心解释：在 `nvidia-smi` 后添加 `-l`（小写 L）参数并指定秒数，可设置自动刷新间隔。例如 `nvidia-smi -l 1` 表示每 1 秒刷新一次，适合在运行 AI 训练任务时实时观察 GPU 资源变化。在 Linux 系统中，也可结合 `watch -n 1 nvidia-smi` 达到类似效果。

- **[nvidia-smi --query-gpu — 自定义查询与 CSV 输出]** — 按需提取关键字段，简化输出信息。
  - 来源文件：云服务器GPU占用查看命令.md
  - 核心解释：当默认输出信息过于繁杂时，使用 `--query-gpu` 参数配合 `--format=csv` 可自定义查询字段并以逗号分隔格式输出。示例命令：`nvidia-smi --query-gpu=index,name,utilization.gpu,memory.used,memory.total --format=csv`。此用法特别适合日志记录和脚本化监控。

- **[nvidia-smi pmon / --query-compute-apps — 定位 GPU 占用进程]** — 找出具体是哪个进程（PID）正在使用 GPU 资源。
  - 来源文件：云服务器GPU占用查看命令.md
  - 核心解释：`nvidia-smi pmon` 可监控 GPU 上正在运行的进程；`nvidia-smi --query-compute-apps=pid,process_name,used_memory --format=csv` 则提供更详细的计算进程信息（PID、进程名、已用显存）。这在多用户共享 GPU 服务器时尤为重要，可帮助定位和避免误杀他人进程。
