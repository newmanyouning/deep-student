    52│ *   **只看关键信息**：如果觉得默认输出太繁杂，可以用 `--query-gpu` 自定义输出格式，方便查看或记录日志：
    53│     ```bash
    54│     # 只查询GPU索引、名称、利用率、显存使用情况
    55│     nvidia-smi --query-gpu=index,name,utilization.gpu,memory.used,memory.total --format=csv
    56│     ```
    57│     这样可以获得一个简洁的、逗号分隔的信息列表 [citation:5][citation:10]。
    58│ 
    59│ *   **查看是谁在用GPU**：想要找出具体是哪个进程（PID）在占用GPU资源，可以使用：
    60│     ```bash
    61│     nvidia-smi pmon  # 监控GPU上的进程
    62│     # 或者查询更详细的计算进程信息
    63│     nvidia-smi --query-compute-apps=pid,process_name,used_memory --format=csv
    64│     ```
    65│     这能帮你定位到具体的任务 [citation:2][citation:10]。
    66│ 
    67│ ### ✨ 更好用的第三方工具
    68│ 
    69│ 如果你觉得 `nvidia-smi` 的输出不够直观，可以试试这两个社区常用的Python工具，它们提供更友好的终端显示 [citation:3]：
    70│ 
    71│ 1.  **gpustat**：对 `nvidia-smi` 的输出进行美化，以更紧凑、彩色的方式展示信息。
    72│     ```bash
    73│     pip install gpustat
    74│     gpustat -cp -i   # 显示进程名和PID，并持续刷新
    75│     ```
    76│ 
    77│ 2.  **nvitop**：堪称GPU界的 `top` 命令，可以实时、交互式地查看GPU和进程信息，强烈推荐。
    78│     ```bash
    79│     pip install nvitop
    80│     nvitop
    81│     ```
    82│ 
    83│ ### ⚠️ 重要前提
    84│ 
    85│ 请务必确保你的云服务器已经安装了 **NVIDIA 驱动**，否则系统会提示 `command not found` [citation:1][citation:7][citation:9]。如果还没有安装驱动，可以先执行 `lspci | grep -i nvidia` 来确认系统是否识别到了NVIDIA的硬件 [citation:7]。

   122│ ### 💻 命令详解
   123│ 
   124│ 1.  **基础必杀技：`nvidia-smi`**
   125│     这是最根本的命令，只要你的 NVIDIA 驱动安装正确，在终端直接输入 `nvidia-smi` 即可。你可以清晰地看到驱动版本、GPU 利用率 (`GPU-Util`)、显存占用 (`Memory-Usage`) 等核心指标 [citation:1][citation:2]。如果想持续监控，可以加上 `-l` 参数，例如每 2 秒刷新一次：`nvidia-smi -l 2`。
   126│ 
   127│ 2.  **交互式监控利器：`nvtop`**
   128│     如果你觉得 `nvidia-smi` 的输出不够直观，那么 `nvtop` (Neat Video Card TOP) 会是一个绝佳选择。它提供了一个类似任务管理器的交互界面，不仅能实时显示 GPU 和显存的总体占用率，还能以进度条和图表的形式展示历史负载，并且可以按进程查看具体占用情况 [citation:8][citation:10]。
   129│     *   **安装方法**：在 Ubuntu 22.04 上，可以直接从系统仓库安装，但版本可能较旧。推荐通过官方 PPA 安装最新版 [citation:8][citation:10]：
   130│         ```bash
   131│         sudo add-apt-repository ppa:flexiondotorg/nvtop
   132│         sudo apt update
   133│         sudo apt install nvtop
   134│         ```
   135│         安装后，直接在终端输入 `nvtop` 即可启动。
   136│ 
   137│ 3.  **Python 生态的交互神器：`nvitop`**
   138│     如果你是一个 Python 开发者，或者喜欢更现代化的终端界面，`nvitop` 会很对胃口。它被誉为“GPU 进程管理的一站式解决方案”，界面比 `nvtop` 更紧凑，信息密度高，并且可以直接在界面中杀死进程，非常方便 [citation:6]。
   139│     *   **安装方法**：推荐使用 `pipx` 安装，以避免依赖冲突 [citation:6]：
   140│         ```bash
   141│         sudo apt install pipx
   142│         pipx install nvitop
   143│         pipx ensurepath
   144│         ```
   145│         之后在终端运行 `nvitop` 即可。
   146│ 
   147│ 4.  **简洁之美：`gpustat`**
   148│     如果你只想在命令行快速看一眼概况，不喜欢交互式界面，那 `gpustat` 就很适合。它本质上是美化版的 `nvidia-smi`，用不同颜色高亮显示关键信息，并可以显示使用 GPU 的进程用户名和命令 [citation:1]。
   149│     *   **安装方法**：它是一个 Python 工具，通过 pip 安装：
   150│         ```bash
   151│         pip install gpustat
   152│         ```
   153│         使用时，直接输入 `gpustat`，或者用 `watch --color -n1 gpustat --color` 实现持续彩色监控。
   154│ 
   155│ ### ⚠️ 重要前提
   156│ 在执行上述任何命令之前，请确保你的 Ubuntu 22.04 系统已经正确安装了 **NVIDIA 显卡驱动**。如果输入 `nvidia-smi` 后提示 `command not found`，说明驱动尚未安装或未正确配置 [citation:2][citation:5]。
