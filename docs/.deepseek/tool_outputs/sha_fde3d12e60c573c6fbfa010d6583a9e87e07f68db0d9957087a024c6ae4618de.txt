    31│ ### 🔑 第一步：核心排查与修复（极大概率能解决问题）
    32│ 
    33│ 这是最关键的一步，几乎所有系统迁移后无法启动的问题都和BIOS设置或引导记录有关。
    34│ 
    35│ #### 1. 检查并修改BIOS设置
    36│ 戴尔G3 3579的BIOS里有两项关键设置需要核对。
    37│ 
    38│ *   **进入BIOS**：开机后，当屏幕出现DELL Logo时，**连续敲击`F2`键**，即可进入BIOS设置界面[reference:0][reference:1]。
    39│ 
    40│ *   **设置硬盘模式为AHCI**
    41│     *   在BIOS中，找到 **`System Configuration`** (系统配置) 选项[reference:2][reference:3]。
    42│     *   进入后，找到 **`SATA Operation`** (SATA操作)，把它旁边的模式从 `RAID On` 改成 **`AHCI`**[reference:4][reference:5][reference:6]。
    43│     *   戴尔出厂默认常设为 `RAID On`，但系统迁移后直接启动可能会无法识别AHCI模式下的硬盘。
    44│     *   修改后，**仔细阅读屏幕弹出的确认框提示**，点击 **`Apply`** (应用)，然后选择 **`Exit`** (退出)[reference:7]。
    45│ 
    46│ *   **调整启动顺序**
    47│     *   在BIOS里找到 **`Boot Sequence`** (启动顺序/引导顺序) 选项[reference:8][reference:9]。
    48│     *   在列表中，找到你的**新固态硬盘**，通过屏幕右侧的说明把它调整到 **第一位**，确保没有将网络启动(Onboard NIC)等选项放在前面[reference:10]。
    49│     *   调整好后，按`F10`键保存并退出。
    50│ 
    51│ #### 2. 修复引导文件 (Bootloader)
    52│ 如果调整BIOS后仍然失败，很可能是引导文件（如BCD）损坏了[reference:11]。
    53│ 
    54│ *   **准备工具**：使用另一台能正常工作的电脑，将Windows 10/11安装镜像写入U盘，制作一个**Windows安装U盘**。
    55│ *   **启动修复**：将U盘插入故障电脑，开机按`F12`选择从U盘启动[reference:12]。进入安装界面后，按`Shift + F10`打开命令提示符，然后依次输入以下命令并执行：
    56│     1.  **`diskpart`**  // 进入磁盘工具
    57│     2.  **`list disk`**  // 列出所有硬盘。请根据容量大小，**记下你的新固态硬盘的编号**（如0、1等）。
    58│     3.  **`select disk X`**  // **把 `X` 换成你刚才记下的新硬盘编号**
    59│     4.  **`list partition`**  // 列出所选硬盘的分区。确认其中的系统分区（类型为“主要”，大小约等于C盘）和EFI分区（类型为“系统”，大小约100MB-500MB）。如果看到EFI分区，请继续；如果没有，说明你可能采用了Legacy引导方式，请直接跳到最后的 `exit` 命令。
    60│     5.  **`select partition Y`**  // **把 `Y` 换成你的EFI分区编号**
    61│     6.  **`assign letter=S`**  // 给这个分区分配一个盘符，例如“S”。如果提示S被占用，也可换其他字母[reference:13]。
    62│     7.  **`exit`**  // 退出diskpart工具。
    63│     8.  **`bcdboot C:\Windows /s S: /f UEFI`**  // 执行修复命令。这个命令的作用是，把 **C盘** 下的Windows引导文件，复制到 **S盘**（也就是你刚才分配的EFI分区），并指定引导模式为UEFI[reference:14]。如果系统不在C盘，请将`C:\windows`换成实际盘符。
    64│     9.  **`exit`**  // 关闭命令提示符，然后重启电脑。
    65│ 
    66│ ### 💡 备选方案：如果以上方法无效

    68│ 如果上述方法都没能解决，可以试试下面的备选方案：

    70│ *   **尝试安全模式启动**：在迁移系统后，硬件的变更有时会导致驱动冲突。可以尝试在开机时强制关机三次，触发Windows自动修复，进入高级选项 -> 疑难解答 -> 高级选项 -> 启动设置，选择启用**安全模式**[reference:15]。如果能进入安全模式，可以卸载潜在的驱动冲突，然后重启。
    71│ *   **考虑更换迁移工具并重新迁移**：如果问题依旧，可能是“分区助手”在迁移过程中出现错误。
    72│     1.  将旧硬盘装回电脑，正常启动进入原系统。

    74│     3.  迁移成功后，重复第一步中的BIOS检查流程，确保启动顺序正确。
