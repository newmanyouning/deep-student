    15│ 嗯，用户在阿里云服务器上用Docker部署了PaddleOCR，现在需要在纯命令行环境下OCR识别一个PDF文件。这是一个很实际的操作问题，用户的核心需求是找到能在服务器上直接运行的OCR命令。

    17│ 我计划从两个角度给出解决方案：一是如果用户部署的是标准OCR服务，可以用命令行工具直接处理PDF；二是如果部署的是API服务，可以用curl调用接口。同时需要把具体命令写清楚，让用户能直接复制使用。

    25│ 我准备分两个方案组织回答：方案一给出命令行工具的直接调用命令，方案二给出API调用的curl示例。最后补充一个实用建议，提醒用户注意PDF文件路径和等待时间。
    26│ 要在只有命令行界面的服务器上，使用 Docker 镜像的 PaddleOCR 识别 PDF 文件，通常有两种最直接的方法。
    27│ 
    28│ 您可以先确认一下您的 Docker 镜像具体包含了哪种服务，然后选择最适合的一种操作：
    29│ 
    30│ ### 方案一：如果您的镜像是 PaddleOCR 的标准运行环境
    31│ 这种情况通常意味着镜像内安装了 PaddleOCR 的 Python 库。您可以直接使用它提供的命令行工具或Python脚本进行识别。
    32│ 
    33│ **步骤 1：找到您的 PDF 文件**
    34│ 首先，您需要知道 PDF 文件在服务器上的准确路径。假设它位于 `/root/documents/your_file.pdf`。
    35│ 
    36│ **步骤 2：进入正在运行的 Docker 容器**
    37│ 在服务器终端执行以下命令，进入您已部署的 PaddleOCR 容器内部。
    38│ ```bash
    39│ # 假设您的容器名称或ID为 paddle_container
    40│ docker exec -it paddle_container /bin/bash
    41│ ```
    42│ 
    43│ **步骤 3：在容器内执行 OCR 识别**
    44│ PaddleOCR 提供了非常方便的**命令行工具**，可以直接处理 PDF 文件。在容器内的终端中，运行如下命令：
    45│ ```bash
    46│ # 使用 PaddleOCR 的命令行工具识别 PDF
    47│ # 将 /path/in/container/your_file.pdf 替换为 PDF 在容器内的实际路径
    48│ paddleocr --image_dir /path/in/container/your_file.pdf --lang ch --use_gpu false --ocr_version PP-OCRv5 --output ./ocr_results
    49│ ```
    50│ *   `--image_dir`：指定输入文件，可以是图片或 PDF。
    51│ *   `--lang ch`：指定语言为中文。
    52│ *   `--use_gpu false`：如果你的服务器没有 GPU 或未配置 GPU 驱动，请加上这个参数使用 CPU 识别。
    53│ *   `--ocr_version`：指定模型版本，通常官方最新镜像会包含较新的模型（如 PP-OCRv5）[citation:4]。
    54│ *   `--output`：指定识别结果的输出目录。
    55│ 这个命令会将 PDF 的每一页作为图像进行处理，并输出识别到的文本信息[citation:4]。
    56│ 
    57│ ### 方案二：如果您的镜像已经启动了一个 OCR API 服务
    58│ 有些部署方式会将 PaddleOCR 封装成一个 HTTP 服务。如果是这种情况，您无需进入容器，可以直接在宿主机（您的服务器）上用 `curl` 命令调用它。
    59│ 
    60│ **步骤 1：确认 API 服务地址和端口**
    61│ 假设您的 OCR 服务运行在容器的 `8000` 端口，并且映射到了宿主机的 `8866` 端口（这是一个常见示例 [citation:2]）。您可以通过 `docker ps` 查看端口映射情况。
    62│ 
    63│ **步骤 2：在服务器终端使用 `curl` 发送请求**
    64│ 如果服务端接受 base64 编码的图片数据，您需要先将 PDF 转换（但这在命令行较复杂）。一个更通用的假设是，如果服务遵循类似 PaddleOCR-VL 或 OpenAI API 的风格，它可能支持直接上传文件 [citation:1]。但最简单的测试方式是，如果服务是参考 FastAPI 搭建的，通常会有一个 `/ocr` 接口。
    65│ 
    66│ 这里提供一个**通用的 `curl` 示例**，假设接口接受 PDF 文件上传：
    67│ ```bash
    68│ # 在服务器终端执行，无需进入容器
    69│ curl -X POST "http://localhost:8866/ocr" \
    70│   -F "file=@/root/documents/your_file.pdf"
    71│ ```
    72│ *   `-F "file=@文件路径"` 是模拟表单上传文件 [citation:1]。
    73│ *   你需要将 `8866` 替换为你容器服务实际的映射端口，将 `/ocr` 替换为实际的 API 路径。
    74│ 
    75│ **💡 小提示：**
    76│ *   **关于 PDF 处理**：PaddleOCR 的命令行工具在后台会自动将 PDF 的每一页转换为图像进行处理，所以您可以像处理图片一样直接处理 PDF 文件 [citation:4][citation:7]。
    77│ *   **文件路径**：如果使用 `docker exec` 进入容器的方法，请确保你的 PDF 文件已被挂载或复制到了容器内部。你可以在 `docker run` 时使用 `-v /root/documents:/documents` 参数来挂载，这样容器内就可以通过 `/documents/your_file.pdf` 访问到了。
