   158│ 下面是详细的配置步骤：
   159│ 
   160│ ### 🔑 **详细配置步骤**
   161│ 
   162│ **1. 生成SSH密钥对**
   163│ 打开终端（Windows的Git Bash、macOS的终端或Linux的Shell），执行以下命令，请将邮箱替换为你自己的GitHub邮箱[citation:1]：
   164│ ```bash
   165│ ssh-keygen -t ed25519 -C "your_email@example.com"
   166│ ```
   167│ *   **算法选择**：`ed25519`是当前更安全、更快的选择[citation:1]。如果你的系统较旧不支持，可改用 `-t rsa -b 4096`[citation:3]。
   168│ *   执行命令后，直接按回车使用默认的密钥存储路径（通常是`~/.ssh/id_ed25519`）。
   169│ *   如果希望增加一层保护，可以设置一个密钥密码（passphrase），这样每次使用密钥时都需要输入。也可以直接回车留空，但这样便利性更高，安全性稍低[citation:5]。
   170│ 
   171│ **2. 将公钥添加到GitHub账户**
   172│ *   **复制公钥内容**：在终端中运行以下命令，然后**完整选中并复制**显示出的所有内容[citation:1][citation:5]。
   173│     ```bash
   174│     cat ~/.ssh/id_ed25519.pub
   175│     ```
   176│ *   **在GitHub网站上添加**：
   177│     1.  登录GitHub，点击右上角头像，进入 **“Settings”**[citation:1]。
   178│     2.  在左侧边栏找到并点击 **“SSH and GPG keys”**[citation:5]。
   179│     3.  点击绿色的 **“New SSH key”** 按钮[citation:1]。
   180│     4.  在“Title”中为这个密钥起个名字（例如“My Work Laptop”），然后将刚才复制的公钥内容粘贴到“Key”框中[citation:5]。
   181│     5.  点击 **“Add SSH key”** 完成添加。
   182│ 
   183│ **3. 测试连接**
   184│ 在终端中运行以下命令来验证配置是否成功[citation:1][citation:5]：
   185│ ```bash
   186│ ssh -T git@github.com
   187│ ```
   188│ 如果看到类似 `Hi [你的用户名]! You've successfully authenticated...` 的欢迎信息，就说明一切正常。
   189│ 
   190│ ### 💡 **重要提示与后续操作**
   191│ *   **密钥与仓库**：SSH密钥是**账户级别**的设置。你只需要配置一次，之后你账户下所有的仓库（包括你有推送权限的他人的仓库）都可以通过SSH协议进行安全的克隆、推送和拉取操作。
   192│ *   **处理连接问题**：如果测试时出现超时错误，可能是网络问题。可以尝试修改SSH配置，让SSH通过`443`端口连接GitHub[citation:1][citation:4]。
   193│ *   **切换现有仓库协议**：如果你之前用HTTPS克隆了自己的仓库，现在想改用SSH，可以在仓库目录下执行：
   194│     ```bash
   195│     git remote set-url origin git@github.com:你的用户名/仓库名.git
   196│     ```
   197│ *   **保管好私钥**：请务必保管好你的私钥文件（如`~/.ssh/id_ed25519`），不要泄露给他人。公钥（`.pub`文件）是可以放心分享的。
