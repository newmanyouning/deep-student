### 一、API 缓存命中与未命中（通用场景）

| 概念 | 定义 | 表现 |
|------|------|------|
| **命中缓存 (Cache Hit)** | 请求的数据在缓存中存在且有效，服务器直接返回缓存结果 | 响应极快（毫秒级），不消耗数据库或后端计算资源 |
| **未命中缓存 (Cache Miss)** | 请求的数据在缓存中不存在或已过期，服务器需从原始源（数据库、后端API等）查询/计算后返回，并写入缓存 | 响应较慢（几十到几百毫秒），消耗后端资源 |

- 例子：频繁查询同一城市天气 → 命中缓存；查询冷门历史数据 → 未命中缓存

### 二、云端服务器缓存的本质与形式

**定义**：一种临时、高速的数据存储层，位于请求路径与慢速持久化存储之间。

**核心目的**：用空间换时间，减少重复计算和IO开销
- 降低响应延迟（让API更快）
- 减轻后端负载（保护数据库）
- 支撑更高并发

**常见缓存类型**：

| 缓存类型 | 典型技术/场景 | 特点 |
|----------|--------------|------|
| **内存缓存** | Redis, Memcached, 应用进程内缓存 | 最快（纳秒/微秒级），容量贵，重启丢失 |
| **分布式缓存** | 集群版Redis | 跨多服务器，容量大，高可用 |
| **CDN缓存** | Cloudflare, Akamai, AWS CloudFront | 全球边缘节点缓存静态资源 |
| **数据库查询缓存** | MySQL Query Cache（已弃用） | 自动缓存查询结果，易失效 |

**典型流程**（以Redis为例）：
1. API收到请求 → 2. 先查缓存 → 3. 命中则直接返回 → 4. 未命中则查数据库 → 写入缓存（设置过期时间） → 返回结果

**缓存三大问题**：
- **缓存穿透**：请求不存在的数据，每次都绕过缓存查数据库 → 解决：缓存空值或布隆过滤器
- **缓存击穿**：热点key刚好过期，大量请求同时打向数据库 → 解决：互斥锁或逻辑过期
- **缓存雪崩**：大量key同时过期或缓存实例宕机 → 解决：过期时间加随机值、缓存高可用
