### 基本概念回顾

*   **平衡因子**：某节点的左子树高度减去右子树高度。平衡因子绝对值超过1（即2或-2）则失衡。
*   **LR旋转**：在节点A的左孩子的右子树上插入新节点，导致A失衡（A的平衡因子为2，A的左孩子平衡因子为-1）。需要先对A的左孩子进行**左旋转**，再对A本身进行**右旋转**。
*   **RL旋转**：在节点A的右孩子的左子树上插入新节点，导致A失衡（A的平衡因子为-2，A的右孩子平衡因子为1）。需要先对A的右孩子进行**右旋转**，再对A本身进行**左旋转**。

---

### 1. LR旋转（左-右旋转）

**失衡场景**： 节点A是失衡节点。A的左孩子是B，B的右孩子是C。在C或其子树上插入节点，导致A的平衡因子变为2。

**核心目标**： 将子树根节点从A -> B -> C 的路径，变为以C为新的根节点。

**我们用一个具体例子来演示**：
初始状态（插入前是平衡的）：
```
        A (平衡因子=1)
       / \
      B   T4
     / \
    T1  C
       / \
      T2 T3
```
在 **C的左子树T2** 或 **右子树T3** 中插入一个节点后，树失衡了：
```
        A (平衡因子=2)  <-- 失衡节点
       / \
   (BF=-1) B   T4
         / \
        T1  C
           / \
          T2 T3
```
*(BF=-1表示B的平衡因子为-1)*

#### **LR旋转两步走：**

**第一步：以B为支点，进行左旋转**

*   **操作**： 将C提升为子树的根，B变为C的左孩子，C原来的左孩子T2变为B的右孩子。
*   **变化过程**：
    *   `B.right = C.left` (即 `B.right = T2`)
    *   `C.left = B`
    *   此时，A的左指针仍然指向这个子树，但现在子树的根是C了。所以 `A.left = C`（这一步在代码逻辑上发生在第二步，但结构上可以这样理解）。

**第一步旋转后的结构**：
```
        A (平衡因子=2)  <-- 失衡节点依旧存在，但结构已为下一步准备好
       / \
      C   T4
     / \
    B   T3
   / \
  T1 T2
```
注意：此时虽然A仍然失衡，但整个树的形状已经变成了LL型（节点A、C、B在一条直线上）。

**第二步：以A为支点，进行右旋转**

*   **操作**： 将C提升为整个子树的根，A变为C的右孩子，C原来的右孩子T3变为A的左孩子。
*   **变化过程**：
    *   `A.left = C.right` (即 `A.left = T3`)
    *   `C.right = A`

**最终旋转平衡后的结构**：
```
        C (平衡因子=0)
       / \
      B   A
     / \ / \
    T1 T2 T3 T4
```
现在，所有节点的平衡因子都在[-1, 0, 1]范围内，树恢复平衡。

### 2. RL旋转（右-左旋转）

**失衡场景**： 节点A是失衡节点。A的右孩子是B，B的左孩子是C。在C或其子树上插入节点，导致A的平衡因子变为-2。

**核心目标**： 将子树根节点从A -> B -> C 的路径，变为以C为新的根节点。

**我们用一个具体例子来演示**：
初始状态（插入前是平衡的）：
```
      A (平衡因子=-1)
     / \
    T1  B
       / \
      C   T4
     / \
    T2 T3
```
在 **C的左子树T2** 或 **右子树T3** 中插入一个节点后，树失衡了：
```
      A (平衡因子=-2)  <-- 失衡节点
     / \
    T1  B (BF=1)
       / \
      C   T4
     / \
    T2 T3
```

#### **RL旋转两步走：**

**第一步：以B为支点，进行右旋转**

*   **操作**： 将C提升为子树的根，B变为C的右孩子，C原来的右孩子T3变为B的左孩子。
*   **变化过程**：
    *   `B.left = C.right` (即 `B.left = T3`)
    *   `C.right = B`
    *   此时，A的右指针仍然指向这个子树，但现在子树的根是C了。所以 `A.right = C`。

**第一步旋转后的结构**：
```
      A (平衡因子=-2)  <-- 失衡节点依旧存在
     / \
    T1  C
       / \
      T2  B
         / \
        T3 T4
```
此时，树的形状变成了RR型（节点A、C、B在一条直线上）。

**第二步：以A为支点，进行左旋转**

*   **操作**： 将C提升为整个子树的根，A变为C的左孩子，C原来的左孩子T2变为A的右孩子。
*   **变化过程**：
    *   `A.right = C.left` (即 `A.right = T2`)
    *   `C.left = A`

**最终旋转平衡后的结构**：
```
        C (平衡因子=0)
       / \
      A   B
     / \ / \
    T1 T2 T3 T4
```
树恢复平衡。

### 总结

| 旋转类型 | 失衡条件 (A为根) | 解决步骤 | 新根节点 |
| :--- | :--- | :--- | :--- |
| **LR旋转** | `A.BF = 2` 且 `A.left.BF = -1` | 1. 对A的左孩子**左旋** <br> 2. 对A**右旋** | **C** (A左孩子的右孩子) |
| **RL旋转** | `A.BF = -2` 且 `A.right.BF = 1` | 1. 对A的右孩子**右旋** <br> 2. 对A**左旋** | **C** (A右孩子的左孩子) |

**关键记忆点**：
*   名字的第一字母（L或R）表示第一次旋转是针对**哪个孩子**进行的。
*   名字的第二字母（L或R）表示第二次旋转是针对**失衡节点A**进行的。
*   最终，处于失衡路径“拐点”的那个节点（LR中的C，RL中的C）会成为新的子树根。
