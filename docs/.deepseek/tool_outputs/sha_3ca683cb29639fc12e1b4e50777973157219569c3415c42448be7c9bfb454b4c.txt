   119│ ### 📦 部署方案一：使用专业工具 `tts-router`（强烈推荐）
   120│ 这是目前最简单、最稳定的方法。`tts-router` 专门为Apple Silicon优化，它会自动处理模型下载和依赖问题。
   121│ 
   122│ 1.  **安装 `tts-router`**
   123│     在“终端”中运行：
   124│     ```bash
   125│     pip install tts-router
   126│     ```
   127│     该工具需要 Python 3.11+[reference:1]。
   128│ 
   129│ 2.  **下载模型并测试**
   130│     ```bash
   131│     # 下载Qwen3-TTS模型（会自动适配MLX优化版）
   132│     tts-router pull qwen3-tts
   133│ 
   134│     # 立即测试，生成一段语音并保存为hello.wav
   135│     tts-router say "你好，欢迎使用Qwen3文本转语音系统" -o hello.wav
   136│     ```
   137│     下载完成后，第一条命令会自动下载适合你Mac的轻量级模型。第二条命令用来测试，你可以在`hello.wav`文件中听到生成的语音。
   138│ 
   139│ ### ⚙️ 部署方案二：使用轻量级MLX优化版本
   140│ 这种方法更灵活，可以让你自由选择下载哪个版本的模型。MLX是苹果官方的机器学习框架，能让模型在M1芯片上运行得更好。
   141│ 
   142│ 1.  **安装依赖**
   143│     ```bash
   144│     # 安装核心的MLX音频库
   145│     pip install mlx-audio
   146│ 
   147│     # 安装音频处理工具
   148│     brew install ffmpeg
   149│     ```
   150│ 
   151│ 2.  **下载轻量级模型 (0.6B版本)**
   152│     为了让你有更清晰的对比，这里列出了MLX社区优化的不同模型版本。请根据你的需求和Mac Mini M1的16GB内存，从下表选择一个下载。
   153│ 
   154│ | 模型变体 | Hugging Face 模型ID | 模型大小 | 内存占用 | 主要用途 |
   155│ | :--- | :--- | :--- | :--- | :--- |
   156│ | **CustomVoice (推荐)** | `mlx-community/Qwen3-TTS-12Hz-0.6B-CustomVoice-4bit` | ~1GB | ~4GB | 使用内置音色 + 风格控制[reference:2] |
   157│ | **Base (语音克隆)** | `mlx-community/Qwen3-TTS-12Hz-0.6B-Base-4bit` | ~1GB | ~4GB | 从参考音频进行语音克隆[reference:3] |
   158│ | VoiceDesign (功能最强) | `mlx-community/Qwen3-TTS-12Hz-1.7B-VoiceDesign-5bit` | ~2GB | ~5GB | 从文字描述创造全新音色[reference:4] |
   159│ 
   160│     根据你的情况，我推荐使用 **CustomVoice** 版本。下载命令如下：
   161│     ```bash
   162│     # 从Hugging Face下载模型
   163│     huggingface-cli download mlx-community/Qwen3-TTS-12Hz-0.6B-CustomVoice-4bit
   164│     ```
   165│ 
   166│ 3.  **编写Python代码进行测试**
   167│     下载后，你需要编写一个简单的Python脚本（例如`test_tts.py`）来调用模型：
   168│     ```python
   169│     from mlx_audio.tts.utils import generate
   170│     import soundfile as sf
   171│ 
   172│     # 替换为你的模型路径
   173│     model_path = "mlx-community/Qwen3-TTS-12Hz-0.6B-CustomVoice-4bit"
   174│ 
   175│     # 生成语音，指定音色（如'Vivian'）和文本
   176│     wav, sr = generate(
   177│         model_path=model_path,
   178│         text="你好，这是来自Mac Mini的Qwen3-TTS测试。",
   179│         voice="Vivian",
   180│         language="Chinese"
   181│     )
   182│ 
   183│     # 保存音频文件
   184│     sf.write("output.wav", wav, sr)
   185│     ```
   186│     然后在终端运行 `python test_tts.py`，你就能在`output.wav`中听到生成的语音了。

   188│ ### 🐳 部署方案三：Docker 容器化部署
   189│ 如果你希望有一个干净的、不污染本地Python环境的方式，可以使用Docker。但请注意，Docker镜像主要为NVIDIA GPU设计，在Mac M1上可能无法获得硬件加速，因此速度会受影响，只适合初次体验。
   190│ 
   191│ 1.  **安装Docker Desktop**
   192│     访问[Docker官网](https://www.docker.com/products/docker-desktop/)下载并安装适用于Apple Silicon的Docker Desktop。
   193│ 
   194│ 2.  **运行Docker容器**
   195│     在“终端”中执行以下命令：
   196│     ```bash
   197│     docker run -d \
   198│       -p 7860:7860 \
   199│       --name qwen3-tts \
   200│       -v $(pwd)/output:/app/output \
   201│       registry.cn-hangzhou.aliyuncs.com/inscode/qwen3-tts-12hz-1.7b-customvoice:latest bash
   202│     ```
   203│     此命令会启动一个容器，并暴露一个Web界面在 `http://localhost:7860`[reference:5][reference:6]。你可以在浏览器中打开这个地址，通过图形界面来使用TTS功能。
   204│ 
   205│ ### 🔗 集成到 OpenMAIC
   206│ 目前，关于Qwen3-TTS在OpenMAIC中的配置，公开信息较少。不过，OpenMAIC本身支持自定义TTS服务，你可以通过以下方式尝试集成。
   207│ 
   208│ 1.  **启动API服务**：你可以使用`tts-router`（部署方案一）启动一个兼容OpenAI接口的API服务[reference:7]：
   209│     ```bash
   210│     tts-router serve --port 8091
   211│     ```
   212│     启动后，你可以在浏览器打开 `http://localhost:8091` 查看API文档。
   213│ 
   214│ 2.  **配置OpenMAIC**：在你的OpenMAIC项目中（通常是`.env.local`或类似配置文件），寻找`TTS`相关的配置项，例如`TTS_PROVIDER`、`TTS_API_URL`、`TTS_API_KEY`等，然后进行如下设置：
   215│     ```
   216│     TTS_PROVIDER=custom  # 或 openai，取决于OpenMAIC的支持方式
   217│     TTS_API_URL=http://localhost:8091/v1/audio/speech
   218│     TTS_API_KEY=not-needed
   219│     ```
   220│     *注意：具体的配置字段名称可能因OpenMAIC版本而异，建议查阅其官方文档或源代码中的配置示例。*
   221│ 
   222│ ### 🎤 功能速览
   223│ 部署成功后，你可以使用以下功能：
   224│ 
   225│ *   **内置音色**：Qwen3-TTS提供了多种预设的、高质量的音色，覆盖不同性别、语言和风格。例如，`Vivian`是明亮年轻的女声，`Uncle_Fu`是权威的新闻主播男声[reference:8]。
   226│ *   **语音克隆**：如果你想让AI模仿某个特定人的声音，可以提供一个5-10秒的参考音频文件，模型就能学习并克隆该声音[reference:9]。
   227│ *   **风格与情感控制**：这是Qwen3-TTS最强大的功能之一。你可以通过简单的指令（`--instruct`参数）来控制语音的情感和风格，例如“用一种激动兴奋的语气说”或“像新闻主播一样，保持冷静权威”[reference:10]。

   229│ ### ⚡ 快速选择建议
   230│ | 你的需求 | 推荐方案 |
   231│ | :--- | :--- |
   232│ | **追求简单、快速、开箱即用** | **部署方案一 (tts-router)**。一条命令下载，一条命令测试，最适合开发者。 |
   233│ | **想深度定制，或集成到自己的Python项目中** | **部署方案二 (MLX优化版本)**。提供了最大的灵活性，方便你编写脚本进行控制。 |
   234│ | **希望隔离环境，或偏爱图形界面** | **部署方案三 (Docker)**。但需注意在Mac上可能无法获得最佳性能。 |
   235│ 
   236│ ### ⚠️ 注意事项
   237│ *   **环境问题**：务必在干净独立的Python虚拟环境中安装依赖，避免版本冲突[reference:11]。
   238│ *   **资源监控**：在运行模型时，可以通过“活动监视器”观察内存占用情况，特别是使用1.7B模型时。
   239│ *   **首次运行**：无论是`tts-router pull`还是模型代码，首次运行都会从Hugging Face下载模型文件，请确保网络通畅，并耐心等待。

   241│ 你可以先试试**部署方案一**中的 `tts-router`，这是最快能让你听到效果的路径。如果在配置OpenMAIC的环节需要更具体的指导，随时可以再来问我。
