  1264│ ### 3.4 Self-Consistency
  1265│ 
  1266│ Self-Consistency通过多次采样并投票选择最一致的答案，显著提高CoT的准确性。
  1267│ 
  1268│ #### 3.4.1 Self-Consistency工作流程
  1269│ 
  1270│ ```python
  1271│ # Self-Consistency伪代码
  1272│ def self_consistency(question, llm, n=5, temperature=0.7):
  1273│     """
  1274│     question: 输入问题
  1275│     llm: 大语言模型
  1276│     n: 采样次数
  1277│     temperature: 采样温度
  1278│     """
  1279│     answers = []
  1280│     reasoning_paths = []
  1281│     
  1282│     for i in range(n):
  1283│         # 每次使用CoT生成推理路径和答案
  1284│         prompt = f"{question}\n让我们一步步思考："
  1285│         response = llm.generate(prompt, temperature=temperature)
  1286│         
  1287│         # 提取最终答案（假设答案在"因此"或"答案是"之后）
  1288│         final_answer = extract_answer(response)
  1289│         answers.append(final_answer)
  1290│         reasoning_paths.append(response)
  1291│     
  1292│     # 投票选择最一致的答案
  1293│     from collections import Counter
  1294│     vote_result = Counter(answers).most_common(1)[0]
  1295│     
  1296│     return {
  1297│         "answer": vote_result[0],
  1298│         "confidence": vote_result[1] / n,
  1299│         "all_answers": answers,
  1300│         "reasoning_paths": reasoning_paths
  1301│     }
  1302│ 
  1303│ # 使用示例
  1304│ question = "一个抽屉里有5只红袜子和4只蓝袜子。如果不看随机拿两只，拿到同色的概率是多少？"
  1305│ result = self_consistency(question, llm, n=5)
  1306│ 
  1307│ print(f"最终答案: {result['answer']}")
  1308│ print(f"置信度: {result['confidence']:.0%}")
  1309│ print(f"所有答案: {result['all_answers']}")
  1310│ 
  1311│ # 可能输出:
  1312│ # 最终答案: 4/9 ≈ 0.444
  1313│ # 置信度: 80%
  1314│ # 所有答案: ['4/9', '4/9', '4/9', '5/9', '4/9']
  1315│ ```

  1332│ 
  1333│ ReAct框架让模型交替进行推理和行动，能够与外部环境交互（如调用API、搜索数据库、执行代码）。
  1334│ 
  1335│ #### 3.5.1 ReAct的核心循环
  1336│ 
  1337│ ```markdown
  1338│ ## ReAct模式模板
  1339│ 
  1340│ <system>
  1341│ 你是一个AI助手，可以执行以下行动：
  1342│ - Search[query]: 搜索知识库
  1343│ - Calculate[expression]: 计算数学表达式
  1344│ - Code[python_code]: 执行Python代码
  1345│ - Finish[answer]: 输出最终答案
  1346│ </system>
  1347│ 
  1348│ <format>
  1349│ Thought: [当前推理]
  1350│ Action: [选择的行动]
  1351│ Observation: [行动结果]
  1352│ (重复上述循环)
  1353│ Thought: 我可以给出最终答案
  1354│ Action: Finish[最终答案]
  1355│ </format>
  1356│ 
  1357│ <example>
  1358│ Question: 2024年奥斯卡最佳影片的导演还导演过哪些电影？
  1359│ 
  1360│ Thought: 我需要先找到2024年奥斯卡最佳影片是什么。
  1361│ Action: Search[2024年奥斯卡最佳影片]
  1362│ Observation: 2024年奥斯卡最佳影片是《奥本海默》
  1363│ 
  1364│ Thought: 现在我知道电影是《奥本海默》，需要找到导演。
  1365│ Action: Search[奥本海默 导演]
  1366│ Observation: 导演是克里斯托弗·诺兰
  1367│ 
  1368│ Thought: 需要找出诺兰导演的其他电影。
  1369│ Action: Search[克里斯托弗·诺兰 电影作品]
  1370│ Observation: 《盗梦空间》、《星际穿越》、《敦刻尔克》、《信条》、《蝙蝠侠》三部曲等
  1371│ 
  1372│ Thought: 我已经收集到足够信息，可以回答。
  1373│ Action: Finish[克里斯托弗·诺兰执导的电影包括：《盗梦空间》(2010)、《星际穿越》(2014)、《敦刻尔克》(2017)、《信条》(2020)、《蝙蝠侠：黑暗骑士》三部曲等]

  1378│ </new_question>
  1379│ ```

  1381│ #### 3.5.2 ReAct实际应用案例
  1382│ 
  1383│ ```python
  1384│ # ReAct实现示例（简化版）
  1385│ class ReActAgent:
  1386│     def __init__(self, llm, tools):
  1387│         self.llm = llm
  1388│         self.tools = tools
  1389│         self.max_iterations = 10
  1390│     
  1391│     def run(self, question):
  1392│         context = f"Question: {question}\n"
  1393│         
  1394│         for _ in range(self.max_iterations):
  1395│             # 生成Thought和Action
  1396│             prompt = self.build_prompt(context)
  1397│             response = self.llm.generate(prompt)
  1398│             
  1399│             # 解析Action
  1400│             action = self.parse_action(response)
  1401│             if action['type'] == 'Finish':
  1402│                 return action['content']
  1403│             
  1404│             # 执行Action
  1405│             observation = self.execute_action(action)
  1406│             context += f"\n{response}\nObservation: {observation}\n"
  1407│         
  1408│         return "达到最大迭代次数，无法完成"
  1409│     
  1410│     def execute_action(self, action):
  1411│         tool_name = action['name']
  1412│         tool_input = action['input']
  1413│         
  1414│         if tool_name in self.tools:
  1415│             return self.tools[tool_name](tool_input)
  1416│         else:
  1417│             return f"错误：未找到工具{tool_name}"
  1418│ 
  1419│ # 使用案例：股票分析
  1420│ tools = {
  1421│     "get_stock_price": lambda x: f"{x}的最新价格是$150.25",
  1422│     "calculate_pe_ratio": lambda x: f"市盈率是25.6",
  1423│     "search_news": lambda x: f"关于{x}的最新新闻：..."
  1424│ }
  1425│ 
  1426│ agent = ReActAgent(llm, tools)
  1427│ result = agent.run("特斯拉当前的估值是否合理？请分析P/E比率和近期新闻")
  1428│ ```
  1429│ 
  1430│ > **专家提示**：ReAct的关键是设计良好的行动空间。行动粒度要适中：太大失去灵活性，太小导致过多迭代。好的行动通常是"原子操作"——不可再分且功能单一。

  1432│ ### 3.6 RAG融合
  1433│ 
  1434│ RAG（Retrieval-Augmented Generation）通过检索外部知识库增强LLM的生成能力，解决知识截止和幻觉问题。
  1435│ 
  1436│ #### 3.6.1 RAG的标准流程
  1437│ 
  1438│ ```markdown
  1439│ ## RAG提示词模板
  1440│ 
  1441│ <system>
  1442│ 你是一个问答助手，可以访问以下检索到的相关文档。
  1443│ </system>
  1444│ 
  1445│ <retrieved_documents>
  1446│ {% for doc in retrieved_docs %}
  1447│ ### 文档{{ loop.index }} (相关性: {{ doc.score }})
  1448│ 来源: {{ doc.source }}
  1449│ 内容: {{ doc.content }}
  1450│ ---
  1451│ {% endfor %}
  1452│ </retrieved_documents>
  1453│ 
  1454│ <instruction>
  1455│ 基于上述检索到的文档回答问题。如果文档中没有相关信息，明确说明"根据现有资料无法回答"。
  1456│ 不要使用文档之外的知识。
  1457│ 引用具体文档编号来支持你的回答。
  1458│ </instruction>
  1459│ 
  1460│ <question>
  1461│ {{ user_question }}
  1462│ </question>
  1463│ 
  1464│ <output_format>
  1465│ ## 回答
  1466│ [你的答案]
  1467│ 
  1468│ ## 依据
  1469│ - 文档{{ doc_id }}: [具体引用内容]
  1470│ - 文档{{ doc_id }}: [具体引用内容]
  1471│ 
  1472│ ## 可信度
  1473│ [高/中/低] - 理由：...
  1474│ </output_format>
  1475│ ```
