    26│ ## Towards Effective and Sparse Adversarial Attacks on Spiking Neural Networks via Breaking Invisible Surrogate Gradients  
    27│ 
    28│ Spiking Neural Networks (SNNs) offer unique advantages in energy efficiency and temporal processing but present distinct challenges for adversarial attacks compared to traditional ANNs. This analysis synthesizes breakthroughs in overcoming gradient invisibility and sparsity constraints, enabling high-success-rate sparse perturbations.  
    29│ 
    30│ ### 1. **Fundamental Challenges in SNN Adversarial Attacks**  
    31│     - **Spatio-Temporal Gradient Acquisition**: SNNs process dynamic inputs across time, requiring gradients that capture both spatial features and temporal dynamics. Traditional ANN methods fail here, but BPTT-inspired algorithms enable precise spatio-temporal gradient maps [citation:1][citation:3].  
    32│     - **Gradient-Spike Incompatibility**: Continuous gradients (e.g., from backpropagation) conflict with binary spike inputs (0/1), preventing direct perturbation application. This necessitates gradient-to-spike conversion [citation:1].  
    33│     - **Vanishing Gradients**: The step-function nature of spiking neurons produces zero-dominated derivatives, causing gradient evaporation and halting sample updates [citation:1].  
    34│     - **Surrogate Gradient Invisibility**: SNNs' non-differentiability requires surrogate gradients for training, but these approximations create "invisible" regions where attacks fail—central to breaking robustness [citation:3].  
    35│ 
    36│ ### 2. **Breakthrough Methods for Gradient Exploitation**  
    37│ #### **A. Gradient-to-Spike (G2S) Converter**  
    38│ Resolves format incompatibility by transforming continuous gradients into ternary values (−1, 0, +1) compatible with spikes:  
    39│ - **Probabilistic Sampling**: Converts gradient magnitudes to spike-flip probabilities, preserving perturbation subtlety.  
    40│ - **Sign Extraction**: Uses gradient polarity to determine spike addition/removal.  
    41│ - **Overflow-Aware Transformation**: Caps perturbations to prevent detectable spike-rate anomalies [citation:1].  
    42│ *Effect*: Reduces pixel-wise perturbations by 70% while maintaining >99% attack success on neuromorphic datasets (N-MNIST, CIFAR10-DVS) [citation:1].  
    43│ 
    44│ #### **B. Gradient Trigger (GT)**  
    45│ Addresses vanishing gradients by injecting controlled randomness:  
    46│ - When gradients are all-zero, GT flips spikes randomly at a tunable rate (e.g., 5–10%).  
    47│ - Dynamically reactivates stalled attacks without excessive perceptibility [citation:1].  
    48│ 
    49│ #### **C. Surrogate Gradient Exploitation**  
    50│ - **Approximation Bypass**: Uses differentiable surrogate functions (e.g., sigmoid derivatives) during attack generation to approximate SNN gradients, creating effective perturbations despite non-differentiability [citation:3].  
    51│ - **Leakage Factor Manipulation**: Increasing leaky integrate-and-fire (LIF) neuron leakage reduces nonlinearity, weakening robustness. Attackers exploit this to amplify perturbation propagation [citation:3].

    55│     - **Loss Function & Threshold Tuning**:
