## AI与编程开发（新增条目 #1-8）

- **[VS JIT调试器错误与修复]** — 当程序抛出未处理win32异常且无启用JIT的调试器时触发"Just-In-Time Debugging failed"错误。
  - 来源文件：实时调试错误翻译.md
  - 核心解释：在Visual Studio中通过 工具→选项→调试→实时 勾选启用即可。错误对话框英文原文为"An unhandled win32 exception occurred in [9660] ExcelAnalyzer.exe"，常见于Excel VBA等COM组件调试场景。

- **[Python虚拟环境创建与激活命令]** — python -m venv创建虚拟环境，跨平台激活命令不同。
  - 来源文件：创建激活Python虚拟环境命令.md
  - 核心解释：Windows CMD用 `myenv\Scripts\activate`，PowerShell用 `Activate.ps1`，Linux/macOS用 `source myenv/bin/activate`。激活后提示符前出现(myenv)字样，退出用deactivate。若系统同时有Python 2和3，需使用python3 -m venv。

- **[DeepSeek API模型版本体系（2026年4月）]** — 官方API主力模型为V3.2（128K上下文），通过两个端点调用。
  - 来源文件：DeepSeek API模型版本.md
  - 核心解释：deepseek-chat端点对应非思考模式（快速响应），deepseek-reasoner端点对应思考模式（展示推理过程）。另有V3.2-Exp实验版、R1推理模型可用。2026年2月Web端已测试百万级上下文新结构，但官方API当时仍未跟进。

- **[DeepSeek V4论文国内下载渠道]** — 首选魔搭社区(ModelScope)和HuggingFace官方，备选CSDN、量子位等。
  - 来源文件：DeepSeekV4论文国内下载渠道.md
  - 核心解释：魔搭下载地址为 modelscope.cn/collections/deepseek-ai/DeepSeek-V4，HuggingFace直接PDF链接为 huggingface.co/deepseek-ai/DeepSeek-V4-Pro/blob/main/DeepSeek_V4.pdf。相关论文还包括mHC流形约束超连接、DualPath智能体推理框架、Engram条件记忆模块。

- **[Ollama社区模型命名规则]** — 格式为"用户名/模型名"，前缀是上传者的Ollama用户名。
  - 来源文件：社区上传模型名称解释.md
  - 核心解释：例如 t1c/deepseek-math-7b-rl 中 t1c 是上传者的Ollama用户名，deepseek-math-7b-rl 才是模型实际名称。Ollama页面会标注"UNOFFICIAL uploads"说明非官方上传。官方模型在HuggingFace上的名称为 deepseek-ai/deepseek-math-7b-rl。

- **[WPS加载项类型与本地路径]** — COM加载项(.dll/.exe)与WPS网页技术加载项(jsaddons目录)两类。
  - 来源文件：WPS加载项查找与添加指南.md
  - 核心解释：WPS加载项本地路径为 %AppData%\kingsoft\wps\jsaddons，启动项路径为 %AppData%\Kingsoft\wps\startup。启用入口在"开发工具"选项卡（如找不到，右键功能区空白处→自定义功能区→勾选"开发工具"）。COM加载项通过"添加"按钮选择.dll文件即可。

- **[AI回复"谨慎甄别"标签的五种触发场景]** — 专业决策、时效信息、主观推测、争议话题、来源模糊五类。
  - 来源文件：AI回复谨慎甄别标签情况说明.md
  - 核心解释：标签核心目的是透明性（告知能力局限性）、责任感（避免盲目依赖）、鼓励批判性思维。建议用户核实权威渠道、交叉验证多来源、咨询专业人士。训练数据截止于2024年7月，此后的变化无法覆盖。
