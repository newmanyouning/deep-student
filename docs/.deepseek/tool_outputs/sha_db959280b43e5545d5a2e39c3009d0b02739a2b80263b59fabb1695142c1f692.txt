  1580│   - 来源文件：合并md文件脚本编写.md
  1581│   - 核心解释：Python 严格依赖缩进来确定代码块边界。修复方法包括：在编辑器中开启"显示空格/Tab"功能检查缩进，手动删除前导空格后重敲 4 个空格，或将编辑器设置为 Tab 键自动替换为 4 个空格。复制网页代码时可能带入不可见特殊字符，可先粘贴到纯文本编辑器再复制到 IDE 中。

  1583│ - **[Windows下Python文件编码问题及解决方案]** — 在 Windows 系统上用 `open()` 读取 UTF-8 编码的文件时，若不指定编码参数，Python 会使用系统默认编码（中文 Windows 下为 GBK），导致 `UnicodeDecodeError`。
  1584│   - 来源文件：合并md文件脚本编写.md
  1585│   - 核心解释：错误信息 `'gbk' codec can't decode byte 0x80` 表示文件中包含 UTF-8 特有的多字节字符（如中文、特殊符号），GBK 无法解码。解决方案是在 `open()` 中显式指定 `encoding='utf-8'`。推荐在系统终端用 `python 文件名.py` 运行脚本，此时 Python 解释器会自动按文件编码读取。
