   261│ ### 🧹 第一步：清理本地的 PaddleOCR 模型文件
   262│ 
   263│ 在 Mac 上，本地安装的模型通常分散在两个地方。你可以用以下命令来查找并删除它们：
   264│ 
   265│ 1.  **查找并删除通过 `pip` 安装的 `paddleocr` 包**
   266│     ```bash
   267│     pip uninstall paddleocr paddlepaddle
   268│     ```
   269│     执行这个命令后，系统会询问你是否确认删除，输入 `y` 即可。
   270│ 
   271│ 2.  **查找并删除自动下载的模型缓存**
   272│     PaddleOCR 的模型文件通常会下载并缓存在你的用户目录下。你可以通过以下命令找到并删除它们：
   273│     ```bash
   274│     # 查找模型缓存目录
   275│     find ~ -type d -name "paddleocr" -o -name ".paddlex" -o -name "PaddleOCR"
   276│     ```
   277│     如果确认这些是你要删除的模型缓存，可以使用 `rm -rf` 命令将它们彻底删除（请务必仔细确认路径）。
   278│

   280│ 
   281│ ### 🐳 第二步：拉取并运行 PaddleOCR-VL 的 Docker 镜像
   282│ 
   283│ 这是最核心的一步。打开终端，直接复制并运行下面的命令。它会自动从百度云的镜像仓库下载一个包含了 PaddleOCR-VL-1.5 模型和所有依赖的完整环境，并启动容器[citation:1][citation:5]。
   284│ 
   285│ ```bash
   286│ docker run -it \
   287│   --shm-size 32g \
   288│   --ipc=host \
   289│   -v $PWD:/workspace \
   290│   ccr-2vdh3abv-pub.cnc.bj.baidubce.com/paddlepaddle/paddlex-paddle-vllm-amd-gpu:3.4.0-0.14.0rc2 \
   291│   /bin/bash
   292│ ```
   293│ 
   294│ **这条命令的各项参数说明如下：**
   295│ *   `-it`：以交互模式运行，让你可以直接在容器内操作。
   296│ *   `--shm-size 32g`：将容器的共享内存设为 32GB，这对处理大图片很重要。
   297│ *   `--ipc=host`：使用主机的 IPC 命名空间，有助于提升进程间通信效率。
   298│ *   `-v $PWD:/workspace`：将你当前所在的 Mac 目录挂载到容器内的 `/workspace` 目录。这样，容器就能直接访问你 Mac 上的图片文件，处理结果也会保存在这里。
   299│ *   最后一段是完整的镜像地址，这个镜像是专门为在 CPU 上运行而优化的，非常适合你的 Mac[citation:5]。
