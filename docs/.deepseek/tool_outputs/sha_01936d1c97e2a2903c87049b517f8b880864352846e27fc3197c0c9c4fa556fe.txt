Claude Code 本身**没有一个命令能直接“自动放行所有命令行操作”**。因为“自动放行”这个概念，在实际使用中表现为不同的自动化程度和安全性。

我把几类常用的命令整理了一下，方便你根据自己的情况进行选择：

| 自动化级别 | 启动命令 | 适用场景 | 一句话说特点 |
| :--- | :--- | :--- | :--- |
| **智能自动模式**<br>（Auto Mode） | `claude --enable-auto-mode` <br>或 <br>`claude --permission-mode auto` | 日常开发、希望平衡效率与安全的场景 | **AI帮你判断**，放行安全操作，危险动作仍会提示，是大多数人的最佳选择[reference:0][reference:1] |

### 📋 命令详解：深入了解，明智选择

#### 1. 智能自动模式 (Auto Mode)：AI 帮你审批
这是平衡效率与安全的最佳方案。当用户输入 `claude --enable-auto-mode` 启动后[reference:5]：
*   **工作方式**：AI（具体由 Claude Sonnet 4.6 模型执行[reference:6]）会像一个智能法官，先审查你的每一个操作意图。
*   **判断标准**：
    *   **自动放行**：被认为是安全、低风险的操作，如`git status`、`ls`、`npm install`（安装已声明的依赖）、在项目目录内读写文件等[reference:7]。
    *   **需要确认**：遇到潜在危险操作会停下来征求你的意见，例如强制推送 (`git push --force`)、直接向主分支提交代码、从网络下载未经审查的代码等[reference:8]。
*   **模式切换**：你可以在会话中随时用 `Shift+Tab` 快捷键切换模式[reference:9][reference:10]。

#### 2. 完全信任模式 (Bypass Mode)：毫不设防的谨慎之选
这个模式会**100% 跳过所有权限检查**，让 Claude 可以执行任何操作而不受限制[reference:11][reference:12]。
*   **适用场景**：**仅在绝对受控、完全隔离的环境（如临时的 Docker 容器）中，用于执行绝对信任的脚本或任务**[reference:13][reference:14]。
*   **风险极高**：**千万避免在包含个人文件、代码仓库、密钥等敏感信息的真实环境中使用**。一个错误的命令就可能导致无法挽回的数据丢失或安全泄露[reference:15]。

#### 3. 自定义放行列表 (Allowed Tools)：精准授权
这种方法允许你**“白名单化”特定工具，实现精细化的自动化**[reference:16]。
*   **工作方式**：你可以通过 `--allowedTools` 参数指定一组工具，Claude 执行这些工具时无需再向你确认[reference:17]。例如：
    ```bash
    claude --permission-mode acceptEdits --allowedTools "Read,Write,Edit,Bash,Git"
    ```
*   **风险收益**：这种方法让你可以放心地把机器信任的任务交给 Claude 自动完成，同时保持对其他操作的控制，是寻求“最佳平衡点”的不错选择。

### 📁 持久化配置：一次设置，长久省心
如果你选定了一个模式并想固定下来，可以通过配置文件，避免每次启动都输入命令。
在 `~/.claude/settings.json` （macOS/Linux）文件中进行设置[reference:18]：
```json
{
  "permissions": {
    "defaultMode": "auto"  // 你可以将 "auto" 替换为 "plan" 或 "bypass"
  }
}
```
