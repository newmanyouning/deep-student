   451│ ## 🔧 方案二：给终端授予“完全磁盘访问权限”
   452│ 
   453│ 如果你希望脚本以后也能正常执行（比如未来需要再次修改配置），可以给终端授权，让它能写入这些受保护的系统目录。
   454│ 
   455│ ### 步骤：
   456│ 1. 打开 **系统设置** → **隐私与安全性** → **完全磁盘访问权限**。
   457│ 2. 点击左下角的 🔒 解锁，然后点 **+** 号。
   458│ 3. 在应用程序中找到 **终端**（Terminal.app），添加进去。
   459│ 4. 确认终端旁边开关已开启。
   460│ 5. **完全退出终端**（⌘ + Q），重新打开终端，再次运行之前的脚本。
   461│ 
   462│ 授权后，脚本就能顺利写入配置文件了。

   524│ ## 1️⃣ 确认正确的配置文件名和位置
   525│ 
   526│ 在 Docker Desktop 的 macOS 版本中，配置文件名和路径可能因版本不同而存在差异。常见的有两个：
   527│ 
   528│ - **`settings.json`** – 早期版本使用的配置文件。
   529│ - **`settings-store.json`** – 较新版本（约 4.10 以后）使用的配置文件，位于相同目录。
   530│ 
   531│ **你的目录路径**：  
   532│ `~/Library/Group Containers/group.com.docker/DockerDesktop/`
   533│ 
   534│ 在这个文件夹下，请检查是否存在 **两个文件**：`settings.json` 和 `settings-store.json`。  
   535│ 如果 **只有 `settings-store.json`**，那说明 Docker 应该读取它；但如果 **两个文件都存在**，Docker 可能会优先读取 `settings.json` 而忽略 `settings-store.json`。
   536│ 
   537│ **操作**：  
   538│ - 先查看文件夹内文件列表（可以在 Finder 中查看，或终端执行 `ls -la`）。  
   539│ - 如果 **存在 `settings.json`**，请同样在其中添加 `"i18nLocale": "zh-CN"`（注意 JSON 格式）。  
   540│ - 如果 **只有 `settings-store.json`**，但修改后无效，请尝试 **删除 `settings.json`**（如果存在）或 **重命名备份**，让 Docker 只读取 `settings-store.json`。
   541│

   544│ ## 2️⃣ 检查 JSON 格式是否正确
   545│ 
   546│ 你给出的内容显示：
   547│ ```
   548│ {
   549│   "AutoStart": false,
   550│   "DockerAppLaunchPath": "/Applications/Docker.app",
   551│   "EnableDockerAI": true,
   552│   "LastContainerdSnapshotterEnable": 1774269321,
   553│   "LicenseTermsVersion": 2,
   554│   "SettingsVersion": 43,
   555│   "ShowInstallScreen": false,
   556│   "UseContainerdSnapshotter": true,
   557│   "UseVirtualizationFrameworkRosetta": false,
   558│   "i18nLocale": "zh-CN"
   559│ }
   560│ ```
   561│ 
   562│ 这看起来格式正确。但请确保 **文件开头有 `{`**，末尾有 `}`，并且所有键值对之间用逗号分隔（这里最后一行 `"i18nLocale": "zh-CN"` 后面没有逗号，这是正确的）。
   563│ 
   564│ **注意**：有时编辑器会添加额外字符，如 BOM 头，或者保存为 RTF 等格式。建议用纯文本编辑器（如 VS Code、Sublime Text 或终端中的 `nano`）打开，确保保存为 UTF-8 无 BOM。

   568│ ## 3️⃣ 清除 Docker Desktop 的缓存和偏好设置
   569│ 
   570│ 即使配置文件正确，Docker Desktop 可能缓存了旧的设置。执行以下命令彻底清除缓存：
   571│ 
   572│ ```bash
   573│ # 完全退出 Docker Desktop
   574│ osascript -e 'quit app "Docker Desktop"'
   575│ 
   576│ # 删除偏好设置文件（如果存在）
   577│ rm -f ~/Library/Preferences/com.docker.docker.plist
   578│ rm -f ~/Library/Preferences/com.docker.helper.plist
   579│ 
   580│ # 删除可能残留的缓存目录（可选，但安全）
   581│ rm -rf ~/Library/Caches/com.docker.docker
   582│ 
   583│ # 重新启动 Docker Desktop
   584│ open -a "Docker Desktop"
   585│ ```
   586│ 
   587│ 等待 Docker 完全启动后（菜单栏图标稳定），再检查界面语言。

   591│ ## 4️⃣ 确认 Docker Desktop 版本是否支持中文
   592│ 
   593│ 从你配置中的 `"SettingsVersion": 43` 来看，这应该是较新的版本（通常 SettingsVersion 42 及以上对应 Docker Desktop 4.10+）。

   595│ 
   596│ - 点击菜单栏 Docker 图标 → **About Docker Desktop**。

   598│ 
   599│ 如果版本较低，即便配置了 `i18nLocale` 也可能无效。

   603│ ## 5️⃣ 测试中文语言包是否可用
   604│ 
   605│ 即使配置正确，如果当前安装的 Docker Desktop 中不包含中文语言文件，也会显示英文。可以通过临时启动来测试：
   606│ 
   607│ ```bash
   608│ LANG=zh_CN.UTF-8 open -a "Docker Desktop"
   609│ ```
   610│ 
   611│ - 如果启动后是 **中文**，说明语言包存在，问题出在配置文件未被读取。
   612│ - 如果启动后 **仍是英文**，说明 Docker Desktop 安装包可能不包含中文语言（极小概率，通常官方版都包含）。此时可考虑重新安装 Docker Desktop。

   615│ 
   616│ ## 6️⃣ 终极方案：重置 Docker Desktop 配置
   617│ 
   618│ 如果以上都不行，可以重置所有配置，让 Docker 重新生成默认文件，然后再添加语言设置。
   619│ 
   620│ 1. **完全退出 Docker Desktop**。
   621│ 2. **备份并删除整个配置目录**（注意：这会丢失所有自定义设置，如镜像加速器等）：
   622│    ```bash
   623│    mv ~/Library/Group\ Containers/group.com.docker ~/Library/Group\ Containers/group.com.docker.bak
   624│    ```
   625│ 3. **启动 Docker Desktop**，它会自动重新创建配置目录和默认的 `settings-store.json`。
   626│ 4. **再次手动编辑**新生成的 `settings-store.json`，添加 `"i18nLocale": "zh-CN"`。
   627│ 5. **重启 Docker Desktop**。
