Windows中设置网关IP地址，通常是在TCP/IPv4属性中，填写“默认网关”这一项。在macOS中，对应的概念也是“默认网关”或“路由器”。设置位置是在“系统设置”的“网络”里，选择对应网络接口（如Wi-Fi或以太网），点击“详细信息”，然后在“TCP/IP”选项卡中，找到“路由器”一栏。那里填写的IP地址就是网关地址。

所以核心对应关系是：Windows的“默认网关”对应macOS的“路由器”。需要明确指出这个对应关系，并简要说明在macOS中的具体设置路径和位置名称，让用户能直接操作。
在 macOS 中，Windows 的“网关 IP 地址”对应的是**“路由器”**这一设置项。

具体位置在：**系统设置 > 网络 > 选择网络服务（如 Wi-Fi）> 点“详细信息” > TCP/IP 选项卡**。在这里看到的 **“路由器”** 一栏，就等同于 Windows 中的默认网关地址。

简单来说：Windows 叫“默认网关”，Mac 叫“路由器”，两者功能相同，都是填写同一个 IP 地址。

Mac上“路由器”地址显示为灰色通常是因为当前网络连接使用的是动态主机配置协议（DHCP），即由路由器自动分配网络信息，因此这个字段不可手动修改。

解决方法是将IPv4的配置从DHCP模式切换为手动模式，具体可以通过图形界面或命令行来实现：

### 🖥️ 图形界面设置步骤
1.  打开 **“系统设置”** > **“网络”**。
2.  在列表中找到并点选当前使用的网络服务（如“Wi-Fi”或“以太网”），点击 **“详细信息”**[reference:0]。
3.  进入 **“TCP/IP”** 标签页，将“配置IPv4”从“使用DHCP”改为 **“手动”**[reference:1]。
4.  此时“路由器”字段即可编辑，按实际情况输入正确的网关地址、IP地址和子网掩码后点击 **“好”** 保存[reference:2]。*注：如有锁形图标，需点击解锁[reference:3]。*

### ⌨️ 命令行设置方法
如果习惯使用终端，可以使用 `networksetup` 命令来完成，这种方式特别适合需要脚本化操作的场景[reference:4]：

1.  打开 **“终端”**（位于“应用程序 > 实用工具”）。
2.  执行以下命令列出所有网络服务，确认你的目标服务名称（例如 `Wi-Fi` 或 `Ethernet`）[reference:5]：
    ```bash
    networksetup -listallnetworkservices
    ```
3.  执行以下命令来设置静态IP、子网掩码和网关地址（路由器）[reference:6]：
    ```bash
    sudo networksetup -setmanual "网络服务名称" IP地址 子网掩码 网关地址
    ```
    *例如，将Wi-Fi服务的IP设为`192.168.1.100`，子网掩码`255.255.255.0`，网关`192.168.1.1`：*
    ```bash
    sudo networksetup -setmanual "Wi-Fi" 192.168.1.100 255.255.255.0 192.168.1.1
    ```
    *需注意，手动设置IP前，请确认该地址未被网络中其他设备使用，以免造成地址冲突。*

#### 📌 重要提示
*   通过 `route` 命令添加网关是**临时**的，重启或网络切换后可能失效[reference:7][reference:8]。
*   若想恢复动态获取，只需在图形界面将“配置IPv4”重新选回“使用DHCP”[reference:9]，或在终端使用 `sudo networksetup -setdhcp "Wi-Fi"` 命令[reference:10]。

好的，选了“手动”但“路由器”字段依然无法修改，确实挺奇怪的。通常这和一些更底层的配置问题或软件冲突有关，可以按下面的步骤尝试解决：

### 🔓 检查系统权限
查看“网络”设置窗口左下角是否有锁图标。如果图标是闭合的，点击它并输入密码解锁[reference:0][reference:1]，然后再尝试操作。

### 🖥️ 使用终端命令配置
图形界面卡住时，终端是更直接有效的办法。
1.  打开 **“终端”** (在“应用程序” > “实用工具”中)。
2.  输入以下命令查看所有网络服务列表，并记下你正在配置的服务的**准确名称** (例如 `Wi-Fi` 或 `USB 10/100/1000 LAN`)。
    ```bash
    networksetup -listallnetworkservices
    ```
3.  使用 `-setmanual` 命令进行配置。请将命令中的 `"服务名称"`、`你的IP地址`、`你的子网掩码` 和 `你的网关地址` 替换为实际值[reference:2]：
    ```bash
    sudo networksetup -setmanual "服务名称" 你的IP地址 你的子网掩码 你的网关地址
    ```
    *例如：`sudo networksetup -setmanual "Wi-Fi" 192.168.1.100 255.255.255.0 192.168.1.1`*

### 🗑️ 重置网络配置
网络配置文件损坏也可能导致问题，重置能清除潜在的错误配置。
1.  在终端中运行以下两个命令，这会**删除所有网络配置**，并重启电脑：
    ```bash
    sudo rm -f /Library/Preferences/SystemConfiguration/preferences.plist
    sudo reboot
    ```
    重启后，所有网络设置将恢复为默认值，你可以重新进行配置[reference:3][reference:4]。

### 🧐 排查第三方软件冲突
某些VPN客户端（如Cisco AnyConnect）或安全软件可能会接管网络设置，导致手动配置失效[reference:5]。建议**暂时完全退出或卸载**这类软件后再尝试。

### 🔧 其他可能的解决方案
*   **创建新的网络位置**：在“系统设置” > “网络” > “位置”中，点击“编辑位置”，新建一个位置并命名为“手动”之类的名称，然后在新位置下重新尝试设置[reference:6]。
*   **检查物理接口**：如果你用的是USB转接的有线网卡，请尝试换一个USB端口。同时，也可以重启路由器和Mac，清除可能的临时缓存问题[reference:7]。

如果以上方法都无效，建议你联系Apple官方支持以获得更专业的帮助。
