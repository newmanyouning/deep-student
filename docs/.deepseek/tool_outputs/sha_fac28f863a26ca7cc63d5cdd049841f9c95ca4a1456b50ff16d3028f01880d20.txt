**Theme 1 — 为什么推荐用 Docker 部署模型（知识点提炼）：**

1. **解决"依赖地狱"** (lines 25-33) — 模型依赖链复杂（CUDA、cuDNN、nccl、Python包版本），Docker容器化实现"一次构建，处处运行"，避免系统级库冲突导致服务器崩溃。
2. **环境隔离与安全性** (lines 34-38) — 资源隔离（限制CPU/内存/GPU），防止OOM导致物理机宕机；安全沙箱隔离不可信代码。
3. **简化运维与微服务架构** (lines 40-49) — 模型部署常为多微服务组合（API服务器、推理引擎、向量数据库、Redis、Nginx），Docker Compose/K8s通过配置文件一键启动、水平伸缩、自动重启。
4. **硬件抽象与GPU透传** (lines 51-56) — `nvidia-container-toolkit` 使容器内CUDA直接调用宿主机NVIDIA驱动，性能损耗低于1%-3%；运维不关心宿主机驱动版本。
5. **研发与生产环境一致性** (lines 57-60) — 确保开发/测试/生产环境二进制、库文件、配置完全一致，消除"在我电脑上能跑"问题。
6. **快速迭代与回滚** (lines 62-67) — 升级/回滚只需切换镜像，几秒钟完成，避免直接物理机操作导致的系统污染。
7. **云原生与大规模调度** (lines 69-73) — Kubernetes要求服务容器化，才能实现自动扩缩容、负载均衡、故障自愈。

8. **总结与类比** (lines 77-86) — 本质是"模型生态碎片化"与"生产环境稳定性要求"的矛盾。类比："自热预制菜" vs "自己买菜做饭"。注意：个人学习/本地调试用 conda 更轻量；对外服务/团队协作/生产环境 Docker 几乎必选。
