    12│ ## ❓ unknown
    13│ 
    14│ 在树莓派5上运行的Ubuntu系统，我们可以将Java项目打包成可运行的程序。通常Java项目可以打包成可执行的JAR文件，或者使用jlink创建自定义运行时，或者使用jpackage创建安装包（但jpackage在ARM64上可能有限制）。另外，也可以将Java程序编译成原生可执行文件（例如使用GraalVM）。
    15│ 
    16│ 由于树莓派5是ARM64架构，我们需要根据这个架构来打包。以下是一些方法：
    17│ 
    18│ ## 方法1：打包为可执行的JAR文件（最常用）
    19│ 1. 确保已经安装了Java开发工具包（JDK）。
    20│ 2. 将Java项目编译成JAR文件，并指定主类（Main-Class）。
    21│ 3. 创建一个启动脚本，以便运行JAR文件。
    22│ 
    23│ ### 步骤：
    24│ 1. 安装JDK：
    25│    ```bash
    26│    sudo apt update
    27│    sudo apt install openjdk-17-jdk  # 或者你需要的版本
    28│    ```
    29│ 
    30│ 2. 编译项目（如果使用Maven或Gradle）：
    31│    - Maven: `mvn clean package`
    32│    - Gradle: `gradle build`
    33│ 
    34│    如果项目没有使用构建工具，则手动编译并打包：
    35│    ```bash
    36│    javac -d bin src/*.java
    37│    jar cfe myapp.jar com.example.Main -C bin .
    38│    ```
    39│ 
    40│ 3. 创建一个运行JAR文件的脚本，例如`run.sh`：
    41│    ```bash
    42│    #!/bin/bash
    43│    java -jar /path/to/your/myapp.jar
    44│    ```
    45│ 
    46│ 4. 给脚本执行权限：`chmod +x run.sh`
    47│ 
    48│ ## 方法2：使用jlink创建自定义运行时（Java 9及以上）
    49│ jlink可以创建一个包含最小Java运行时的镜像，这样用户就不需要安装完整的JDK。
    50│ 
    51│ 1. 确保安装了JDK 9或以上。
    52│ 2. 使用jlink创建一个自定义运行时：
    53│    ```bash
    54│    jlink --add-modules java.base,java.desktop --output myruntime
    55│    ```
    56│    你需要根据项目使用的模块来调整。
    57│ 
    58│ 3. 使用这个运行时来运行你的JAR：
    59│    ```bash
    60│    myruntime/bin/java -jar myapp.jar
    61│    ```
    62│ 
    63│ ## 方法3：使用jpackage创建安装包（如果支持）
    64│ jpackage可以将Java应用程序打包为平台特定的安装包（如deb、rpm等）。但是，jpackage在ARM64上可能不是所有版本都支持，需要检查你的JDK版本。
    65│ 
    66│ 1. 确保安装了JDK 14或以上（支持jpackage的版本）。
    67│ 2. 使用jpackage打包：
    68│    ```bash
    69│    jpackage --input target/ \
    70│             --name MyApp \
    71│             --main-jar myapp.jar \
    72│             --main-class com.example.Main \
    73│             --type deb \
    74│             --java-options '--enable-preview'
    75│    ```
    76│ 
    77│    注意：jpackage可能需要一些依赖，如`fpm`、`rpm-build`等，具体取决于打包的类型。
    78│ 
    79│ ## 方法4：使用GraalVM将Java程序编译为原生可执行文件
    80│ GraalVM可以将Java程序编译成原生可执行文件，这样就不需要JVM来运行了。但是，GraalVM对ARM64的支持可能有限，需要检查是否有ARM64版本。
    81│ 
    82│ 1. 安装GraalVM（确保选择ARM64版本）和Native Image工具。
    83│ 2. 使用Native Image编译：
    84│    ```bash
    85│    native-image -jar myapp.jar myapp
    86│    ```
    87│    这将在当前目录生成一个名为`myapp`的可执行文件。
    88│ 
    89│ ## 在树莓派5上安装打包好的程序
    90│ 
    91│ ### 对于deb包（使用jpackage或手动打包成deb）：
    92│ ```bash
    93│ sudo dpkg -i myapp.deb
    94│ ```
    95│ 
    96│ ### 对于可执行JAR或脚本：
    97│ 你可以将JAR文件和启动脚本放在一个目录中，然后创建一个桌面文件（.desktop）以便在应用菜单中显示。
    98│ 
    99│ 1. 将JAR文件和脚本放在一个目录，例如`/opt/myapp/`。
   100│ 2. 创建一个桌面文件`/usr/share/applications/myapp.desktop`：
   101│    ```
   102│    [Desktop Entry]
   103│    Type=Application
   104│    Name=MyApp
   105│    Comment=My Java Application
   106│    Exec=/opt/myapp/run.sh
   107│    Icon=/opt/myapp/icon.png
   108│    Terminal=false
   109│    Categories=Utility;
   110│    ```
   111│ 
   112│ 3. 确保脚本和JAR文件有适当的权限。
   113│ 
   114│ ## 总结
   115│ 对于树莓派5（ARM64）上的Ubuntu，推荐使用方法1（可执行JAR）或方法4（GraalVM原生可执行文件，如果有ARM64版本）。如果项目简单，方法1是最直接的。如果希望获得更好的启动性能和更小的依赖，可以尝试方法4。

   118│ 在树莓派5的Ubuntu系统上打包和安装Java项目，有以下几种方法：
   119│ 
   120│ ## 方法一：打包为可执行JAR文件（最简单）
   121│ 
   122│ ### 1. 确保Java环境已安装
   123│ ```bash
   124│ # 安装JDK
   125│ sudo apt update
   126│ sudo apt install default-jdk
   127│ # 验证安装
   128│ java -version
   129│ javac -version
   130│ ```
   131│ 
   132│ ### 2. 编译和打包项目
   133│ ```bash
   134│ # 创建项目结构
   135│ mkdir -p myapp/src/myapp
   136│ mkdir -p myapp/classes
   137│ 
   138│ # 创建示例Java文件
   139│ cat > myapp/src/myapp/Main.java << 'EOF'
   140│ package myapp;
   141│ public class Main {
   142│     public static void main(String[] args) {
   143│         System.out.println("Hello from MyApp!");
   144│     }
   145│ }
   146│ EOF
   147│ 
   148│ # 编译Java文件
   149│ javac -d myapp/classes myapp/src/myapp/Main.java
   150│ 
   151│ # 创建可执行JAR
   152│ echo "Main-Class: myapp.Main" > manifest.txt
   153│ jar cvfm myapp.jar manifest.txt -C myapp/classes .
   154│ ```
   155│ 
   156│ ### 3. 创建安装脚本
   157│ ```bash
   158│ cat > install.sh << 'EOF'
   159│ #!/bin/bash
   160│ # 创建安装目录
   161│ sudo mkdir -p /opt/myapp
   162│ # 复制JAR文件
   163│ sudo cp myapp.jar /opt/myapp/
   164│ # 创建启动脚本
   165│ sudo tee /usr/local/bin/myapp << 'EOT'
   166│ #!/bin/bash
   167│ java -jar /opt/myapp/myapp.jar "\$@"
   168│ EOT
   169│ sudo chmod +x /usr/local/bin/myapp
   170│ echo "安装完成！使用 'myapp' 命令运行程序"
   171│ EOF
   172│ 
   173│ chmod +x install.sh
   174│ ./install.sh
   175│ ```
