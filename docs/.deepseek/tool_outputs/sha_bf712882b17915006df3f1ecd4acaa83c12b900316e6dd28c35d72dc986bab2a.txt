+### 基于 Shell 的文件监控自动提交方案
+**标签**: Git, 自动化, Shell 脚本, Bash
+**要点**:
+1. 使用 while true + sleep 构建轮询循环，每30秒检查一次仓库状态。
+2. git status --porcelain 以简洁格式输出状态，管道到 grep -q . 检测是否存在变更。
+3. 使用 PID 文件（.git-auto-commit.pid）防止脚本重复运行，trap 捕获 SIGINT/SIGTERM 实现优雅退出。
+4. nohup 后台运行并重定向日志到文件，配合 start/stop 脚本实现启停管理。
+
+**关键命令**:
+```bash
+#!/bin/bash
+PID_FILE=".git-auto-commit.pid"
+if [ -f "$PID_FILE" ]; then
+    echo "系统已在运行 (PID: $(cat $PID_FILE))"
+    exit 1
+fi
+echo $$ > "$PID_FILE"
+trap 'rm -f "$PID_FILE"; exit 0' SIGINT SIGTERM
+
+while true; do
+    if git status --porcelain | grep -q .; then
+        git add . 2>/dev/null
+        git commit -m "自动提交: $(date '+%Y-%m-%d %H:%M:%S')" --quiet 2>/dev/null
+    fi
+    sleep 30
+done
+
+# 后台启动: nohup ./git-auto-commit.sh > .auto-commit.log 2>&1 &
+# 停止: pkill -f git-auto-commit.sh
+```

+### Node.js + chokidar 文件监控自动提交
+**标签**: Git, Node.js, 自动化
+**要点**:

+3. 通过防抖（debounce）合并短时间内的多次变更，避免频繁提交产生大量垃圾记录。
+4. 使用 child_process.exec 依次执行 git add . 和 git commit。

+**关键代码**:
+```javascript
+const chokidar = require('chokidar');
+const { exec } = require('child_process');
+
+const watcher = chokidar.watch('.', {
+    ignored: /(^|[/\\])\\.|node_modules/,
+    persistent: true,
+    awaitWriteFinish: { stabilityThreshold: 2000 }
+});
+
+let debounceTimer;
+watcher.on('change', () => {
+    clearTimeout(debounceTimer);
+    debounceTimer = setTimeout(() => {
+        exec('git add . && git commit -m "自动提交" --quiet');
+    }, 3000);
+});
