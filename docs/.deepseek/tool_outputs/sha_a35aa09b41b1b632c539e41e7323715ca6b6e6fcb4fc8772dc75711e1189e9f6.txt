    61│ ### 🤔 为什么会出现“a+b=10”的答案？
    62│ 要得到“a+b=10”的输出，前提必须是`a`或`b`的数值被改变，或者变量本身发生了变化。以下是几种最可能的情况：
    63│ 
    64│ **1. 变量被重新赋值**
    65│ 这是最常见的原因。在`printf`语句执行前，变量`a`或`b`的值被再次修改了。
    66│ ```c
    67│ int a=0, b=0;
    68│ a = 10;
    69│ b = 20;
    70│ b = 10; // b被重新赋值为10
    71│ printf("a+b=%d", a+b); // 输出：a+b=20
    72│ // 或者 a = 0; 则输出 a+b=10
    73│ ```
    74│ 
    75│ **2. 同名变量（作用域嵌套）**
    76│ 在C语言中，内部代码块可以定义与外部同名的变量，遵循“就近原则”[citation:5]。内部的`a`会暂时“遮盖”外部的`a`。
    77│ ```c
    78│ int a=0, b=0;
    79│ a = 10;
    80│ b = 20;
    81│ {
    82│     int a = 0; // 这是一个新的、只在此代码块内有效的变量a
    83│     printf("a+b=%d", a+b); // 这里的a是内部变量0，输出：a+b=20
    84│     // 如果内部 b=10; 则输出 a+b=10
    85│ }
    86│ ```
    87│ 
    88│ **3. 使用了复合赋值或增量运算符**
    89│ 运算符的副作用可能在无意中改变了变量的值。
    90│ ```c
    91│ int a=0, b=0;
    92│ a = 10;
    93│ b = 20;
    94│ a += b; // 等价于 a = a + b; 执行后a变为30
    95│ // 或者 a = b; 执行后a变为20
    96│ printf("a+b=%d", a+b); // 输出结果将取决于修改后的值
    97│ ```
