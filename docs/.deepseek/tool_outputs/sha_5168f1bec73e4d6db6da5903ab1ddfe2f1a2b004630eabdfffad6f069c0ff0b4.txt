   441│ 
   442│ 另外用户提到“确认了代理服务正在运行”，但端口可能是其他？但用户之前的错误显示连接 127.0.0.1:7890 失败，说明代理服务要么没在 7890 监听，要么不是 HTTP 代理。用户应确认代理软件的 HTTP 代理端口。但鉴于用户说代理服务正在运行，可能是端口不匹配，但更简单的做法是让用户彻底禁用 Git 代理，而不是依赖代理。因为用户 ping github.com 是通的，直接直连应该可以。
   443│ 
   444│ 所以解决步骤：检查所有 Git 配置中的代理并清除，然后重试。
   445│ 
   446│ 最后提示：检查代理软件的实际 HTTP 代理端口，如果确实需要代理，设置正确的端口。
   447│ 你的 Git 配置中仍然残留了指向 `127.0.0.1:7890` 的代理，但该端口的代理服务可能没有运行或不是 HTTP 代理。既然 `ping` 正常，建议**彻底移除 Git 代理**，直接直连 GitHub。

   451│ **解决步骤（无需 sudo）：**
   452│ 
   453│ 1. **查看当前所有 Git 配置中的代理设置（全局 + 本地）**：
   454│    ```bash

   456│    git config --global --get-regexp https.*proxy
   457│    git config --local --get-regexp http.*proxy   # 在项目目录下执行
   458│    git config --local --get-regexp https.*proxy
   459│    ```
   460│ 
   461│ 2. **清除全局和本地的所有代理配置**：
   462│    ```bash
   463│    git config --global --unset-all http.proxy
   464│    git config --global --unset-all https.proxy
   465│    git config --local --unset-all http.proxy      # 如果有本地配置
   466│    git config --local --unset-all https.proxy
   467│    ```
   468│

   470│    ```bash
   471│    git config --global --list | grep proxy

   473│    ```
