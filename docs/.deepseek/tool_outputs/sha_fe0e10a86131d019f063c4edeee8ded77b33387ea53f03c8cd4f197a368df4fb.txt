   143│         *   表示是**上下文动态的**：同一个Token（如“bank”）在不同句子中（“河岸” vs “金融机构”）会拥有不同的向量表示，因为Transformer的自注意力机制会根据周围的Token动态调整其表示。这是LLM理解歧义和复杂语义的关键。
   144│ 
   145│ 3.  **粒度**：
   146│     *   **关键词**：通常是**词级**的。有时会扩展到N-Grams（词组）以捕获一些短语信息，但本质上还是离散的词或词组单元。
   147│     *   **Token**：粒度更灵活，取决于分词策略：
   148│         *   **词级**：一个词一个Token（常见于一些中文分词或早期模型）。
   149│         *   **子词级**：最主流的方式（如Byte Pair Encoding - BPE， WordPiece, SentencePiece）。将词拆分成更常见的子单元（如前缀、后缀、词根）或字符。例如：
   150│             *   “unhappiness” -> `["un", "happi", "ness"]`
   151│             *   “学习” -> `["学", "习"]` (取决于分词器)
   152│         *   **字符级**：每个字符作为一个Token（较少见，效率低）。
   153│         *   这种子词级粒度极大地缓解了**未登录词**问题，因为新词可以被拆分成已知的子词来表示。
   154│ 
   155│ 4.  **上下文与关系**：
   156│     *   **关键词**：在传统词袋模型中，**忽略顺序和上下文**。“猫追老鼠”和“老鼠追猫”可能被表示为相同的特征向量（包含“猫”、“追”、“老鼠”）。顺序信息和词间关系基本丢失。
   157│     *   **Token**：**顺序和上下文至关重要**。Transformer架构的核心（自注意力机制和前馈网络）显式地建模Token之间的依赖关系，无论它们相距多远。模型能理解“猫追老鼠”和“老鼠追猫”是完全不同的意思，以及代词指代、长距离依赖等复杂语言现象。
   158│ 
   159│ 5.  **特征学习**：
   160│     *   **关键词**：特征（哪些词是重要的关键词）很大程度上**依赖人工设计和选择**（特征工程）。模型主要学习这些预定义特征的权重。
   161│     *   **Token**：模型**自动学习**从原始Token序列到有意义的内部表示的映射。通过海量数据的预训练（如掩码语言建模、下一个Token预测），模型学会了丰富的语言知识，并将这些知识编码在Token的嵌入表示和模型参数中。特征学习是端到端过程的一部分。
   162│ 
   163│ 6.  **处理新词/稀有词**：
   164│     *   **关键词**：如果遇到词汇表中没有的新词或非常稀有的词（OOV问题），传统方法通常无法有效处理（可能直接忽略或用特殊标记代替，损失信息）。
   165│     *   **Token**：得益于子词分词，即使一个词没在训练数据中完整出现过，其子词单元很可能出现过，模型可以利用这些子词的组合来表示和理解新词。例如，即使没训练过“抗核抗体”，模型知道“抗”、“核”、“抗体”的含义，能进行一定程度的组合推理。
   166│ 
   167│ **总结**：
   168│ 
   169│ *   **关键词** 是传统机器学习中**人工选择或加权的、离散的、上下文无关的**文本特征单元，主要用于构建分类或检索模型的输入特征向量。
   170│ *   **Token** 是大语言模型中**经过分词器切分后的、原始的、连续的（通过嵌入）、上下文动态的**基本输入单元。它们是神经网络理解和生成语言的基石，模型通过复杂的架构（如Transformer）在这些Token及其交互关系上学习丰富的语言知识和表示。
   171│ 
   172│ 简而言之，关键词是**特征工程的终点**，是模型的**输入特征**；而Token是**模型处理的起点**，是**学习表示的基础**。LLM通过处理Token序列，在内部自动学习到了比传统关键词方法更丰富、更灵活、更上下文感知的“特征”或“概念”表示，这是其强大能力的关键来源之一。
