    45│ 
    46│ - 模型下载到 44% 时中断。
    47│ - 提示“max retries exceeded”（超过最大重试次数），即多次尝试下载同一数据块失败。
    48│ - 具体原因：`read tcp ... connection reset by peer`，表示 TCP 连接被远程服务器主动重置（相当于服务器端突然关闭了连接）。
    49│ 
    50│ ### 可能的原因
    51│ 1. **网络不稳定**：Wi-Fi 信号弱、网络波动大，导致连接中断。
    52│ 2. **服务器端临时问题**：Ollama 使用的 Cloudflare R2 存储可能出现瞬时故障或高负载。
    53│ 3. **IPv6 连接问题**：你的机器使用了 IPv6 地址（`[2409:...]`），而目标服务器是 IPv6（`[2606:...]`），某些网络环境下 IPv6 路由可能不稳定或被阻断。
    54│ 4. **防火墙或代理干扰**：本地防火墙、VPN 或代理软件可能中断了下载连接。
    55│ 5. **DNS 解析问题**：域名解析到的 IP 可能不可达。
    56│ 
    57│ ### 解决方法
    58│ 1. **重试下载**  
    59│    直接再次运行 `ollama run deepseek-coder:6.7b`，Ollama 会从断点续传，一般临时问题会解决。
    60│ 
    61│ 2. **检查网络连接**  
    62│    - 确保网络稳定，可以尝试 ping 一下 `dd20bb891979d25aebc8bec07b2b3bbc.r2.cloudflarestorage.com` 或使用 `curl` 测试连通性。
    63│    - 如果使用 Wi-Fi，可以切换到有线网络或换个热点试试。
    64│ 
    65│ 3. **禁用 IPv6 或调整优先级**  
    66│    有时 IPv6 连接不稳定，可临时让系统优先使用 IPv4：
    67│    - macOS：在“系统设置” → “网络” → 选择当前网络 → “详细信息” → “TCP/IP” → 将“配置 IPv6”改为“仅限本地链接”或“关闭”。
    68│    - 或者通过终端临时禁用 IPv6（需重启网络）：  
    69│      `sudo networksetup -setv6off Wi-Fi`（将 Wi-Fi 换成你的网络服务名）。
    70│ 
    71│ 4. **关闭代理/VPN**  
    72│    如果你开启了 VPN 或代理，尝试关闭后再下载，因为某些代理可能干扰大型文件传输。
    73│ 
    74│ 5. **更换 DNS**  
    75│    尝试将 DNS 改为公共 DNS（如 114.114.114.114 或 8.8.8.8），避免 DNS 解析到不稳定的节点。
    76│ 
    77│ 6. **使用镜像加速**  
    78│    如果多次失败，可以考虑配置 Ollama 镜像（例如使用国内镜像站），具体参考 Ollama 官方文档或社区提供的镜像地址。
