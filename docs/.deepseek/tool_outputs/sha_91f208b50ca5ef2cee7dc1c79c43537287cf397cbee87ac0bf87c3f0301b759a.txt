   318│ ## 第一步：确定默认用户的用户名
   319│ 
   320│ 在容器内执行以下命令，找到容器启动时使用的用户（通常是 PID 1 的进程所有者）：
   321│ 
   322│ ```bash
   323│ ps aux | grep "^USER\|^root\|^app" | head -5
   324│ ```
   325│ 
   326│ 或者直接查看 `/etc/passwd` 中除了 root 和系统服务外的普通用户：
   327│

   329│ grep -E ":/home/" /etc/passwd
   330│ ```
   331│ 
   332│ 如果输出包含类似 `appuser:x:1000:1000:,,,:/home/appuser:/bin/bash` 的行，则默认用户就是 `appuser`（或其他名称）。  
   333│ 若没有输出，可能容器始终以 root 运行，那么“默认用户”就是 root，但这种情况通常不会出现你最初描述的包不一致问题。
   334│ 
   335│ 假设找到的默认用户名为 `appuser`，家目录为 `/home/appuser`。如果找不到，你可以通过 `docker inspect` 从宿主机查看：
   336│ 
   337│ ```bash
   338│ docker inspect <容器名或ID> --format '{{.Config.User}}'
   339│ ```
   340│ 
   341│ 若输出为空或 `root`，则默认用户是 root，问题不存在；否则会显示用户名或 UID。

   358│ ## 第三步：选择转移方案
   359│ 
   360│ ### 方案一：重新安装为系统级包（推荐，干净可靠）
   361│ 
   362│ 将 root 用户级包重新安装到系统目录（如 `/usr/local/lib/python3.x/site-packages`），所有用户都能看到。
   363│ 
   364│ ```bash
   365│ # 1. 卸载用户级包（可选，避免冲突）
   366│ pip3 uninstall -y -r /tmp/root_packages.txt
   367│ 
   368│ # 2. 以 root 身份将包安装到系统目录
   369│ pip3 install -r /tmp/root_packages.txt   # 不加 --user，默认装到系统
   370│ ```
   371│ 
   372│ **优点**：包全局可用，无需针对每个用户单独操作。  
   373│ **缺点**：需要联网重新下载（如果本地有 wheel 缓存则可避免）。
   374│

   377│ ### 方案二：复制 root 用户级包目录给默认用户（快速，但有风险）
   378│ 
   379│ 直接将 root 的 `site-packages` 目录复制到默认用户的家目录下，并修正权限。
   380│ 
   381│ ```bash
   382│ # 假设 Python 版本为 3.10（请根据实际修改）
   383│ ROOT_SITE_PACKAGES="/root/.local/lib/python3.10/site-packages"
   384│ USER_SITE_PACKAGES="/home/appuser/.local/lib/python3.10/site-packages"
   385│ 
   386│ # 创建目标目录并复制
   387│ mkdir -p "$USER_SITE_PACKAGES"
   388│ cp -r "$ROOT_SITE_PACKAGES"/* "$USER_SITE_PACKAGES"/
   389│ 
   390│ # 修改所有权
   391│ chown -R appuser:appuser /home/appuser/.local
   392│ ```
   393│ 
   394│ **优点**：无需网络，速度快。  
   395│ **缺点**：  
   396│ - 某些包含有编译的二进制扩展（`.so`），复制后可能因路径依赖失效。
