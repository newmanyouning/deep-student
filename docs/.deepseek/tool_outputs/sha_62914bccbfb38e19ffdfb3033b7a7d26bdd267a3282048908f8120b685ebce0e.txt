   486│ ### 方案 C：本地预处理测试输出（立即实施）
   487│ 
   488│ 无论使用什么 Agent，您都可以在本地包装测试命令，让 Agent 只能看到精简后的结果。
   489│ 
   490│ 创建一个脚本 `smart_test.sh` (Linux/Mac) 或 `smart_test.bat` (Windows)：
   491│ 
   492│ ```bash
   493│ #!/bin/bash
   494│ # 运行测试，并将输出重定向到临时文件
   495│ pytest --tb=short --no-header -q 2>&1 | tee /tmp/test_output.txt
   496│ 
   497│ # 只保留失败行，并限制行数
   498│ grep -E "FAILED|ERROR|AssertionError|E       " /tmp/test_output.txt | head -20 > /tmp/filtered.txt
   499│ 
   500│ # 如果输出超过 2000 字符，截断
   501│ if [ $(wc -c < /tmp/filtered.txt) -gt 2000 ]; then
   502│     head -c 1000 /tmp/filtered.txt > /tmp/short.txt
   503│     echo "\n... [截断] ...\n" >> /tmp/short.txt
   504│     tail -c 1000 /tmp/filtered.txt >> /tmp/short.txt
   505│     cat /tmp/short.txt
   506│ else
   507│     cat /tmp/filtered.txt
   508│ fi
   509│ ```
   510│ 
   511│ 然后在 Agent 配置中，将测试命令从 `pytest` 改为 `bash smart_test.sh`。这样 Agent 收到的测试输出永远不超过 2000 字符（约 500 token）。
