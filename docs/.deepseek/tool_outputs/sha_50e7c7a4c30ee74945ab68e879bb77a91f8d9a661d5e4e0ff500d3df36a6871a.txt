- **[确定容器ID或名称]** — 使用 `docker ps` 列出运行中的容器，获取 CONTAINER ID 或 NAMES。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：在查看 Docker 容器内模型版本前，首先需要在宿主机执行 `docker ps` 命令，查看当前正在运行的容器列表。输出中的 CONTAINER ID 列（如 `abc123def456`）和 NAMES 列（如 `my_model_container`）都可以作为后续进入容器的标识符。

- **[进入容器交互式Shell]** — 使用 `docker exec -it <container> /bin/bash` 进入容器内部。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：通过 `docker exec -it` 命令可在运行中的容器内启动一个交互式 shell 会话，`-it` 选项保证终端可交互。大多数 Linux 容器默认提供 `/bin/bash`，若提示 `bash: not found`，可改用 `/bin/sh`。成功进入后提示符变为容器内部环境。

- **[Python包版本查询]** — 通过 pip show 或 Python 一行命令查看模型依赖包的版本。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：如果模型是通过 TensorFlow、PyTorch 等 Python 包部署的，在容器内使用 `pip show <包名>` 可查看包的详细信息（含 Version 字段）。也可用 `python -c "import <包名>; print(<包名>.__version__)"` 快速获取版本号，例如 `pip show torch` 或 `python -c "import torch; print(torch.__version__)"`。

- **[系统包管理器版本查询]** — 使用 apt 或 dpkg 命令查看通过系统包管理器安装的模型版本。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：当模型是通过 apt 系统包管理器安装时（较少见），在容器内执行 `apt list --installed | grep <关键词>` 或 `dpkg -l | grep <关键词>` 可以过滤出已安装的相关软件包及其版本信息。

- **[可执行文件版本查询]** — 对编译好的可执行文件运行 `--version` 参数获取版本。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：当模型以编译好的二进制可执行文件形式存在时（如 llama.cpp），在其所在目录执行 `<程序名> --version` 通常可以直接输出版本号。示例：`./main --version` 可用于 llama.cpp 的主程序。

- **[环境变量与文件中查询版本]** — 从环境变量或特定文件中提取模型版本信息。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：部分容器在构建时会将版本号写入环境变量或文件。在容器内执行 `env | grep VERSION` 可筛选包含 VERSION 关键字的环境变量；执行 `cat /path/to/version.txt` 可读取显式存放版本号的文件；Dockerfile 中定义的变量也可能是版本信息的来源。

- **[模型文件名识别版本]** — 通过列出模型目录中带版本号的文件名来判断版本。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：模型文件本身的名字经常包含版本标识，例如 `bert-base-uncased`、`llama-2-7b.Q4_K_M.gguf`。在容器内使用 `ls -l /path/to/model/` 列出模型存放目录，直接从文件名中就能识别版本号或模型规格。

- **[退出容器宿主机恢复]** — 在容器内输入 `exit` 即可退出并返回宿主机环境。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：完成容器内的版本查询后，执行 `exit` 命令可安全退出交互式 shell，回到宿主机的原始终端环境，不影响容器的持续运行。

- **[容器外查询镜像版本]** — 当容器内无版本信息时，从宿主机通过镜像标签和历史进行追溯。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：如果容器构建时未显式记录版本，可在宿主机执行 `docker images | grep <镜像名>` 查看镜像标签（如 `my-model:v1.2`），标签通常暗示版本；执行 `docker history <镜像名>:<标签>` 可查看镜像构建历史中是否记录版本；执行 `docker inspect <container>` 则可查看容器启动时的完整环境变量配置。

- **[Hugging Face模型版本查看]** — 检查 Hugging Face 模型缓存目录，目录名含版本哈希。
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：对于部署了 Hugging Face 模型的容器，在容器内执行 `ls ~/.cache/huggingface/hub/` 可以列出模型缓存目录。Hugging Face 会将下载的模型按仓库组织和版本哈希存储，目录名中的哈希值可用于精确定位模型的具体版本。
