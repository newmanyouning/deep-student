  1606│ ### 🧐 调研结论：为什么不能“直接添加”
  1607│ 
  1608│ 1.  **产品定位不同**：根据 `cc-switch` 的官方介绍，它是一款专注于**管理配置和切换模型**的桌面应用，而不是一个通用的 UI 工具集成平台[reference:0][reference:1]。
  1609│ 2.  **核心功能不匹配**：`cc-switch` 的主要能力在于**模型切换**和**MCP配置管理**，并未提供用于集成或启动外部自定义 UI 工具的接口[reference:2][reference:3]。
  1610│ 3.  **管理范围限定**：`cc-switch` 明确说明了它能够管理 **Claude Code、Codex、Gemini CLI 等 5 款主流 AI 编程 CLI 工具**[reference:4]，这意味着 `deepseek-tui` 不在此列。
  1611│ 
  1612│ ### 💡 解决方案：利用 MCP 实现“间接”统一管理
  1613│ 
  1614│ 虽然不能直接添加，但我们可以用“曲线救国”的思路，通过 **MCP 服务器**作为桥梁，实现统一管理。
  1615│ 
  1616│ #### 核心思路
  1617│ 1.  在 `cc-switch` 中安装一个“**MCP 管理工具**”（例如 `mcp-registry` 或 `mcp-tools-manager`），它可以用来发现、安装和管理各种 MCP 服务器。
  1618│ 2.  通过这个管理工具，安装你需要的 MCP 服务器（比如 `duckduckgo-mcp-server`）。
  1619│ 3.  `cc-switch` 会将安装的 MCP 服务器配置同步到它所管理的 AI CLI 工具（例如 Claude Code）。
  1620│ 4.  最后，你可以手动将 `cc-switch` 同步出的 MCP 配置，应用到 `deepseek-tui` 上。
  1621│ 
  1622│ #### 具体操作步骤
  1623│ 
  1624│ **第一步：在 cc-switch 中添加“MCP 管理工具”**
  1625│ 1.  打开 `cc-switch`，进入 “MCP” 管理界面。
  1626│ 2.  点击“添加”按钮，选择“Stdio” 类型[reference:5]。
  1627│ 3.  在命令配置中，输入一个 MCP 管理工具的安装命令，例如：
  1628│     *   `npx @anthropic-ai/mcp-tools-manager` (具体工具名请以实际为准)
  1629│ 4.  保存配置。
  1630│ 
  1631│ **第二步：通过 cc-switch 安装 DuckDuckGo 搜索工具**
  1632│ 1.  重启 `cc-switch`，确保新添加的“MCP 管理工具”处于 “Connected” 状态。
  1633│ 2.  在 `cc-switch` 中，通过该管理工具的命令行界面（如果提供）或交互界面，执行搜索工具的安装命令：
  1634│     ```bash
  1635│     # 安装 DuckDuckGo MCP 服务器
  1636│     npx @anthropic-ai/mcp-tools-manager install duckduckgo-mcp-server
  1637│     ```
  1638│ 3.  安装成功后，检查 `cc-switch` 的 MCP 列表中是否已出现 `duckduckgo` 工具，并确认其状态为 “Connected”[reference:6]。
  1639│ 
  1640│ **第三步：验证 cc-switch 的同步效果**
  1641│ 1.  打开 `Claude Code`，在对话中输入 `/mcp`，检查 `duckduckgo` 工具是否已出现在列表中并处于可用状态。
  1642│ 2.  这验证了 `cc-switch` 确实能够管理 MCP 配置。
  1643│ 
  1644│ **第四步：将 MCP 配置应用到 deepseek-tui**
  1645│ 1.  **查找配置文件**：在 `cc-switch` 同步配置后，尝试在其数据目录或 Claude Code 的配置目录（通常为 `~/.claude/` 或 `%APPDATA%\Claude\`）中，找到与 `duckduckgo` 相关的配置信息[reference:7]。
  1646│ 2.  **手动复制配置**：将找到的配置信息（`command` 和 `args` 字段），手动复制到 `deepseek-tui` 的配置文件 `C:\Users\1\.deepseek\mcp.json` 中[reference:8][reference:9][reference:10]。
  1647│ 3.  **重启并验证**：保存 `deepseek-tui` 的配置文件，并重启 `deepseek-tui`。在对话中输入 `/mcp`，检查 `duckduckgo` 工具是否已可用。

  1649│ ### ⚠️ 注意事项
  1650│ 
  1651│ *   **工具名称**：以上步骤中使用的 “MCP 管理工具” 名称是示例，你需要自行调研当前主流的、可以实现MCP服务器管理的工具。
