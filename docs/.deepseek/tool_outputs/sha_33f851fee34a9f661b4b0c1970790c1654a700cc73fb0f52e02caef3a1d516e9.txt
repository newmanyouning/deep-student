   124│ 如果你从头写一个 `Modelfile`，核心在于 `FROM` 指令要指向你已经下载好的基础模型[citation:1][citation:4][citation:10]。
   125│ 
   126│ 1.  创建一个 `Modelfile` 文件，内容如下：
   127│     ```dockerfile
   128│     # 这一行直接写你已有的模型名字，而不是重新下载
   129│     FROM deepseek-r1:7b
   130│     
   131│     # 定义新角色
   132│     SYSTEM """
   133│     你现在是一位充满智慧的法师，说话方式神秘，喜欢用比喻。
   134│     """
   135│     
   136│     # 可选：微调一下温度，让回复更有“法师”的随性感觉
   137│     PARAMETER temperature 0.9
   138│     ```
   139│ 
   140│ 2.  创建模型：
   141│     ```bash
   142│     ollama create deepseek-r1:7b-法师 -f ./Modelfile
   143│     ```
   144│ 
   145│ ### 3. 运行时的内存占用
   146│ 
   147│ 当你运行 `ollama run deepseek-r1:7b` 和 `ollama run deepseek-r1:7b-法师` 时：
   148│ 
   149│ -   **如果你只运行其中一个**：内存占用和之前一样（约 4-5 GB）。
   150│ -   **如果你同时运行两个**：Ollama 支持并发，但因为是同一个基础模型，它会**共享内存中的权重部分**，额外占用的内存极少，主要是用于维护各自的对话上下文（KV Cache）[citation:2][citation:5]。
   151│ 
   152│ 你可以随时用 `ollama list` 查看当前所有模型，新创建的模型体积那一列会显示为很小的值，或者直接用 `ollama ps` 查看运行时的内存复用情况[citation:2]。
   153│ 
   154│ 创建完新模型后，需要我帮你梳理一下，如何通过调整 `temperature` 或 `top_p` 等参数，让这个“法师”角色的回答风格更符合你的预期吗？
