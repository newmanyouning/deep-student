    24│ Claude Desktop 是 Anthropic 推出的一个**桌面版 AI 助手全家桶**。但在配置管理方面，它与 Claude Code 完全独立，并不能管理后者的配置。
    25│ 
    26│ ### 🤖 Claude Desktop 是什么？
    27│ 
    28│ 你可以把它理解为一个集成了多种 AI 工作模式的桌面软件，把不同的 AI 工作模式都整合在了一个应用里[reference:0]。它主要通过三个标签页来满足不同场景的需求[reference:1]：
    29│ 
    30│ *   **Chat**：用于提问、写作、分析文档等通用对话[reference:2]。
    31│ *   **Code**：内嵌的AI编程环境，可直接在项目文件夹里修改代码[reference:3][reference:4]。
    32│ *   **Cowork**：在云端自动执行任务的后台工作模式[reference:5]。
    33│ 
    34│ 对于开发者，Code 模式功能强大，支持可视化代码差异对比、多会话并行、内嵌预览等多种编程场景。
    35│ 
    36│ ### ⚙️ 那它能管理 Claude Code 的配置吗？
    37│ 
    38│ **答案是：不能。** 它们是两个独立的产品，拥有两套完全独立的配置系统[reference:6]。
    39│ 
    40│ *   **Claude Desktop** 的配置通过应用本身的图形界面来完成，设置项不会影响 Claude Code[reference:7]。
    41│ *   **Claude Code CLI** 的配置则通过编辑家目录下的 `~/.claude/` 文件夹中的 `settings.json` 等文件来管理[reference:8][reference:9]。
    42│ 
    43│ 这说明 Desktop 应用内的 Code 模块是一个**集成功能**，而非 Claude Code CLI 的配置管理器。二者的核心差异可以参考下表：
    44│ 
    45│ | 对比维度 | ✨ Claude Desktop (App) | ⚙️ Claude Code (CLI) |
    46│ | :--- | :--- | :--- |
    47│ | **产品形态** | 一个集成了多种功能的**桌面应用**，是“全家桶”[reference:10]。 | 一个专注于编程的**命令行工具**，是“引擎”[reference:11]。 |
    48│ | **配置方式** | 通过**应用内的图形界面** (`Settings`) 修改，存储在应用目录下[reference:12]。 | 通过编辑 **`~/.claude/` 目录下的 JSON 配置文件**来管理，与 Desktop 完全隔离[reference:13]。 |
    49│ | **主要用途** | 提供 **Chat、Code、Cowork** 三种模式，覆盖通用对话、编程、后台任务等多种场景[reference:14]。 | **专门为编程任务设计**，在终端中使用，适合深度集成到个人开发工作流中[reference:15]。 |
    50│ | **项目感知** | **不支持**项目级配置，所有设置均为全局生效[reference:16]。 | **支持**项目级配置（如项目专属的 `.mcp.json` 或 `.claude/` 目录），更灵活[reference:17]。 |
    51│ | **指令文件支持** | **不支持** `CLAUDE.md` 文件[reference:18]。 | **支持** `CLAUDE.md` 文件，可通过它在项目或全局层面设定行为指令[reference:19]。 |
    52│ 
    53│ **💎 总结一下**：如果你想直接管理 Claude Code CLI 的核心文件或 JSON 配置，需要去它的安装目录手动编辑。Desktop 应用只负责它自己内部 Code 模式的一些基本设置，无法“跨应用”管理 CLI 的深层配置。

    91│ ### 🚀 方案一：快速接入图像生成（推荐新手）
    92│ 
    93│ ModelScope 官方将图像生成 MCP 工具打包发布在了 npm 上，配置最简单。
    94│ 
    95│ **1. 获取 API Key**
    96│ 登录 [ModelScope 官网](https://modelscope.cn)，进入「个人中心」→「API 密钥管理」，创建并复制你的 API Key（格式通常为 `ms-...`）[reference:4][reference:5]。
    97│ 
    98│ **2. 安装与配置 MCP 服务**
    99│ 你可以通过全局安装或直接使用 `npx` 来运行服务。打开终端并执行：
   100│ 
   101│ **方式一：npm 全局安装（推荐）**
   102│ ```bash
   103│ npm install -g moda-image-mcp
   104│ ```
   105│ 
   106│ **方式二：直接使用 npx（无需预安装）**
   107│ 无需单独安装，Claude Code 会在需要时自动处理。
   108│ 
   109│ 然后，向 Claude Code 声明这个 MCP 服务器。根据你想要生效的范围，配置文件位置不同：
   110│ *   **仅当前项目生效**：在项目根目录下创建或编辑 `.mcp.json` 文件[reference:6][reference:7]。
   111│ *   **所有项目全局生效**：编辑用户目录下的 `~/.claude.json` 文件[reference:8]。
   112│ 
   113│ 在对应文件中加入以下配置：
   114│ ```json
   115│ {
   116│   "mcpServers": {
   117│     "moda-image": {
   118│       "command": "npx",
   119│       "args": ["-y", "moda-image-mcp"],
   120│       "env": {
   121│         "MODA_API_KEY": "你的魔搭API密钥"
   122│       }
   123│     }
   124│   }
   125│ }
   126│ ```
   127│ 如果已将 `moda-image-mcp` 包全局安装（方式一），你也可以使用更简洁的写法[reference:9]：
   128│ ```json
   129│ {
   130│   "mcpServers": {
   131│     "moda-image": {
   132│       "command": "moda-image-mcp",
   133│       "env": {
   134│         "MODA_API_KEY": "你的魔搭API密钥"
   135│       }
   136│     }
   137│   }
   138│ }
   139│ ```
   140│ **✅ 完成验证**：重启 Claude Code，在对话中输入 `/mcp` 查看 `moda-image` 是否在列表中[reference:10][reference:11]。

   144│ ### ⚙️ 方案二：接入对话模型 (CCR)
   145│ 
   146│ 通过 Claude Code Router (CCR)，可以使用 ModelScope 的 Qwen 系列代码模型。
   147│ 
   148│ **1. 安装 CCR 与 ModelScope 配置工具**
   149│ ```bash
   150│ # 安装 Claude Code Router
   151│ npm install -g @leason/claude-code-router
   152│ # 安装 ModelScope 配置工具
   153│ npm install -g @leason/claude-code-config
   154│ ```
   155│ 
   156│ **2. 配置 API Key**
   157│ 推荐使用环境变量，即永久设置[reference:12]。例如将：
   158│ ```bash
   159│ export MODELSCOPE_API_KEY="你的ModelScope_API_KEY"
   160│ ```
   161│ 添加到你的 shell 配置文件（如 `.zshrc` 或 `.bashrc`）中。
   162│ 
   163│ **3. 生成配置文件**
   164│ ```bash
   165│ ccr-modelscope
   166│ ```
   167│ 这条命令会在 `~/.claude-code-router/config.json` 自动生成一份可用的配置文件，默认集成了 `Qwen3-Coder` 等模型[reference:13]。
   168│ 
   169│ **4. 启动 Claude Code**
   170│ 之后，你需要通过 CCR 来启动 Claude Code：
   171│ ```bash
   172│ ccr code
   173│ ```
   174│ 现在，Claude Code 的底层模型已切换到 ModelScope，你可以开始使用了[reference:14]。
   175│ 
   176│ ---
   177│ 
   178│ ### 🔧 方案三：从 MCP 广场接入其他工具
   179│ 
   180│ ModelScope 的 MCP 广场提供了更多样的工具。接入方式通常是 MCP 客户端标准流程：
   181│ 
   182│ 1.  访问 [ModelScope MCP 广场](https://modelscope.cn)，寻找需要的工具（如“Fetch 网页内容抓取”）[reference:15]。
   183│ 2.  在工具详情页，复制其提供的 **Streamable HTTP** 配置脚本[reference:16]。
   184│ 3.  将复制的配置脚本粘贴到你的 `.mcp.json` 或 `~/.claude.json` 配置文件中。
   185│ 4.  重启 Claude Code 即可生效。
   186│ 
   187│ ---
   188│ 
   189│ ### 💎 核心参数速查与文件作用域说明
   190│ 
   191│ *   **API Key**：所有方案都需要，格式为 `ms-...`，或者通过 `env` 环境变量注入[reference:17]。
   192│ *   **图像生成默认模型**：`Qwen/Qwen-Image`，支持 `prompt`（必填）、`model`、`save_path` 参数[reference:18]。
   193│ *   **CCR 集成模型**：包括 `Qwen/Qwen3-Coder-480B-A35B-Instruct`、`Qwen3-235B-A22B-Thinking-2507` 等[reference:19]。
   194│ *   **MCP 广场工具**：根据官方指引，通过 Streamable HTTP、SSE 等方式连接[reference:20][reference:21]。
   195│ 
   196│ 此外，不同作用域的 MCP 配置会写入到不同文件：编辑 `~/.claude.json` 作为用户级全局配置[reference:22]，或在项目级配置中使用 `.mcp.json` 并在其中引用 `mcpServers` 对象[reference:23]——这样做便于团队共享和版本管理[reference:24]。
   197│ 
   198│ 如果遇到服务未加载、API 报错或生成超时，请检查 Node.js 版本不低于 18、API 密钥有效、网络连接正常，并核对 `~/.claude.json` 配置中 `command` 路径是否匹配[reference:25][reference:26]。
