   211│ ### **方法二：安装官网下载的 .tar.gz 压缩包（最通用）**
   212│ 如果你从 [Adoptium](https://adoptium.net/) 或 Oracle 官网下载了 `*-linux-aarch64_bin.tar.gz` 文件，请按以下步骤操作：
   213│ ```bash
   214│ # 1. 进入下载目录，假设压缩包名为 jdk-17_linux-aarch64_bin.tar.gz
   215│ cd ~/Downloads
   216│ 
   217│ # 2. 创建Java安装目录（如果不存在）
   218│ sudo mkdir -p /usr/lib/jvm
   219│ 
   220│ # 3. 解压压缩包到该目录
   221│ sudo tar -xzf jdk-17_linux-aarch64_bin.tar.gz -C /usr/lib/jvm/
   222│ 
   223│ # 4. 配置环境变量，让系统知道Java的位置
   224│ # 编辑你的shell配置文件（如果你不确定就用 ~/.bashrc）
   225│ echo 'export JAVA_HOME=/usr/lib/jvm/jdk-17.0.9' >> ~/.bashrc
   226│ echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
   227│ 
   228│ # 5. 使配置立即生效
   229│ source ~/.bashrc
   230│ ```
   231│ **注意**：`jdk-17.0.9` 是示例，实际解压出的文件夹名需根据你下载的版本确定，请使用 `ls /usr/lib/jvm` 命令查看。

   233│ ### **方法三：安装官网下载的 .deb 包**
   234│ 如果你下载的是 `*_aarch64.deb` 包，可以用Ubuntu的原生工具安装。
   235│ ```bash
   236│ # 1. 进入下载目录
   237│ cd ~/Downloads
   238│ 
   239│ # 2. 使用dpkg命令安装（以jdk-17_linux-aarch64_bin.deb为例）
   240│ sudo dpkg -i jdk-17_linux-aarch64_bin.deb
   241│ 
   242│ # 3. 如果报告依赖问题，运行以下命令修复
   243│ sudo apt -f install
   244│ ```

   246│ ### **重要提示与验证**
   247│ *   **绝对不要安装x64或.rpm包**：对于 `.rpm` 格式的包，虽然可以通过 `alien` 工具转换，但在树莓派上极易出错，**强烈不建议**。
   248│ *   **如何验证安装成功**：无论用哪种方法，安装后都请在终端执行：
   249│     ```bash
   250│     java -version
   251│     ```
   252│     如果输出信息中包含 **“openjdk version”** 和 **“aarch64”** 字样，就说明ARM64版本的Java已正确安装。
