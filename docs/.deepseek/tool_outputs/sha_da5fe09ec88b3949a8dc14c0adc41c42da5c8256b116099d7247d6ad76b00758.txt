    26│ 在云服务器的终端里，查看GPU占用情况最通用、最核心的命令是 **`nvidia-smi`** [citation:1][citation:2][citation:7]。
    27│ 
    28│ 这个命令会显示GPU的实时状态，包括显存使用率、GPU计算单元利用率、温度、功耗等关键信息 [citation:2][citation:7]。
    29│ 
    30│ ### 💻 基础用法与输出解读
    31│ 
    32│ 直接在终端输入以下命令即可：
    33│ 
    34│ ```bash
    35│ nvidia-smi
    36│ ```
    37│ 
    38│ 执行后，你会看到类似下图的输出，重点关注两个核心指标 [citation:2]：
    39│ *   **`GPU-Util`**： 表示GPU计算核心的利用率。如果这个数值接近100%，说明计算单元正在全力工作。
    40│ *   **`Memory-Usage`**： 表示显存的使用情况（已用/总量）。这能帮你判断显存是否足够或已满。
    41│ 
    42│ ### 🚀 实用进阶命令
    43│ 
    44│ 除了基础查看，还有一些更实用的用法：
    45│ 
    46│ *   **持续监控（实时刷新）**：如果你想动态观察GPU使用率的变化，可以加上 `-l` 参数指定刷新间隔（秒）：
    47│     ```bash
    48│     nvidia-smi -l 1   # 每1秒刷新一次
    49│     ```
    50│     这个命令非常适合在运行训练任务时，实时监控资源占用 [citation:1][citation:2][citation:6]。在Linux系统中，也可以结合 `watch` 命令达到类似效果 [citation:4][citation:8][citation:10]。
    51│ 
    52│ *   **只看关键信息**：如果觉得默认输出太繁杂，可以用 `--query-gpu` 自定义输出格式，方便查看或记录日志：
    53│     ```bash
    54│     # 只查询GPU索引、名称、利用率、显存使用情况
    55│     nvidia-smi --query-gpu=index,name,utilization.gpu,memory.used,memory.total --format=csv
    56│     ```
    57│     这样可以获得一个简洁的、逗号分隔的信息列表 [citation:5][citation:10]。
    58│ 
    59│ *   **查看是谁在用GPU**：想要找出具体是哪个进程（PID）在占用GPU资源，可以使用：
    60│     ```bash
    61│     nvidia-smi pmon  # 监控GPU上的进程
    62│     # 或者查询更详细的计算进程信息
    63│     nvidia-smi --query-compute-apps=pid,process_name,used_memory --format=csv
    64│     ```
    65│     这能帮你定位到具体的任务 [citation:2][citation:10]。
    66│ 
    67│ ### ✨ 更好用的第三方工具
    68│ 
    69│ 如果你觉得 `nvidia-smi` 的输出不够直观，可以试试这两个社区常用的Python工具，它们提供更友好的终端显示 [citation:3]：
    70│ 
    71│ 1.  **gpustat**：对 `nvidia-smi` 的输出进行美化，以更紧凑、彩色的方式展示信息。
    72│     ```bash
    73│     pip install gpustat
    74│     gpustat -cp -i   # 显示进程名和PID，并持续刷新
    75│     ```
    76│ 
    77│ 2.  **nvitop**：堪称GPU界的 `top` 命令，可以实时、交互式地查看GPU和进程信息，强烈推荐。
    78│     ```bash
    79│     pip install nvitop
    80│     nvitop
    81│     ```
    82│ 
    83│ ### ⚠️ 重要前提
    84│ 
    85│ 请务必确保你的云服务器已经安装了 **NVIDIA 驱动**，否则系统会提示 `command not found` [citation:1][citation:7][citation:9]。如果还没有安装驱动，可以先执行 `lspci | grep -i nvidia` 来确认系统是否识别到了NVIDIA的硬件 [citation:7]。
