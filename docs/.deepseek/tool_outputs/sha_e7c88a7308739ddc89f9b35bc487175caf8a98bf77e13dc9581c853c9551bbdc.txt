## AI与编程开发（新增条目 #9-16）

- **[AI与人类对刺激反应的差异]** — AI没有本能感受，其"反应"本质是模式匹配和数据计算，非主观体验。
  - 来源文件：AI对色情图片的反应机制.md
  - 核心解释：从技术角度，能触发AI强烈反应的东西包括训练数据中的极端分布模式、对抗性样本（人眼难察觉的噪点可使分类置信度99%变为错误结果）、以及安全机制设定的红线内容。AI的反应"只是数学计算和模式匹配的过程"。

- **[WPS加载项三步启用流程]** — 环境准备→功能启用→加载项部署。
  - 来源文件：WPS加载项启用配置指南.md
  - 核心解释：①安装Node.js和wpsjs脚手架（npm install -g wpsjs）；②在cfgs/oem.ini添加 [Support] JsApiPlugin=true，个人版用wpsjs debug；③部署三种方式：wpsjs debug调试模式、publish.html在线分发、jsplugins.xml企业统一配置。如加载项被禁用，在authaddin.json中将enable改为true。

- **[Claude Code国内源切换与升级]** — 推荐npm config set registry https://registry.npmmirror.com（阿里巴巴镜像，延迟仅几分钟）。
  - 来源文件：Claude Code国内源切换与升级指南.md
  - 核心解释：升级优先用 claude update，备用 npm install -g @anthropic-ai/claude-code。版本未变更时先 npm cache clean --force 清理缓存。验证升级成功用 claude --version / npm list -g @anthropic-ai/claude-code / npm view @anthropic-ai/claude-code version 三条命令。备用源可选腾讯云镜像。

- **[Claude Code技能生态与安全风险]** — 无统一官方商店，社区平台有SkillsMP(80万+)、skills.sh(9万+)等，最大风险是SKILL.md自然语言指令劫持。
  - 来源文件：Claude Code技能市场指南.md
  - 核心解释：SKILL.md风险在于看似无害的英语句子可诱骗AI执行恶意命令（如 curl发送环境变量到攻击者服务器）。安全建议：优先选官方仓库和带安全检查的平台（如SkillsLLM），安装前检查SKILL.md是否有敏感指令。生态系统总计90万+被索引技能。

- **[需求文档8类常见缺陷]** — 结构混乱、颗粒度不均、缺非功能性需求、缺验收标准、术语不一致、业务流程缺失、技术约束未说明、排版语病。
  - 来源文件：需求文档问题分析.md
  - 核心解释：改进建议包括按功能模块重组（基础数据/组织权限/检测业务/门户消息等），每条需求补充优先级、可量化的验收条件、依赖关系、界面原型参考，并附上业务流程图和标准术语表。

- **[FileWriteTool代码潜在问题]** — 异常处理不完整、未验证绝对路径、缺少路径沙箱保护。
  - 来源文件：cc-mini写文件源码问题分析.md
  - 核心解释：代码仅捕获OSError，未覆盖UnicodeEncodeError（content包含无法编码字符时崩溃）和IsADirectoryError等。行数计算逻辑本身正确（统计\n数量，非换行结尾+1）。建议增加绝对路径验证和更广泛的异常捕获。

- **[Ghostty终端模拟器核心特征]** — GPU加速渲染+Zig语言编写+原生框架+可嵌入库libghostty。
  - 来源文件：Ghostty开源终端模拟器介绍.md
  - 核心解释：macOS用Swift/AppKit原生框架，Linux用Zig+GTK4，避免Electron迟钝感。支持Kitty图形协议（终端内显示图片）、字体连字、100+配置主题。macOS安装 brew install ghostty，Arch Linux可从官方仓库安装。由HashiCorp联合创始人Mitchell Hashimoto发起，已获5万+ GitHub Stars。

- **[Claude Code三种权限模式]** — 智能自动模式/完全信任模式/自定义白名单模式。
  - 来源文件：Claude自动放行命令.md
  - 核心解释：①--enable-auto-mode（AI判断放行安全操作如git status，危险操作如git push --force仍需确认）；②--dangerously-skip-permissions（完全跳过所有检查，仅限CI/CD沙箱环境）；③--permission-mode acceptEdits --allowedTools "Read,Write,Edit,Bash,Git"（精准白名单授权）。可持久化到 ~/.claude/settings.json 的 permissions.defaultMode。
