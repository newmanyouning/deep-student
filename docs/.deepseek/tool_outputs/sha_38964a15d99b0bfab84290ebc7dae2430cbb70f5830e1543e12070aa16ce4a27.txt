    12│ ## ❓ unknown
    13│ 
    14│ 嗯，用户遇到了一个问题：下载到本地的数学模型，在发送“简单介绍你自己”这样的指令时，没有输出预期的模型信息，而是输出了类似“我是张三爱好跑步”之类的无关信息。
    15│ 
    16│ 用户想知道应该检查模型的什么设置。这个问题涉及模型的使用配置。需要分析可能的原因：可能是模型加载了错误的对话模板（如使用了角色扮演模板），或者系统提示词被覆盖，又或者模型本身是经过微调的特定版本（如角色扮演模型）。需要引导用户检查模型的加载方式、模板设置、系统提示等关键配置。
    17│ 
    18│ 想到了可以给出具体的检查步骤，比如检查transformers库的加载参数、tokenizer的对话模板、是否设置了system prompt、模型来源等。回答要清晰、有条理，帮助用户定位问题。
    19│ 你遇到的情况，本质上是**模型加载了错误的“对话模板”或“系统提示词”**。
    20│ 
    21│ 当你下载一个本地数学模型（如 DeepSeek-Math、Qwen2.5-Math 等）时，它本身是**基础模型（Base Model）** 或**经过特定微调（SFT）的模型**。正常情况下，发送“简单介绍你自己”，它应该回复“我是 DeepSeek-Math，一个专注于数学问题的模型……”之类的信息。
    22│ 
    23│ 如果它回复“我是张三，爱好跑步”，说明模型**没有以正确的“角色”启动**，而是被强制套用了另一个对话场景。你需要重点检查以下 3 个方面的设置：
    24│ 
    25│ ### 1. 检查“系统提示词”是否被错误覆盖
    26│ 这是最常见的原因。你在使用模型时（无论是通过 Ollama、LM Studio、Text Generation WebUI 还是 Python 脚本），可能设置了一个**全局系统提示词**。
    27│ 
    28│ -   **现象**：系统提示词里写了“你叫张三，爱好跑步”。
    29│ -   **解决**：
    30│     -   **LM Studio / Ollama**：查看右侧或顶部的“System Prompt”或“Instructions”栏，清空其中的内容。
    31│     -   **Text Generation WebUI**：检查“Parameters” -> “Instruction template”是否选错。如果给数学模型套用了“ChatML”以外的、带有角色扮演性质的模板，就会出现这种问题。
    32│     -   **Python 代码**：检查 `messages` 列表中 `role: system` 的内容，删掉或改为标准提示词。
    33│ 
    34│ ### 2. 检查“对话模板”是否匹配
    35│ 如果你是通过 **transformers** 库加载的模型，且使用了 `pipeline` 或 `apply_chat_template`，但**没有指定正确的 `tokenizer.chat_template`**，模型可能会“张冠李戴”。
    36│ 
    37│ -   **现象**：你的代码可能是为一个通用聊天模型（如 Mistral）写的，但加载的是数学模型的权重。代码中的模板强制让模型以“虚构人物”的身份回复。
    38│ -   **解决**：
    39│     -   确保你使用的是模型官方仓库推荐的加载方式。
    40│     -   如果你在加载时手动传入了 `tokenizer.chat_template`，建议先注释掉，让 tokenizer 自动加载模型自带的模板。
    41│ 
    42│ ### 3. 检查是否下载错了模型权重
    43│ 有时候模型文件（`.gguf` 或 `.safetensors`）本身可能是**经过特定角色扮演数据微调过的变体**，而不是原版的数学模型。
    44│ 
    45│ -   **现象**：文件名中带有 `Roleplay`、`RP`、`Instruct` 但不是 Math 专用版本，或者是从非官方渠道下载的、被人修改过训练数据的版本。
    46│ -   **解决**：
    47│     -   确认下载源。例如 DeepSeek-Math 的官方 Hugging Face 仓库名称应为 `deepseek-ai/deepseek-math-7b-instruct`。如果你下载的是 `TheBloke/deepseek-math-7b-instruct-GGUF`，那也是正确的。
    48│     -   如果文件名里没有 `Math` 或 `Instruct`，而是 `Base`，它本身不具备对话能力，如果强行用对话模板去套，可能会吐出训练数据里的随机内容（包括“张三”这种预训练语料里的常见名字）。
    49│ 
    50│ ### 4. 如果是在使用 Ollama
    51│ 如果你是通过 Ollama 运行的，这种情况非常典型。
    52│ 
    53│ -   **原因**：Ollama 的 `Modelfile` 中定义了 `TEMPLATE` 和 `SYSTEM`。
    54│ -   **解决**：
    55│     -   运行 `ollama show [模型名] --modelfile` 查看配置。
    56│     -   如果发现 `SYSTEM` 那一行写着“你是张三”，说明你用的这个 Modelfile 是别人做好的角色扮演版本。
    57│     -   你可以重新创建一个 Modelfile，删掉 `SYSTEM` 行，只保留 `FROM` 路径和正确的 `TEMPLATE`（数学模型通常用 `TEMPLATE """{{ .Prompt }}"""` 或标准的 chat 模板）。
    58│ 
    59│ ### 总结排查步骤
    60│ 建议你按以下顺序快速定位：
    61│ 
    62│ 1.  **看界面**：如果你用的是 LM Studio 或 Ollama WebUI，看一眼对话框上方的 **“System Prompt”** 框，清空它。
    63│ 2.  **看代码**：如果你在写 Python，检查 `messages` 列表的第一条是不是 `{“role”: “system”, “content”: “你是张三…”}`。
    64│ 3.  **看模板**：确认你的加载代码没有强制指定一个错误的 `chat_template`。
    65│ 
    66│ 如果你能告诉我你是通过什么软件（Ollama、LM Studio、transformers 代码）运行的，我可以给你更具体的检查指令。
