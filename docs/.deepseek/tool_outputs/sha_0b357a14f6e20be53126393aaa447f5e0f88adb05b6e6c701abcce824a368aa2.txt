    26│ 在已部署好的 PaddleOCR 官方容器内，将服务器上的 PDF 文件转化为 Markdown（MD）文件，最直接的方法是利用 PaddleOCR 自带的**PP-StructureV3 文档解析引擎**[citation:3][citation:6]。你可以通过编写一段 Python 代码，在容器内部调用其强大的版面分析能力，自动识别文本、表格、标题等元素，并输出结构清晰的 Markdown 文件。

    30│ ### ⚙️ 方案一：使用 Python 脚本进行解析（最推荐）
    31│ 
    32│ 这是最灵活、最通用的方法，适合希望在容器内进行自动化或批量处理的场景。
    33│ 
    34│ 1.  **进入容器**：首先，你需要进入到已经运行的 PaddleOCR 容器内部。通常使用命令：
    35│     ```bash
    36│     docker exec -it <你的容器名或ID> /bin/bash
    37│     ```
    38│ 
    39│ 2.  **准备Python脚本**：在容器内创建一个新的 Python 文件，例如 `pdf_to_md.py`，并写入以下核心代码。这段代码会读取你服务器上的 PDF 文件，并调用 PaddleOCR 的 `PPStructure` 引擎进行处理[citation:3][citation:5]。
    40│ 
    41│     ```python
    42│     import os
    43│     from paddleocr import PPStructure, save_structure_res
    44│ 
    45│     # 1. 初始化解析引擎
    46│     # 中英文文档用 'ch'，其他语言可更换，如 'en'
    47│     table_engine = PPStructure(show_log=True, image_orientation=True, layout=True, lang='ch')
    48│ 
    49│     # 2. 指定服务器上的PDF文件路径
    50│     # 例如：save_dir = '/path/on/your/server/'
    51│     pdf_path = '/path/to/your/document.pdf' # <--- 请修改为你的实际路径
    52│     output_dir = '/path/to/output/markdown'  # <--- Markdown文件输出目录
    53│ 
    54│     # 3. 执行解析
    55│     # 返回结果为列表，包含每一页的解析数据
    56│     result = table_engine(pdf_path)
    57│ 
    58│     # 4. 保存结果为Markdown文件
    59│     # save_structure_res 会将结果保存为三种格式：markdown、html、json
    60│     # 其中 markdown 格式就是我们需要的
    61│     save_structure_res(result, output_dir, os.path.basename(pdf_path).split('.')[0])
    62│ 
    63│     # 可选：如果你想合并所有页到一个Markdown文件，需要额外处理
    64│     print(f"处理完成！Markdown文件已保存至：{os.path.join(output_dir, os.path.basename(pdf_path).split('.')[0])}")
    65│     ```
    66│ 
    67│ 3.  **运行脚本**：在容器内执行脚本即可。
    68│     ```bash
    69│     python pdf_to_md.py
    70│     ```

    73│ 
    74│ 除了上面的Python脚本方案，还有两种方式可以作为备选或进阶玩法：
    75│ 
    76│ -   **方案二：利用 PaddleOCR MCP 服务器** [citation:6]
    77│     这是一种更现代化的集成方式，可以将 PaddleOCR 的解析能力封装成一个标准协议（MCP）服务器。如果你使用支持 MCP 协议的客户端（如 Claude for Desktop、某些 IDE 插件），可以通过配置直接调用容器内的 PaddleOCR 服务来转换 PDF。这种方式特别适合将 OCR 能力集成到更大的 AI 应用工作流中。
    78│ 
    79│ -   **方案三：使用基于 PaddleOCR 的第三方库（如 Doctra）** [citation:4]
    80│     Python 生态中也有一些库（例如 `doctra`）对 PaddleOCR 进行了更高层次的封装，提供了更简洁的接口。例如：
    81│     ```python
    82│     from doctra.parsers.structured_pdf_parser import StructuredPDFParser
    83│     parser = StructuredPDFParser()
    84│     parser.parse("/path/to/your/document.pdf") # 直接解析
    85│     ```
    86│     不过请注意，这种方式是在你的容器内**额外安装** `doctra` 库，而非直接使用你原有的 PaddleOCR 镜像，可能会引入新的依赖。
    87│ 
    88│ ### ✨ 关键点提醒
    89│ 
    90│ -   **路径映射**：确保你的 Python 脚本中指定的 PDF 文件路径（如 `/path/to/your/document.pdf`）是容器**内部**能访问到的路径。如果你的 PDF 文件存放在宿主机上，你需要在启动容器时通过 `-v` 参数进行**目录挂载**（例如：`-v /host/pdf/path:/container/pdf/path`）。
    91│ -   **输出结果**：`save_structure_res` 函数默认会在输出目录下创建一个以PDF文件名命名的子文件夹，并在其中生成 `页_序号.md` 等多个文件。你可以根据自己的需求，编写额外的代码将这些文件合并成一个完整的文档[citation:3]。
