    37│ ### 第一步：对症下药，查找原因
    38│ 
    39│ 你可以先通过一个简单的命令，观察终端窗口的宽度，快速判断问题根源。

    42│ # 查看终端当前宽度（列数）
    43│ echo $COLUMNS

    45│ 
    46│ ### 第二步：分步排查，各个击破
    47│ 
    48│ #### 1. 最简单快捷：调整窗口大小
    49│ 这是最直接的方法，但有时并非根本原因，调整后可能再次出现。只需用鼠标将终端窗口**手动拉宽**，检查文本显示是否正常。如果问题复现，则需检查其他设置[reference:0][reference:1]。
    50│ 
    51│ #### 2. 核心排查：检查和同步终端列数（`stty`命令）
    52│ 这是最常见的原因。前端窗口大小与后端程序（如SSH会话）记录的列数不同步，导致换行逻辑错乱[reference:2]。解决方法如下：
    53│ 
    54│ *   **第一步**：将终端窗口调整到合适的宽度。
    55│ *   **第二步**：执行 `resize` 命令，它会自动从你的终端模拟器中读取最新的窗口大小。
    56│ *   **第三步**：再次执行 `echo $COLUMNS` 确认列数已更新。
    57│ 
    58│ 如果 `resize` 命令不可用，可以手动指定一个较大的列数来临时解决，例如 `stty cols 200`[reference:3][reference:4]。
    59│ 
    60│ #### 3. 根本解决：固定终端列数设置
    61│ 为SSH连接等场景创建固定配置，避免每次手动调整。
    62│ 
    63│ *   **方法一**：在终端配置文件中（如 `~/.bashrc`, `~/.zshrc` 等），设置一个固定的列数：
    64│     ```bash
    65│     stty cols 120  # 将120替换为你期望的终端宽度
    66│     ```
    67│ *   **方法二**：使用 `resize` 命令的动态设置，将其添加到你的配置文件（如 `~/.bashrc`）中，这样每次打开新终端或登录远程服务器时，它会自动运行并同步列数。
    68│ 
    69│ #### 4. 常见但易忽视：检查字符编码
    70│ 终端和输出文本的编码不一致也可能导致显示错乱[reference:5]。
    71│ 
    72│ *   **Linux/macOS**：使用 `locale` 命令查看当前设置，确保 `LANG` 或 `LC_CTYPE` 等变量包含 `UTF-8`[reference:6]。
    73│ *   **Windows (CMD)**：使用 `chcp` 命令查看活动代码页[reference:7][reference:8]，执行 `chcp 65001` 可切换到 UTF-8 编码[reference:9]。
    74│ 
    75│ #### 5. 影响美观的细节：调整字体设置
    76│ 确保终端使用**等宽字体**（如 `Consolas`, `Courier New`, `Fira Code`, `Source Code Pro` 等），这对字符对齐至关重要[reference:10][reference:11]。对于Windows CMD，可在属性窗口的“字体”选项卡中更改；对于macOS的iTerm2，可在 `Preferences` > `Profiles` > `Text` 中修改字体[reference:12]。
    77│ 
    78│ #### 6. 深层修复：调整“歧义字符”宽度（macOS iTerm2）
    79│ 一些中文或特殊符号的宽度模棱两可，可能导致显示或删除异常[reference:13][reference:14]。在 iTerm2 中，进入 `Preferences` > `Profiles` > `Text`，在下方找到并尝试勾选 **`Treat ambiguous-width characters as double width`** 选项，通常可以解决这类问题[reference:15][reference:16]。
