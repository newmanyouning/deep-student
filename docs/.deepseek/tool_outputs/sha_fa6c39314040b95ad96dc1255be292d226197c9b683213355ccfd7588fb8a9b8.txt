我们有一个问题：在Windows终端执行脚本命令卡死如何强行中断？用户想知道在Windows终端（如cmd或PowerShell）中，当执行脚本命令卡住时，如何强制中断。

需要给出清晰的操作步骤。可能的答案：
- 对于cmd：按Ctrl+C（通常用于中断当前进程），有时按Ctrl+Break。
- 对于PowerShell：同样Ctrl+C，或者Ctrl+Break。
- 如果Ctrl+C无效，可能需要关闭终端窗口或者使用任务管理器结束进程。
- 另外，在某些情况下，可以按Ctrl+Z（发送EOF）但不太常见。
- 还可以使用任务管理器结束对应的进程（如python.exe、node.exe等）。

考虑到是“卡死”，可能完全无响应，那么可能需要强制结束终端本身。所以回答应涵盖几种方法：首选Ctrl+C，无效则尝试Ctrl+Break，再无效则关闭终端窗口或使用任务管理器结束进程。

另外，注意区分不同终端：Windows Terminal, conhost, PowerShell ISE等，但基本一致。

提供具体快捷键和操作步骤。
在 Windows 终端（CMD 或 PowerShell）中执行脚本命令时，如果程序卡死（无响应或无限循环），可以尝试以下方法强制中断：

### 1. 首选快捷键：`Ctrl + C`
- **作用**：发送中断信号（SIGINT），大多数命令行程序会响应并终止当前任务。
- **适用场景**：程序仍在处理输入，只是运行缓慢或陷入循环。
- **注意**：按一次即可，多次按也无妨；如果程序完全卡住不响应，可能无效。

### 2. 备选快捷键：`Ctrl + Break`（或 `Ctrl + Pause`）
- **作用**：发送更强硬的中断信号，部分程序对 `Ctrl+C` 无反应时可能有效。
- **按键位置**：键盘上的 `Break` 或 `Pause` 键（通常在功能键区，如 `F12` 右侧，或需 `Fn` 组合）。
- **笔记本**：可能需要 `Ctrl + Fn + Pause` 或 `Ctrl + Fn + B`（视品牌而定）。

### 3. 如果以上均无效（终端完全无响应）
- **关闭终端窗口**：直接点击窗口右上角的 `×`，或按 `Alt + F4`。会强制终止整个终端进程。
- **使用任务管理器**：
  1. 按 `Ctrl + Shift + Esc` 打开任务管理器。
  2. 在“进程”选项卡中找到对应的终端进程：
     - CMD：`Windows 命令处理程序` 或 `cmd.exe`
     - PowerShell：`PowerShell` 或 `pwsh.exe`
     - Windows Terminal：`Windows Terminal`（其内部会话可能单独列在“详细信息”中）
  3. 右键点击该进程，选择 **“结束任务”**。
- **使用 `taskkill` 命令**（如果还能打开另一个终端）：
  ```cmd
  taskkill /f /im cmd.exe
  ```
  或
  ```cmd
  taskkill /f /im powershell.exe
  ```

### 4. 避免数据丢失的提示
- 强制中断可能使未保存的脚本输出或临时文件丢失。
- 如果卡死的是 Python、Node.js 等脚本，`Ctrl+C` 通常能触发 `KeyboardInterrupt`，安全退出。
- 实在卡死只能强杀进程，之后再检查是否有残留的临时文件或锁定的资源。

### 总结步骤
1. **按 `Ctrl+C`** → 等待 3~5 秒  
2. **无效则按 `Ctrl+Break`**（可能需要 `Fn`）  
3. **仍无效** → 关闭窗口或打开任务管理器结束进程

> 对于 **PowerShell 7+**，还可以尝试 `Ctrl+C` 两次，或 `Ctrl+Shift+C`（但后者常用于复制）。如果脚本调用了 Windows API 导致死锁，只能强杀进程。
