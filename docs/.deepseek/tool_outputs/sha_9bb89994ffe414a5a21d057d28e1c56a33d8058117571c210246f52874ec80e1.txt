+## 修改清单（7 项）
+
+### 1. 新增「快速判别入口」—— 替代原文"默认速查，信号后切"
+
+**原位置**：双模行为概览末尾一句「刚进入一段新对话时，默认假定为速查模式。当出现深潜模式信号时再切换。」
+
+**改为**：在双模行为概览**之前**插入独立章节「一、快速判别入口」，用伪代码给出 4 步决策链：
+- 单行无上文 → 速查
+- 深潜信号（代码块/日志/数值/隐喻/术语/300字/追问）≥1 → 深潜
+- 冲突裁决：深潜优先
+- 无命中 → 默认速查
+
+**收益**：AI 不再需要读完整个文档才能判断模式，首屏 15 行即可完成判决，减少 token 消耗和推理延迟。
+

+
+### 2. 偏好按作用域重组 —— 替代原文按「验证次数」分类
+
+**原结构**：
+```
+核心偏好（5段以上，双模）    → 1/2/3
+重要偏好（3-4段，深潜为主）  → 4/5/6
+特色偏好（1-2段，深潜专属）  → 7/8/9/10
+速查专属                     → 11/12/13
+```
+
+**改为**：
+```
+三、全局偏好（双模生效）        → 1/2/3
+四、深潜专属                    → 4/5/6/7/8/9/10
+五、速查专属                    → 11/12/13
+```
+
+**收益**：AI 搜索偏好时按「当前模式→对应章节」直接定位，不再需要看 3 个章节的标题判断哪个适用。
+

+### 3. 量化极简边界 —— 替代原文「2–3 倍」模糊表述
+
+**原位置**：偏好 11「极简问答」中「回答控制在提问规模的 2–3 倍以内」
+
+**改为**：分层表格
+
+| 提问规模 | 回复上限 | 示例 |
+|---------|---------|------|
+| 单行（≤80字符） | ≤5行 | "什么是MCP？" → 3行 |
+| 短段落（2–5行） | ≤行数×2 | 问3行→回复≤6行 |
+| 中段落（6–20行） | ≤1.5倍 | 问10行→回复≤15行 |
+
+**收益**：AI 不再需要在「2 倍还是 3 倍」之间摇摆，每种提问尺寸有明确上限。
+
+---

+### 4. 话题表扩展双模行为列 —— 替代原文仅有「适用模式」列
+
+**原列**：`| 情境 | 触发信号 | 调整策略 | 适用模式 |`
+
+**改为**：`| 情境 | 触发信号 | 速查模式表现 | 深潜模式表现 |`
+
+每行明确给出两种模式下的具体行为差异。原先标注为「速查/深潜」的双用话题（工具/操作、概念对比）现在拆解为两个独立的行为描述。
+
+**收益**：AI 在速查模式遇到「工具/操作」话题时，不再需要猜测「要不要给多方案对比」，表内直接规定了速查=1个最佳命令。
+

+### 5. 禁止行为 → 替代动作映射 —— 彻底解决「纯否定」问题
+
+**原格式**：
+```
+- ❌ 在用户粘贴完整错误日志前猜测原因
+```
+
+**改为**：
+```
+- ❌ 在用户粘贴完整错误日志前猜测原因  
+  → ✅ 先确认日志覆盖根因行；不完整则要求补充
+```
+
+全部 8 条禁止行为逐条添加了正面替代动作。AI 不再需要从否定推导肯定行为，直接映射。
+
+**收益**：消除执行层的二次推理，减少「我知道不能做 X，那我该做什么」的决策成本。
+

+### 6. 新增「冲突裁决规则」
+
+**原位置**：不存在。当速查信号（单行提问）与深潜信号（附带代码块）同时出现时，没有明确裁决。
+
+**新增**：在「快速判别入口」和「模式切换信号」两处注入：
+- 「冲突裁决：代码块/日志/数值/隐喻 ≥ 1 则深潜优先」
+- 「速查信号与深潜信号同时出现时→深潜优先」
+
+**收益**：解决边界情况的不确定性。一句话提问但粘贴了完整 traceback 的场景现在有明确指令。
+

+
+### 7. 新增「灰色地带兜底行为」
+
+**原位置**：不存在。当 AI 无法确定速查/深潜时，没有兜底策略。
+
+**新增**：
+1. 按深潜格式回应（结构化、可验证）
+2. 长度限速查 2 倍上限
+3. 末尾追加询问是否深入
+4. 用户回复"深"→切换完整深潜
+
+**收益**：覆盖最后一类不确定场景。AI 不会在模糊地带选择沉默或猜测错误。
