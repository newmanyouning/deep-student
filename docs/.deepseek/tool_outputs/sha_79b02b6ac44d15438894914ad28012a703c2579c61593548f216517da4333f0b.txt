    34│ ### ⚙️ 快速上手：三种主流方法
    35│ 
    36│ 这里总结了三种方法，你可以根据自己的需求来选择。
    37│ 
    38│ #### 方法一：临时设置 (快速测试)
    39│ *   **特点**：只在当前终端窗口有效，关闭后即失效。
    40│ *   **适用场景**：临时测试，不想让配置一直生效。
    41│ 
    42│ 在终端直接执行以下命令（替换为你自己的代理信息）：
    43│ ```bash
    44│ # 设置 HTTP 代理 (大部分情况只需设置这个)
    45│ export http_proxy="http://your-proxy-address:port"
    46│ export https_proxy="http://your-proxy-address:port"
    47│ 
    48│ # 对于 SOCKS5 代理
    49│ export all_proxy="socks5://your-proxy-address:port"
    50│ ```
    51│ *   如果需要使用 `username:password` 认证，可以放在 `@` 符号前面：`http://username:password@address:port`[reference:0]。
    52│ *   临时关闭代理：执行 `unset http_proxy https_proxy all_proxy`[reference:1]。
    53│ 
    54│ #### 方法二：持久化配置 (一劳永逸)
    55│ *   **特点**：配置永久生效，每次打开终端都会自动应用。
    56│ *   **适用场景**：长期固定使用一个代理。
    57│ 
    58│ 1.  根据你使用的 Shell 类型，用编辑器打开对应的配置文件[reference:2]：
    59│     *   **zsh** (macOS Catalina 及以上默认)：`~/.zshrc`
    60│     *   **bash** (旧版macOS)：`~/.bash_profile`
    61│ 2.  在文件末尾，粘贴你的代理配置，例如：
    62│     ```bash
    63│     export http_proxy="http://your-proxy-address:port"
    64│     export https_proxy="http://your-proxy-address:port"
    65│     ```
    66│ 3.  保存文件后，执行 `source ~/.zshrc` (或 `source ~/.bash_profile`) 使配置立即生效[reference:3]。
    67│ 
    68│ #### 方法三：创建便捷开关 (高效切换)
    69│ *   **特点**：通过简单的 `proxy_on` 和 `proxy_off` 命令，随时开启或关闭代理。
    70│ *   **适用场景**：需要在多个不同代理间切换或频繁开关代理。
    71│ 
    72│ 1.  同样编辑你的 Shell 配置文件（`~/.zshrc` 或 `~/.bash_profile`），将以下函数添加到文件末尾：
    73│     ```bash
    74│     function proxy_on() {
    75│         export http_proxy="http://your-proxy-address:port"
    76│         export https_proxy="http://your-proxy-address:port"
    77│         echo "代理已开启"
    78│     }
    79│ 
    80│     function proxy_off() {
    81│         unset http_proxy
    82│         unset https_proxy
    83│         echo "代理已关闭"
    84│     }
    85│     ```
    86│     *   如果需要同时支持 `HTTP_PROXY` 等大写形式，可将 `proxy_on` 函数中的内容替换为[reference:4]：
    87│         ```bash
    88│         export no_proxy="localhost,127.0.0.1,localaddress,.localdomain.com"
    89│         export http_proxy="http://your-proxy-address:port"
    90│         export https_proxy=$http_proxy
    91│         export HTTP_PROXY=$http_proxy
    92│         export HTTPS_PROXY=$http_proxy
    93│         ```
    94│ 2.  保存文件后，执行 `source ~/.zshrc` (或 `source ~/.bash_profile`) 使函数生效[reference:5]。
    95│ 3.  之后，你就可以在终端里使用 `proxy_on` 和 `proxy_off` 来轻松切换代理状态了[reference:6]。
    96│ 
    97│ ### 🚀 进阶技巧：系统级代理
    98│ 
    99│ *   **方法一：修改系统网络设置**
   100│     通过 `networksetup` 命令，可以修改整个系统的代理设置，让几乎所有应用（包括终端和浏览器）都通过代理访问网络。
   101│ 
   102│     1.  **查看当前网络服务**：执行 `networksetup -listallnetworkservices`[reference:7]。记住你正在使用的服务名称，通常是 "Wi-Fi" 或 "Ethernet"。
   103│     2.  **开启 HTTP 代理**：执行 `networksetup -setwebproxy "Wi-Fi" your-proxy-address port`，然后执行 `networksetup -setwebproxystate "Wi-Fi" on`[reference:8]。
   104│     3.  **关闭代理**：执行 `networksetup -setwebproxystate "Wi-Fi" off`[reference:9]。
   105│ 
   106│ *   **方法二：使用系统“位置”快速切换**
   107│     macOS 自带的“网络位置”功能可以让你像切换环境一样切换网络配置。在 **系统设置 > 网络 > 位置** 中新建几个配置（例如“工作代理”、“无代理”），每次只需在菜单栏点击网络图标，就可以在不同“位置”间快速切换[reference:10][reference:11]。
   108│ 
   109│ ### ✅ 验证代理是否生效
   110│ 配置完成后，可以通过以下命令验证：
   111│ ```bash
   112│ # 查看当前出口 IP 地址
   113│ curl cip.cc
   114│ ```
   115│ 如果返回的 IP 地址是代理服务器的地址，说明配置成功[reference:12]。
   116│ 
   117│ ### 🔧 常见问题排查
   118│ *   **检查环境变量**：配置完成后，通过 `echo $http_proxy` 检查环境变量是否已正确设置。
   119│ *   **刷新 DNS 缓存**：如果网络请求异常，可尝试刷新 DNS 缓存：`sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder`[reference:13]。
   120│ *   **其他可能原因**：检查防火墙是否拦截代理端口[reference:14]、代理服务器认证信息是否正确[reference:15]，或某些工具（如 `git`, `npm`）可能需要单独配置代理[reference:16]。
