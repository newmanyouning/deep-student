在WPS中，查找和添加加载项主要有两种路径，它们的核心区别在于你打算安装的加载项类型：

*   **COM 加载项**：通常用于加载传统类型的插件，文件后缀多为 `.dll` 或 `.exe`[reference:0]。
*   **WPS 加载项**：是WPS基于网页技术开发的新一代加载项，通常存放在 `jsaddons` 文件夹中[reference:1][reference:2]。

### 📂 如何查找本地加载项

*   **通过WPS界面查看**
    1.  打开WPS，点击顶部菜单栏的 **「开发工具」** 选项卡[reference:3][reference:4]。（若找不到，可在顶部空白处右键，选择“自定义功能区”，在右侧勾选“开发工具”）
    2.  点击 **「COM 加载项」** 或 **「加载项」** 按钮[reference:5][reference:6]。
    3.  在弹出的窗口中，即可看到当前可用的加载项列表[reference:7]。
*   **直接访问本地文件夹**
    *   **WPS加载项**：在文件管理器地址栏输入 `%AppData%\kingsoft\wps\jsaddons` 并回车[reference:8][reference:9]。
    *   **启动项**：输入 `%AppData%\Kingsoft\wps\startup` 并回车，此处存放随WPS启动的模板插件（如Zotero）[reference:10]。
    *   **配置文件**：通过WPS快捷方式右键“打开文件所在位置”，在目录中找到 `cfgs` 文件夹，可查看 `oem.ini` 等配置文件[reference:11][reference:12]。

### ➕ 如何添加新加载项

*   **添加 COM 加载项**
    1.  在 **「COM 加载项」** 窗口，点击 **「添加」** 按钮[reference:13][reference:14]。
    2.  浏览并选择你的加载项文件（通常是 `.dll` 文件），点击“打开”即可完成添加[reference:15]。
*   **添加 WPS 加载项**
    此方法通常适用于从官方应用商店下载或开发者提供的加载项。
    1.  双击下载的 `.wps` 加载项安装包，按提示完成安装[reference:16]。
    2.  安装后重启WPS，通常会在 **「开发工具」** 选项卡或 **「工具」** 菜单下出现新图标，点击即可启用[reference:17][reference:18]。

### ⚠️ 常见问题与解决技巧

*   **WPS重启后插件未加载？**
    可尝试手动复制插件。找到Word启动项目录（如`%AppData%\Microsoft\Word\STARTUP`），将其中的 `.dotm` 模板文件复制到WPS启动项目录 `%AppData%\Kingsoft\wps\startup` 下[reference:19]。
*   **在演示文稿(PPT)中找不到加载项？**
    默认情况下可能不在功能区。可点击 **「文件」** -> **「选项」** -> **「自定义功能区」**，在左侧找到加载项，添加到右侧的 **「开发工具」** 选项卡下[reference:20]。
*   **找不到“开发工具”选项卡？**
    在顶部菜单栏空白处右键，选择 **「自定义功能区」**，在右侧列表框中勾选 **「开发工具」** 即可。
