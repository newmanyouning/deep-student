   365│ ### 🔬 第四步：调试与测试
   366│ 
   367│ 在项目目录下运行 `wpsjs debug`，它会自动启动 WPS 并加载你的插件，并开启热更新服务，修改代码保存后无需重启[reference:13][reference:14]。确保 WPS 中已启用“开发者工具”和加载项功能[reference:15]。
   368│ 
   369│ ### 📦 第五步：打包与发布
   370│ 
   371│ 调试无误后即可发布。
   372│ 
   373│ 1.  **打包插件**：运行 `wpsjs publish`，它会将项目打包成一个 `.7z` 文件，并生成一个用于发布的 `publish.html` 页面[reference:16]。
   374│ 2.  **发布插件**：有两种方式[reference:17]：
   375│     *   **Publish 模式**（推荐）：将打包文件和 `publish.html` 部署到服务器，用户访问网页即可安装[reference:18]。
   376│     *   **Jsplugins.xml 模式**：适合企业环境，通过修改 WPS 的 `oem.ini` 配置文件指向插件列表文件，实现集中分发[reference:19]。
   377│ 
   378│ ### 📚 学习资源推荐
   379│ 
   380│ *   **官方资源**：最权威的文档和示例在 **[WPS开放平台 (open.wps.cn)](https://open.wps.cn)**[reference:20]。
   381│ *   **社区与案例**：遇到问题可在 **[WPS官方论坛 (bbs.wps.cn)](https://bbs.wps.cn)** 的插件板块搜索或提问[reference:21]。CSDN、掘金等平台也有大量开发实战分享可供参考[reference:22]。
   382│ 
   383│ ### 💎 总结
   384│ 
   385│ 总的来说，你的开发流程就是：
   386│ 
   387│ 1.  **准备**：装好 WPS, Node.js, Python, Ollama 和 `cc-mini`。
   388│ 2.  **创建**：用 `wpsjs create` 建好项目。
   389│ 3.  **开发**：在项目目录里，用 `cc-mini` 帮你写代码，用 `wpsjs debug` 看效果。
   390│ 4.  **发布**：完成后用 `wpsjs publish` 打包。
   391│ 
   392│ 这样一来，整个工具链就打通了，你可以专注在插件的业务逻辑上，编码的重复劳动就交给 `cc-mini`。

   417│ 
   418│ ### 📦 第一步：打包你的加载项
   419│ 
   420│ 这一步会在你的开发电脑上完成，把所有源码变成可以分发的安装包。
   421│ 
   422│ 1.  **打包项目**：在命令行中进入你的WPS加载项项目根目录，运行打包命令。
   423│     ```bash
   424│     wpsjs build
   425│     ```
   426│     工具会询问你选择“在线模式”还是“离线模式”，请选择 **离线模式**[reference:0]。
   427│ 
   428│ 2.  **获取安装包**：打包成功后，会在你的项目文件夹下生成一个名为 `wps-addon-build` 的目录，里面有一个`.7z`压缩包，这就是你的离线安装包[reference:1]。你可以把它复制到U盘里，方便拷贝到其他电脑。
   429│ 
   430│ ### 💻 第二步：在本地电脑上运行/安装
   431│ 
   432│ 现在，你有了`.7z`安装包，可以把它放到任何一台WPS上使用了。有两种安装方式，建议你先尝试第一种。
   433│ 
   434│ #### 方法一：通过发布页面安装 (推荐，操作最简单)
   435│ 
   436│ 这种方法是官方推荐的，也最简单[reference:2]。
   437│ 
   438│ 1.  **生成发布页**：回到开发电脑，在同一个项目目录下运行发布命令。当工具询问发布地址时，可以直接按回车跳过（因为我们不需要真实的网络地址）。
   439│     ```bash
   440│     wpsjs publish
   441│     ```
   442│     成功后，项目里会多出一个`wps-addon-publish`文件夹，里面有一个`publish.html`文件[reference:3]。
   443│ 
   444│ 2.  **打包“安装器”**：把**整个`wps-addon-publish`文件夹**和那个`.7z`安装包，一起拷贝到目标电脑的同一个文件夹里。建议新建一个专门的文件夹（例如 `D:\MyWPSPlugin`）来存放。
   445│ 
   446│ 3.  **开始安装**：在目标电脑上，用**管理员身份**运行WPS（这步很重要，可以避免权限问题），然后直接双击打开`publish.html`文件。页面会显示你的插件，点击“安装”按钮即可自动完成安装[reference:4]。
   447│ 
   448│ > **⚠️ 提示**：新版本WPS的安全策略可能会阻止本地`publish.html`的安装。如果方法一失败，请使用下面的方法二。
   449│ 
   450│ #### 方法二：手动配置安装 (可靠，一步到位)
   451│ 
   452│ 这个方法是直接修改WPS的配置文件来加载插件，虽然步骤稍多，但非常可靠，适合企业和批量部署[reference:5]。
   453│ 
   454│ 1.  **放置插件文件**：
   455│     *   首先，把`.7z`安装包**解压**，你会得到一个以插件名命名的文件夹（比如 `HelloWps_1.0.0`）。
   456│     *   然后，把这个**文件夹**复制到WPS的官方插件目录。在文件资源管理器的地址栏输入以下路径并回车：
   457│         `%AppData%\kingsoft\wps\jsaddons`
   458│     *   如果`jsaddons`文件夹不存在，就手动创建一个[reference:6]。
   459│ 
   460│ 2.  **创建配置文件**：在刚才那个 `...\jsaddons` 文件夹里，新建一个名为 `jsplugins.xml` 的文本文件，用记事本打开，粘贴以下内容。**注意：要把`url`中的路径改成你自己的插件文件夹名**。
   461│     ```xml
   462│     <jsplugins>
   463│         <jsplugin name="你的插件名" 
   464│                   type="wps" 
   465│                   url="file:///%APPDATA%/kingsoft/wps/jsaddons/你的插件文件夹名" 
   466│                   version="1.0.0" 
   467│                   enable="true" />
   468│     </jsplugins>
   469│     ```
   470│     *   **路径说明**：`%APPDATA%`是一个系统变量，指向 `C:\Users\你的用户名\AppData\Roaming`。`url`就是告诉WPS去你的插件文件夹里找代码。
   471│ 
   472│ 3.  **修改WPS配置文件**：
   473│     *   找到WPS的安装目录，通常在 `C:\Users\你的用户名\AppData\Local\Kingsoft\WPS Office\版本号\office6\cfgs\`[reference:7]。
   474│     *   在这个文件夹里，找到或新建一个名为 `oem.ini` 的文件，用记事本打开。
   475│     *   在文件末尾添加或修改以下几行，告诉WPS去哪里找插件的配置列表：
   476│         ```ini
   477│         [Support]
   478│         JsApiPlugin=true
   479│         
   480│         [Server]
   481│         JSPluginsServer=%APPDATA%\kingsoft\wps\jsaddons\jsplugins.xml
   482│         ```
   483│     *   保存文件，然后**重启WPS**，你的插件应该就能在“加载项”或“开发工具”选项卡中出现了[reference:8][reference:9]。
   484│ 
   485│ ### ⚠️ 一些需要留意的地方
   486│ 
   487│ *   **WPS版本兼容性**：为了获得最好的兼容性，建议将**开发电脑和目标电脑的WPS都更新到最新版本**。有用户反馈，较旧的WPS（如11.8.2版本）可能因内置浏览器版本过低（Chrome/87），导致现代打包工具（如vite）生成的代码无法运行[reference:10]。升级WPS是解决这类问题最直接有效的方法。
   488│ *   **注意关键差异**：在开发和调试阶段，WPS需要访问本地服务器（`http://localhost`），但离线插件无法访问，所以不要在你的代码中写死指向`localhost`的请求[reference:11]。
