### .gitignore 文件的编写与最佳实践
**标签**: Git, 版本控制, 项目配置
**要点**: 
1. `.gitignore` 文件放在项目根目录，用于指定 Git 不应追踪的文件和目录，避免将临时文件、日志、依赖包等纳入版本控制。
2. 支持通配符模式匹配：`*.log` 匹配所有日志文件，`node_modules/` 匹配整个目录，`.vscode/` 忽略编辑器配置。
3. `.DS_Store`（macOS）和 `Thumbs.db`（Windows）是常见的系统生成文件，应列入忽略列表以防止跨平台协作时的噪音。
4. 运行中动态生成的文件（如 `.auto-commit.log`、`.pid` 文件）也应在 `.gitignore` 中声明，保持仓库干净。

**关键命令/代码**:
```
# 忽略自动提交相关的文件
.auto-commit.log
.git-auto-commit.pid
*.tmp
*.log

# 忽略编辑器文件
.vscode/
.idea/
*.swp

# 忽略系统文件
.DS_Store
Thumbs.db

# 忽略依赖文件
node_modules/
__pycache__/
```

### Bash 循环监控与自动提交脚本
**标签**: Bash, Shell脚本, Git自动化
**要点**: 
1. 使用 `while true; do ... done` 构建无限循环，配合 `sleep 30` 实现每30秒轮询一次文件变更。
2. `git status --porcelain` 以简洁格式输出仓库状态，适合脚本解析；管道到 `grep -q .` 检测是否有任何输出（即是否存在变更）。
