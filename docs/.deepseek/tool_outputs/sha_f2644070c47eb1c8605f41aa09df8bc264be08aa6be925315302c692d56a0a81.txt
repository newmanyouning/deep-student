首先考虑的是如何快速确认键盘已被系统识别。根据搜索结果，[citation:1][citation:2][citation:3]都提到了使用`xinput list`命令，这个命令能够显示所有已识别的输入设备，包括键盘、鼠标和触摸板等，操作简单直观。[citation:3]还补充了`lsusb`命令，它可以列出所有USB设备，对于有线键盘来说特别有用。这些方法都很基础，适合快速检查。

接下来需要提供更详细的设备信息查询方法。[citation:2][citation:4]中提到的`xinput --list-props <device_id>`命令可以显示特定设备的属性信息。[citation:4]详细介绍了`lsusb -v`命令，它能提供设备的厂商ID、产品ID等详细信息。这些信息在键盘无法正常工作时对排查问题很有帮助。

对于键盘功能的测试，[citation:1]和[citation:3]提到了`xev`工具，它能够创建一个窗口来实时显示键盘按键事件，是验证键盘工作状态的理想选择。[citation:8]还提到了`showkey`命令，不过它主要在命令行环境下使用。

考虑到用户可能会遇到键盘无法使用的情况，[citation:3]和[citation:5]提供了一些实用的故障排查步骤，比如检查USB接口、查看系统日志等。特别是[citation:3]中提到的恢复模式操作，对于解决登录界面键盘失效的情况很有参考价值。

在Ubuntu系统中检查有线键盘的连接和状态，可以通过一系列命令从“是否被识别”和“能否正常工作”两个层面来排查。

下面的表格汇总了几种核心的检查方法：

| 检查方向 | 命令/方法 | 作用与说明 |
| :--- | :--- | :--- |
| **快速状态检查** | `xinput list`[citation:1][citation:2] | 列出所有**已识别**的输入设备（键盘、鼠标等），可快速确认键盘是否被系统识别。 |
| | `lsusb`[citation:2][citation:3][citation:4] | 列出所有USB设备，有线键盘通常会显示在其中。 |
| **查看设备详情** | `xinput --list-props <设备ID>`[citation:1] | 查看指定输入设备的详细属性。 |
| | `lsusb -v`[citation:4] 或 `usb-devices`[citation:4] | 查看USB设备的**详细信息**，如厂商ID、产品ID，有助于排查驱动问题。 |
| **测试键盘功能** | `xev`[citation:3][citation:8] | 打开一个测试窗口，**实时显示按键事件**，是测试键盘是否正常工作的最佳工具。 |
| | `showkey`[citation:3][citation:8] | 在终端内测试按键，主要显示扫描码，适合纯命令行环境。 |

### 🔍 分步操作指南

你可以按以下顺序进行检查：

1.  **打开终端**
    同时按下 `Ctrl` + `Alt` + `T` 打开终端窗口。如果当前键盘无响应，可以尝试使用屏幕键盘（在登录界面或系统设置中启用“辅助功能”里的屏幕键盘）来操作。

2.  **执行检查命令**
    *   **基础检查**：在终端中输入 `xinput list` 并回车。查看输出列表中是否包含“Keyboard”或你的键盘品牌名称（如“Logitech”）的条目[citation:1][citation:2]。
    *   **USB连接确认**：输入 `lsusb` 并回车。查看输出中是否有键盘相关的描述[citation:2][citation:4]。
    *   **功能测试**：输入 `xev` 并回车，会弹出一个名为“Event Tester”的小窗口。将鼠标焦点放入该窗口，然后按动有线键盘的按键。如果在终端里能看到**KeyPress**和**KeyRelease**事件反馈，就证明键盘硬件和基础驱动是正常的[citation:3]。

### ⚠️ 如果键盘未被识别或无法使用

如果以上步骤发现键盘未被列出或没有响应，可以尝试以下排查方法：

*   **检查物理连接**：拔下键盘，换一个**主板上不同位置的USB接口**重新插入，并检查USB线缆是否完好。
*   **查看内核日志**：在终端输入 `dmesg | tail -20`，查看系统最近的日志。插入键盘前后，观察输出中是否有关于USB设备或“input”的新信息，这能显示系统检测到硬件的过程[citation:2][citation:3]。
*   **检查并重置输入系统**：有时图形界面的输入服务可能出现问题。可以尝试重启显示管理器：
    ```bash
    sudo systemctl restart gdm3  # 适用于使用GNOME桌面环境的Ubuntu
    ```
    或者通用的：
    ```bash
    sudo systemctl restart display-manager
    ```
*   **使用恢复模式**：如果在**登录界面**键盘就完全失灵，可以重启电脑，在启动时按 `Esc` 或 `Shift` 键进入GRUB菜单，选择 **“Advanced options for Ubuntu”**，然后进入 **“(recovery mode)”** 恢复模式。在恢复菜单中，依次选择 `network`（启用网络）、`root`（进入root shell），然后执行以下命令重装输入驱动[citation:3]：
    ```bash
    mount -o rw,remount /
    apt install --reinstall xserver-xorg-input-all
    reboot
    ```

### 💎 核心建议
多数情况下，`xinput list` 和 `xev` 组合使用就能快速定位问题。如果键盘在 `xinput list` 中有显示但在 `xev` 中无反应，可能是硬件或驱动故障；如果 `xinput list` 中根本没有，则问题更可能出在USB连接、电源或系统底层识别上[citation:3][citation:4]。
