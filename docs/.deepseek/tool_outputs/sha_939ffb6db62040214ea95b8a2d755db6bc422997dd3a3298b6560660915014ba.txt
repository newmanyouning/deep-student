    60│ 要在 Docker 容器中查看已部署模型的版本，你需要**进入容器内部**，然后根据模型在容器中的安装方式执行相应的查询命令。以下是通用的一步步操作指南：
    61│ 
    62│ ---
    63│ 
    64│ ### **步骤 1：确定你的 Docker 容器 ID 或名称**
    65│ 首先，列出当前正在运行的容器，找到你部署模型的容器。
    66│ ```bash
    67│ docker ps
    68│ ```
    69│ 输出示例：
    70│ ```
    71│ CONTAINER ID   IMAGE                COMMAND                  CREATED       STATUS       PORTS                    NAMES
    72│ abc123def456   my-model:latest      "python app.py"          2 hours ago   Up 2 hours   0.0.0.0:5000->5000/tcp   my_model_container
    73│ ```
    74│ 记下 `CONTAINER ID`（例如 `abc123def456`）或 `NAMES`（例如 `my_model_container`），后续步骤中用 `<container>` 代替。
    75│ 
    76│ ---
    77│ 
    78│ ### **步骤 2：进入容器的交互式 Shell**
    79│ 使用 `docker exec` 命令进入容器。大多数 Linux 容器（包括 Ubuntu）都有 `/bin/bash` 或 `/bin/sh`。
    80│ ```bash
    81│ docker exec -it <container> /bin/bash
    82│ ```
    83│ 如果提示 `bash: not found`，可以尝试 `sh`：
    84│ ```bash
    85│ docker exec -it <container> /bin/sh
    86│ ```
    87│ 成功进入后，你会看到类似 `root@abc123def456:/#` 的提示符，表示现在已经在容器内部了。
    88│ 
    89│ ---
    90│ 
    91│ ### **步骤 3：在容器内查看模型版本**
    92│ 现在你处于容器内部的 Ubuntu 环境，根据模型部署的方式选择以下方法之一：
    93│ 
    94│ #### **情况 A：模型是通过 Python 包（如 TensorFlow、PyTorch）安装的**
    95│ - 查看特定 Python 包的版本：
    96│   ```bash
    97│   pip show <包名>
    98│   ```
    99│   例如：
   100│   ```bash
   101│   pip show torch
   102│   ```
   103│ - 或者用 Python 一行命令查看：
   104│   ```bash
   105│   python -c "import <包名>; print(<包名>.__version__)"
   106│   ```
   107│   例如：
   108│   ```bash
   109│   python -c "import torch; print(torch.__version__)"
   110│   ```
   111│ 
   112│ #### **情况 B：模型是通过系统 apt 安装的（较少见）**
   113│ - 查看已安装的 deb 包：
   114│   ```bash
   115│   apt list --installed | grep <关键词>
   116│   ```
   117│   或
   118│   ```bash
   119│   dpkg -l | grep <关键词>
   120│   ```
   121│ 
   122│ #### **情况 C：模型是编译好的可执行文件（如 llama.cpp）**
   123│ - 尝试运行带版本参数的命令：
   124│   ```bash
   125│   <程序名> --version
   126│   ```
   127│   例如：
   128│   ```bash
   129│   ./main --version   # 如果程序在当前目录
   130│   ```
   131│ 
   132│ #### **情况 D：模型版本信息写在环境变量或文件中**
   133│ - 查看环境变量：
   134│   ```bash
   135│   env | grep VERSION
   136│   ```
   137│ - 查看特定文件（如 `version.txt`、`Dockerfile` 中定义的变量）：
   138│   ```bash
   139│   cat /path/to/version.txt
   140│   ```
   141│ 
   142│ #### **情况 E：模型文件本身包含版本信息（如模型文件名）**
   143│ - 列出模型存放目录：
   144│   ```bash
   145│   ls -l /path/to/model/
   146│   ```
   147│   文件名可能包含版本号，例如 `bert-base-uncased` 或 `llama-2-7b.Q4_K_M.gguf`。
   148│ 
   149│ ---
   150│ 
   151│ ### **步骤 4：退出容器**
   152│ 查看完毕后，输入 `exit` 即可退出容器，回到宿主机。
   153│ 
   154│ ---
   155│ 
   156│ ### **如果容器内没有找到任何版本信息怎么办？**
   157│ 有时容器构建时没有显式记录版本号，此时可以退而求其次：
   158│ 
   159│ - **查看镜像的标签**（在宿主机执行）：
   160│   ```bash
   161│   docker images | grep <镜像名>
   162│   ```
   163│   镜像标签（如 `my-model:v1.2`）可能暗示了版本。
   164│ 
   165│ - **检查镜像的构建历史**，看 Dockerfile 中是否指定了版本：
   166│   ```bash
   167│   docker history <镜像名>:<标签>
   168│   ```
   169│ 
   170│ - **查看容器启动时的环境变量**（宿主机执行）：
   171│   ```bash
   172│   docker inspect <container> | jq '.[0].Config.Env'   # 需要安装 jq，或直接查看完整 JSON
   173│   ```
   174│ 
   175│ ---
   176│ 
   177│ ### **示例：在一个部署了 PyTorch 模型的容器中查看版本**
   178│ 假设你的容器名为 `my_torch_app`，步骤如下：
   179│ 
   180│ 1. `docker exec -it my_torch_app /bin/bash`
   181│ 2. 进入后执行 `pip show torch`，看到输出：
   182│    ```
   183│    Name: torch
   184│    Version: 1.13.1
   185│    ...
   186│    ```
   187│ 3. 退出：`exit`
   188│ 
   189│ 这样你就得到了 PyTorch 的版本号。如果需要查看模型本身的版本（例如 Hugging Face 模型），可以检查模型缓存目录：`ls ~/.cache/huggingface/hub/`，目录名通常包含版本哈希。
