    40│ ### 🛠️ WPS加载项开发与本地加载指南
    41│ 
    42│ WPS 加载项是官方主推的开发方式，技术门槛也相对友好，主要面向具备Web前端开发（HTML、CSS、JavaScript）经验的开发者[reference:8]。你可以通过以下步骤在个人版上进行本地开发和调试。
    43│ 
    44│ #### 1. 开发准备
    45│ *   **安装Node.js**：建议安装LTS版本（如18.x），用于运行WPS插件构建工具[reference:9]。
    46│ *   **安装WPS**：确保你的WPS版本号不低于`11.2.0`[reference:10]。
    47│ 
    48│ #### 2. 核心流程：开发与调试
    49│ WPS官方提供了一个名为`wpsjs`的命令行工具，可以快速生成项目模板并进行本地调试，这是个人版下最推荐的开发方式[reference:11][reference:12]。
    50│ 
    51│ 1.  **安装开发工具**：在命令行中全局安装`wpsjs`工具包：
    52│     ```bash
    53│     npm install -g wpsjs
    54│     ```
    55│     [reference:13]
    56│ 
    57│ 2.  **创建项目**：使用`wpsjs`创建一个新项目，工具会引导你选择插件适用的组件（文字、表格、演示）和代码风格（原生JS或Vue框架）[reference:14]。
    58│     ```bash
    59│     wpsjs create MyFirstAddon
    60│     ```
    61│ 3.  **开始调试**：进入项目目录，运行调试命令，此命令会自动打开WPS并加载你的插件，支持代码热更新[reference:15]。
    62│     ```bash
    63│     cd MyFirstAddon
    64│     wpsjs debug
    65│     ```
    66│ 4.  **编写代码**：用你喜欢的编辑器（如VS Code）打开项目文件夹，开始编写插件的业务逻辑。
    67│ 
    68│ #### 3. 正式使用：加载您的插件
    69│ 调试完成后，你可以参考CSDN上的教程，通过修改WPS安装目录下的`oem.ini`和`jsplugins.xml`两个配置文件，来正式加载插件[reference:16]。
    70│ 
    71│ ### ⚠️ 需要留意的其他几点
    72│ *   **开发中心**：插件开发的核心资源都在 **[WPS开放平台 (open.wps.cn)](https://open.wps.cn)**，可以在此查阅官方文档、下载SDK[reference:17]。
    73│ *   **账号要求**：早期WPS开发需企业账号，但据称新版已开放个人开发者[reference:18]。不过，要在开放平台发布、分发插件供他人使用，可能需要完成服务商认证[reference:19]。
    74│ *   **发布与分发**：个人用户开发插件主要用于**自用**或**内部团队使用**。如果你想将插件公开发布，供其他用户下载，则需要了解WPS开放平台的应用上架流程和规则。
    75│ *   **官方技术交流**：如果在开发中遇到问题，可以到 **WPS官方论坛 (bbs.wps.cn)** 的插件开发板块搜索或提问[reference:20]，那里有不少开发者和官方人员，是很好的求助渠道。
