   192│ ### 📚 官方核心资料
   193│ 这是你查阅文档的第一步，是进行WPS VBA开发的基石。根据WPS版本不同，核心资料分为两类：
   194│ 
   195│ *   **WPS API V9 开发帮助文档**：这是一份社区流传的经典资源，广泛适用于 **WPS Office 2019及后续专业版**。它包含了WPS文字、表格、演示三大组件的二次开发帮助，详细介绍了VBA编程的相关知识和操作方法[reference:0][reference:1]。
   196│ *   **WPS Office三大组件开发文档合集 (CHM格式)**：这是WPS二次开发的**官方技术基石**，是一套面向开发者的API参考手册，内容详实、结构清晰[reference:2]。它们通常作为离线帮助文件，专供开发人员查阅API细节，在企业级应用集成中不可或缺[reference:3]。包含三份核心文件：
   197│     *   `ET.CHM`：对应WPS表格，详细解析了Workbook、Worksheet、Range等对象模型[reference:4]。
   198│     *   `WPS.CHM`：对应WPS文字，涵盖Application、Document、Paragraph等对象[reference:5]。
   199│     *   `WPP.CHM`：对应WPS演示，定义了Presentation、Slide、Shape等对象层次结构[reference:6]。
   200│ 
   201│ ### 📖 权威参考手册
   202│ 除了官方资料，网络上还有一些非常权威的参考手册，它们是开发者的“案头必备”。
   203│ 
   204│ *   **Word与WPS文档自动化编程接口参考手册**：这是一份由官方或资深社区整理的API参考手册，包含了`Word+VB+参考.CHM`和`wps.chm`[reference:7]。它详细阐述了WPS的对象模型，并强调了与MS Office VBA语法的高度兼容性，极大降低了从Office生态转向WPS生态的学习成本[reference:8][reference:9]。
   205│ *   **WPS Excel集成VBA宏开发环境与自动化表格处理指南**：这份指南详细介绍了WPS表格中VBA的集成使用，包括录制宏、编写代码、调用对象模型、引用外部库等[reference:10]。

   207│ ### 📘 学习与实践指南
   208│ 如果你是初学者或希望系统学习，以下从入门到进阶的指南将非常有帮助。
   209│ 
   210│ *   **《VBA在WPS中的应用指南》**：这份免费资源非常适合入门，内容覆盖了从VBA的基础语法、环境搭建到高级技巧的全过程，并提供了在WPS文字、表格和演示等不同组件中的具体应用实例[reference:11][reference:12]。
   211│ *   **WPS宏与Office宏差异分析文章**：许多技术文章专门分析了WPS与Office在VBA对象模型、事件支持等方面的关键差异[reference:13][reference:14][reference:15]。这对于将Excel VBA代码迁移到WPS环境尤为重要。

   307│ ### 🛠️ 第一步：准备“教材”——为AI搜集权威文档
   308│ 
   309│ 你需要主动为AI准备并指明参考资料，建议优先使用这三类：
   310│ 
   311│ 1.  **基础语法**：在`GitCode`或`CSDN`等平台搜索《VBA在WPS中的应用指南》[reference:2]。这份指南覆盖了基础语法、环境搭建等，适合作为入门教材。
   312│ 2.  **核心对象模型**：首选是 **《WPS API V9 开发帮助文档》**[reference:3][reference:4]和 **《Word与WPS文档自动化编程接口参考手册》**[reference:5]。它们能提供最核心、最权威的参考。如果在使用中遇到疑难，可以作为补充。
   313│ 3.  **差异与避坑**：当遇到代码在WPS中运行出错时，可以搜索 **《WPS宏与Office宏在VBA语法和对象模型上有哪些关键差异？》** 这类文章[reference:6][reference:7]。它详细列出了WPS与MS Office VBA的差异点，能有效帮你“避坑”。
   314│ 
   315│ 如果找不到合适的纯文本（Markdown）格式文档，也可以将找到的CHM或PDF文件内容手动复制，然后粘贴到与AI的对话框里。

   317│ ### ✍️ 第二步：下达指令——给AI明确的“开发规范”
   318│ 
   319│ 有了教材，还需要一个清晰的“开发规范”来约束AI的行为。这通常以提示词（Prompt）的形式给出，确保生成的代码是合规可用的。
   320│ 
   321│ 一个有效的“开发规范”提示词应该包含以下关键部分：
   322│ 
   323│ *   **角色与目标**：明确AI的身份和核心任务。
   324│ *   **知识范围**：指定AI必须遵循的文档，同时考虑到WPS VBA与标准Office VBA的差异。
   325│ *   **约束与指令**：对代码格式、注释、测试等提出具体要求。
