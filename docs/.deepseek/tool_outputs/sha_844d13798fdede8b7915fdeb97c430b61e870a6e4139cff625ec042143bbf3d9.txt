    18│ 因为问题比较基础，适合给出清晰的答案。
    19│ 如果你已经从 GitHub 拉取（clone 或 pull）了一个项目到本地，当远程仓库有更新时，可以使用以下 Git 命令来拉取最新版本：
    20│ 
    21│ ### 1. 最简单的方法：`git pull`
    22│ ```bash
    23│ git pull
    24│ ```
    25│ 这会将当前分支对应的远程分支的更新拉取下来，并自动合并到你的本地分支。
    26│ 
    27│ 如果当前分支没有设置上游跟踪，可以显式指定远程和分支名：
    28│ ```bash
    29│ git pull origin main
    30│ # 或 master（取决于默认分支名）
    31│ ```
    32│ 
    33│ ### 2. 更安全的方法：先 `fetch` 再 `merge`
    34│ ```bash
    35│ git fetch origin        # 获取远程更新，但不自动合并
    36│ git merge origin/main   # 手动合并到当前分支
    37│ ```
    38│ 这样做的好处是可以在合并前查看远程的变化，避免自动合并带来的冲突。
    39│ 
    40│ ### 3. 如果本地有未提交的修改
    41│ 当本地存在未提交的更改时，`git pull` 可能会被拒绝。这时可以：
    42│ - 先提交本地修改：`git add . && git commit -m "..."`，然后再 `git pull`
    43│ - 或者临时储藏修改：`git stash`，拉取后再恢复 `git stash pop`
    44│ 
    45│ ### 4. 强制拉取覆盖本地修改（⚠️ 谨慎使用）
    46│ 如果你想完全用远程版本覆盖本地修改（会丢失本地未提交的更改）：
    47│ ```bash
    48│ git fetch origin
    49│ git reset --hard origin/main
    50│ ```
    51│ 
    52│ ---
    53│ 
    54│ **总结**：常规更新只需在项目目录下执行 `git pull`，建议养成先 `git status` 查看本地状态的好习惯。
