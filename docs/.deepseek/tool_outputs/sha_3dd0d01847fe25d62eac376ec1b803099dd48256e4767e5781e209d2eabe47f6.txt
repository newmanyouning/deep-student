   228│ ## 1️⃣ 先确认脚本是否真的执行成功了
   229│ 
   230│ 打开终端（Terminal），**重新运行一次脚本**，仔细观察输出信息：
   231│ 
   232│ - 如果看到 `已写入 i18nLocale=zh-CN 到 ...` 字样，说明脚本修改配置文件成功。
   233│ - 如果没有任何输出或报错，说明脚本执行中断了。可能原因：
   234│   - 终端权限不足（可尝试直接复制整段脚本粘贴运行）。
   235│   - Docker Desktop 未完全退出（脚本第一行已尝试强制退出，但可能被其他进程占用）。
   236│   - Python 环境问题（macOS 自带 Python3，一般没问题）。
   237│ 
   238│ **解决方法**：  
   239│ 如果脚本没成功，可以直接跳到下面的**手动修改配置文件**（方法二）。

   242│ 
   243│ ## 2️⃣ 手动修改配置文件（最稳定）
   244│ 
   245│ 脚本本质就是修改一个 JSON 文件，手动操作更直观。
   246│ 
   247│ ### 步骤：
   248│ 1. **彻底退出 Docker Desktop**  
   249│    点击菜单栏 Docker 图标 → 选择 **Quit Docker Desktop**。
   250│ 
   251│ 2. **打开配置文件所在文件夹**  
   252│    在终端中运行：
   253│    ```bash
   254│    open ~/Library/Group\ Containers/group.com.docker/DockerDesktop
   255│    ```
   256│    会弹出一个 Finder 窗口，里面有一个 `settings.json` 文件。
   257│ 
   258│ 3. **编辑 `settings.json`**  
   259│    - 用“文本编辑”或 VS Code 打开 `settings.json`。
   260│    - 确保文件内容是一个完整的 JSON 对象（花括号 `{}` 包裹）。
   261│    - 在花括号内**添加或修改**下面这一行。  
   262│      **注意**：如果已有其他配置项，请在前一行末尾加上英文逗号 `,`。
   263│ 
   264│      ```json
   265│      {
   266│        "i18nLocale": "zh-CN"
   267│      }
   268│      ```
   269│      例如，原文件可能长这样：
   270│      ```json
   271│      {
   272│        "analyticsEnabled": false,
   273│        "somethingElse": true
   274│      }
   275│      ```
   276│      修改后应为：
   277│      ```json
   278│      {
   279│        "analyticsEnabled": false,
   280│        "somethingElse": true,
   281│        "i18nLocale": "zh-CN"
   282│      }
   283│      ```
   284│ 
   285│ 4. **保存文件并重启 Docker Desktop**  
   286│    重新打开 Docker Desktop，界面应该就变成中文了。
   287│

   290│ ## 3️⃣ 如果手动修改后仍无效
   291│ 
   292│ 可能原因及解决方法：
   293│ 
   294│ ### 🔸 配置文件没被读取（缓存问题）
   295│ 执行以下命令清除 Docker Desktop 的偏好设置缓存，然后重启：
   296│ ```bash
   297│ rm -f ~/Library/Preferences/com.docker.docker.plist
   298│ rm -f ~/Library/Preferences/com.docker.helper.plist
   299│ ```
   300│ 
   301│ ### 🔸 文件权限问题
   302│ 确保 `settings.json` 的权限正确（一般为 `-rw-------`）。可以终端运行：
   303│ ```bash
   304│ ls -l ~/Library/Group\ Containers/group.com.docker/DockerDesktop/settings.json
   305│ ```
   306│ 如果权限不对，用 `chmod 600` 修改。
   307│ 
   308│ ### 🔸 Docker Desktop 版本太老
   309│ Docker Desktop 在 **4.10 版本左右**开始正式支持多语言切换。如果你用的是更早的版本，可能配置文件中的 `i18nLocale` 字段无效。  
   310│ **建议升级到最新版**：在 Docker Desktop 菜单中点击 **Check for Updates**，或直接去官网下载。

   375│ ## ❓ unknown
   376│ 
   377│ 我们注意到用户粘贴了终端输出，其中显示了脚本执行时出现 PermissionError: [Errno 1] Operation not permitted，表明脚本在尝试创建或写入目录时权限不足。用户的问题是“一键脚本永久汉化发现不行，我应该怎么做？”现在我们看到错误是权限问题。
   378│ 
   379│ 需要指导用户解决权限问题。可能原因是 macOS 的隐私权限限制（比如“完全磁盘访问权限”或“文件和文件夹”权限），或者目录本身权限设置不正确。我们可以提供几个步骤：
   380│ 
   381│ 1. 确认 Docker Desktop 是否已完全退出（退出后配置文件可能可写）。
   382│ 2. 检查 Group Containers 目录的权限，可能需要给终端（或 Python）授予“完全磁盘访问权限”。
   383│ 3. 手动修改配置文件的方法（绕过脚本）。
   384│ 4. 如果手动修改也遇到权限问题，可能需要通过 Finder 中的“获取信息”修改权限，或者用 sudo（但需谨慎）。
   385│ 
   386│ 用户已经尝试了脚本，但权限错误导致失败。我们应提供解决方案，让用户能够成功修改配置文件。
   387│ 
   388│ 我们应解释错误原因：macOS 对 `~/Library/Group Containers/` 等目录有严格的访问控制，脚本运行时可能因为沙盒限制无法写入。解决方法通常是通过系统设置授予终端（或运行脚本的 app）完全磁盘访问权限，或者使用 Finder 手动编辑文件（因为 Finder 有访问权限）。手动编辑是最简单的。
   389│ 
   390│ 我们还要注意，用户可能是直接粘贴脚本到终端，终端应用需要权限。可以建议用户在“系统设置 > 隐私与安全性 > 完全磁盘访问权限”中添加“终端”。

   398│ 最后，如果手动修改仍不行，建议删除 Docker 配置文件夹并重装。
