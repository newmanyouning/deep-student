   602│ ### 1.1 功能范围
   603│ 项目是一个基于 PyQt5 的 Excel/WPS 分析工具，支持：
   604│ - 打开/解析 Excel 文件（.xls, .xlsx, .xlsm, .et）
   605│ - 单元格内容、格式、公式依赖分析
   606│ - 页眉页脚图片管理（插入/移除/批量）
   607│ - 批量数据提取与导入
   608│ - 模板保存与加载
   609│ - WPS Office 环境检测与 COM 连接
   610│ - VBA 宏动态注入与执行（数据提取、格式读取、依赖分析等）
   611│ 
   612│ ### 1.2 技术栈
   613│ - GUI: PyQt5
   614│ - Office 交互: win32com (通过 `Ket.Application` 或 `Excel.Application`)
   615│ - VBA 注入: 运行时将 `.bas` 模块注入 Excel/WPS，执行宏并解析 JSON 输出
   616│ - 打包: PyInstaller (单 EXE)
   617│ - 其他: pandas, openpyxl (备用), sqlite3 (模板存储)
   618│ 
   619│ ### 1.3 已知问题（从修复记录中提取）
   620│ - **导入路径混乱**：大量使用相对导入 (`from ..logic...`) 和绝对导入混用，导致 `ModuleNotFoundError` 频发。
   621│ - **重复代码与重复定义**：如 `show_progress` 方法定义两次。
   622│ - **信号槽连接错误**：按钮无响应、菜单方法缺失或连接错误。
   623│ - **打包后模块缺失**：`pandas._libs`、`sqlite3`、`PyQt5.sip` 等需要反复添加到 `hiddenimports`。
   624│ - **VBA 解析失败**：文件路径问题（`".\\template.xls"` 无法找到）、WPS 宏安全设置问题。
   625│ - **MVC 迁移不彻底**：`main_window.py` 仍有大量业务逻辑，控制器职责不清晰，部分控制器为空壳。
   626│ - **COM 资源泄漏**：未正确释放 WPS 应用对象，可能导致进程残留。
   627│ - **错误处理薄弱**：很多异常被捕获后只打印日志，没有用户友好提示。

   636│ ## 二、重新编写的核心理念
   637│ 
   638│ **不要尝试“修补”当前项目**，而是**基于现有功能需求，从头设计一个简洁、稳定、可维护的新项目**。可以复用部分稳定模块（如 VBA 注入器、WPS 检测器），但必须经过清理和标准化。
   639│ 
   640│ ### 2.1 设计原则
   641│ 1. **单一职责**：每个模块只做一件事。
   642│ 2. **依赖倒置**：面向接口编程（如定义 `IExcelApplication` 抽象，分别实现 `WPSApplication` 和 `ExcelApplication`）。
   643│ 3. **显式优于隐式**：使用绝对导入，项目根目录明确，避免动态修改 `sys.path`。
   644│ 4. **测试驱动**：关键逻辑（COM 调用、解析结果）必须有单元测试和集成测试。
   645│ 5. **配置分离**：数据库路径、日志级别、VBA 模块目录等使用配置文件（JSON / YAML）。
   646│ 
   647│ ### 2.2 技术选型保持不变
   648│ - PyQt5 (或升级到 PyQt6，但兼容性需评估)
   649│ - win32com
   650│ - PyInstaller (但使用 `--onedir` 模式便于调试，最终可转为单文件)
   651│ - SQLite (模板存储)
   652│ - 日志标准库 `logging`

   711│ ### 3.2 关键模块说明
   712│ 
   713│ #### 3.2.1 服务层（services）
   714│ - **`IExcelService`** 抽象类：
   715│   ```python
   716│   class IExcelService(ABC):
   717│       @abstractmethod
   718│       def open_workbook(self, path: str) -> Workbook:
   719│           pass
   720│       @abstractmethod
   721│       def get_cell_value(self, sheet: str, cell: str) -> Any:
   722│           pass
   723│       # ... 其他方法
   724│   ```
   725│ - **`WpsService`** 实现：封装 `WPSConnector`、`VBAInjector`，处理 COM 生命周期。
   726│ - **`VBAInjector`** 独立类：负责将 `.bas` 文件内容注入到当前活动工作簿，执行宏并返回 JSON 结果。
   727│ 
   728│ #### 3.2.2 模型层（models）
   729│ - 使用普通数据类（`@dataclass`），不包含业务逻辑。
   730│ - `Workbook` 包含 `sheets`、`cells` 字典等。
   731│ 
   732│ #### 3.2.3 视图层（views）
   733│ - **`MainWindow`** 只负责：
   734│   - 创建 UI 控件（通过 `setupUi` 或手动布局）
   735│   - 定义信号（例如 `file_open_requested`）
   736│   - 连接控制器的信号到 UI 更新槽函数
   737│   - **不包含任何业务逻辑**（不直接调用 COM、不操作数据库）
   738│ 
   739│ #### 3.2.4 控制器层（controllers）
   740│ - 接收视图的信号，调用服务层方法，通过信号将结果发送回视图。
   741│ - 例如：
   742│   ```python
   743│   class MainController:
   744│       def __init__(self, view: MainWindow, excel_service: IExcelService):
   745│           self.view = view
   746│           self.service = excel_service
   747│           self.view.file_open_requested.connect(self.on_open_file)
   748│       
   749│       def on_open_file(self, path: str):
   750│           try:
   751│               workbook = self.service.open_workbook(path)
   752│               self.view.display_workbook(workbook)
   753│           except Exception as e:
   754│               self.view.show_error(str(e))
   755│   ```
   756│

   759│ - 在项目根目录运行 `python -m src.main`，确保 `PYTHONPATH` 包含项目根目录。
