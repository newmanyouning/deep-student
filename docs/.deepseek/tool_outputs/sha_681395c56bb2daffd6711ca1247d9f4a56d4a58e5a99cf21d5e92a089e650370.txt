    32│ 
    33│ ### 🛡️ 第一步：迁移前的“安全绳”
    34│ 
    35│ 在开始任何操作前，务必花几分钟做好这步，这是最关键的安全保障：
    36│ 
    37│ *   **创建系统还原点**：一个万能的“后悔药”，如果迁移出现问题，可以一键将系统恢复到迁移前的状态[reference:0]。
    38│     1.  按 `Win + R`，输入 `sysdm.cpl` 并回车。
    39│     2.  切换到“系统保护”选项卡，选中系统盘（通常是C盘），点击“创建”。
    40│     3.  输入一个描述（如“应用迁移前”），点击“创建”即可。
    41│ 
    42│ *   **完全备份目标应用**：直接把程序的安装文件夹复制粘贴到另一个安全的地方[reference:1]。万一迁移失败，还可以复制回来。
    43│ 
    44│ ### 🔧 第二步：选择合适的迁移方法
    45│ 
    46│ #### 方案一：使用系统自带功能（仅限微软商店应用）
    47│ 
    48│ 这是最简单的方法，但只适用于从Microsoft Store下载的应用（通常是UWP应用或部分打包的桌面应用）[reference:2][reference:3]。
    49│ 
    50│ 1.  按 `Win + I` 打开“设置”，进入“应用” > “已安装的应用”[reference:4]。
    51│ 2.  找到想迁移的应用，点击右侧的“三个点”（...），选择“移动”（如果此按钮是灰色，说明该应用不支持此方法）[reference:5][reference:6]。
    52│ 3.  在弹出的窗口中选择目标驱动器（如D盘），点击“移动”并等待完成即可[reference:7]。
    53│ 
    54│ #### 方案二：使用专业数据迁移工具（通用且高效）
    55│ 
    56│ 这是最推荐的方式，因为它们能自动处理复杂的注册表项、依赖文件等，成功率高，适用范围广。
    57│ 
    58│ *   **代表工具**：EaseUS Todo PCTrans、傲梅分区助手等[reference:8][!reference:9]。
    59│ *   **核心原理**：自动扫描并打包所有相关文件，并修复系统内部路径引用，就像程序从未移动过一样[reference:10]。
    60│ 
    61│ **以 EaseUS Todo PCTrans 为例的简要步骤[reference:11]：**
    62│ 1.  启动软件，在主界面选择“**Local Migration**”（本地迁移）功能。
    63│ 2.  在源盘（C盘）中勾选你想要迁移的程序。
    64│ 3.  在下方目标位置处，选择目标驱动器和文件夹。
    65│ 4.  点击“**Transfer**”（迁移）并等待完成，最后重启电脑。
    66│ 
    67│ #### 方案三：使用“符号链接”手动迁移（万能方案）
    68│ 
    69│ 这是一个非常强大的“手动+自动”方案，能完美欺骗程序[reference:12]。原理是将程序文件**先复制到D盘**，然后在**原C盘位置创建一个“智能路标”（符号链接）**[reference:13]。当程序访问原路径时，系统会自动将它导向D盘的新位置，实现无感知迁移[reference:14]。
    70│ 
    71│ 这个方案有两种操作方式：
    72│ 
    73│ *   **简易方式：使用FolderMove小工具**
    74│     [FolderMove](https://foldermove.com/) 是一款免费、绿色、几百KB的小工具，能帮你自动完成上述复杂的移动和创建链接过程，非常适合不习惯命令行的用户[reference:15]。
    75│     *   **操作**：以管理员身份运行，选择程序当前的“Move From”路径和想要迁移到的“Move To”路径，点击“Move and set Symbolic link”即可[reference:16]。
    76│ 
    77│ *   **进阶方式：使用命令行（mklink /J）**
    78│     这能让你完全掌控整个过程，适合希望通过命令行操作的用户。
    79│     1.  **复制文件夹**：将程序的整个安装文件夹从C盘**剪切**到D盘的目标位置，例如 `D:\MyApps\Chrome`[reference:17]。
    80│     2.  **以管理员身份打开命令提示符**：按 `Win` 键，输入 `cmd`，右键点击“命令提示符”，选择“以管理员身份运行”[reference:18]。
    81│     3.  **创建链接**：输入以下命令并按回车。
    82│         ```
    83│         mklink /J "C:\原程序路径" "D:\新程序路径"
    84│         ```
    85│         **例如**：移动Chrome浏览器
    86│         ```
    87│         mklink /J "C:\Program Files\Google\Chrome\Application" "D:\MyApps\Chrome"
    88│         ```
    89│         > **注意**：如果原路径包含空格，必须用英文双引号 `"` 括起来[reference:19]。
    90│     4.  **验证**：执行成功后，你会看到提示 `Junction created for ...`。此时，在C盘原位置会看到一个带快捷方式图标的文件夹，而双击进入它，实际会跳转到D盘的真实文件。
    91│ 
    92│ #### 三种方案对比总结
    93│ 
    94│ | 方案 | 适用应用类型 | 优点 | 缺点 |
    95│ | :--- | :--- | :--- | :--- |
    96│ | **1. 系统自带功能** | 仅限微软商店下载的UWP应用[reference:20] | 操作最简单，系统原生支持 | 适用范围极窄[reference:21] |
    97│ | **2. 专业迁移工具** | 几乎所有桌面应用、游戏、大型软件[reference:22] | 成功率高，自动化处理，风险低 | 部分高级功能可能需要付费 |
    98│ | **3. 符号链接** | 几乎所有桌面应用，理论上的万能方案[reference:23] | 免费、效果完美，欺骗操作系统 | 需要一定手动操作，对用户要求稍高 |
    99│ 
   100│ ### ⚠️ 第三部分：重要注意事项
   101│ 
   102│ 1.  **哪些应用不能/不建议移？**
   103│     *   **系统核心组件**：任何位于 `C:\Windows` 目录下的文件[reference:24]。
   104│     *   **驱动级程序**：如杀毒软件、硬件驱动。
   105│     *   **与系统深度集成的软件**：如Visual Studio的部分组件。若强行迁移，可能导致它们更新失败或运行异常[reference:25]。
   106│ 
   107│ 2.  **迁移失败怎么办？**
   108│     *   如果程序无法启动，首先利用第一步中创建的“**系统还原点**”进行回滚。
   109│     *   如果没有还原点，可以尝试将迁移过去的文件**手动复制回原来的位置**。
   110│     *   对于符号链接方案，你只需要**删除C盘的链接，再把D盘的文件复制回原位**即可恢复。
   111│ 
   112│ 如果你的应用是比较特殊的类型（比如通过Steam等平台安装的游戏），或者你不太确定该选哪一种，可以告诉我具体情况，我帮你看看怎么操作最稳妥～

   124│ 我们被问到“大部分是普通应用程序，但是也有matlab这种”。用户之前的问题是如何在Win11上将已安装应用从一个磁盘完全迁移到另一个磁盘，原安装包已丢失。我之前给出了几种方案：系统自带（仅UWP）、专业迁移工具（EaseUS等）、符号链接（FolderMove或mklink /J）。现在用户补充说大部分是普通应用，但也有MATLAB这种。我需要进一步给出针对MATLAB的建议。MATLAB是一个大型复杂软件，有许可证管理、大量文件、注册表项等。需要明确迁移MATLAB是否可能，以及注意事项。我可以提供专门针对MATLAB这类大型商业软件的迁移建议，包括使用符号链接可能可行，但要说明许可证和路径相关的问题。也可能推荐使用专业迁移工具，因为工具可能能处理注册表。但注意，迁移MATLAB可能面临许可证重新激活的问题，因为机器硬件ID或安装路径变化可能会被检测到。需要提醒用户考虑这一点。应结合之前方案，给出针对MATLAB的特别说明。回答要实用、贴心。
   125│ 针对你的具体情况，特别是还涉及到 **MATLAB** 这类大型专业软件，我们来细化一下方案。MATLAB 的迁移确实是这里面最需要谨慎对待的，因为它有复杂的许可证机制和系统集成。
   126│ 
   127│ ### 🎯 核心建议：先分类，再动手
   128│ 
   129│ 你可以把所有需要迁移的软件简单分成两类，分别处理，这样最稳妥。
   130│ 
   131│ #### 1. 普通应用程序
   132│ 包括微信、浏览器、播放器、普通工具软件等。这些可以放心使用我之前提到的 **方案二（专业迁移工具）** 或 **方案三（符号链接）**，成功率很高。**个人强烈推荐用 EaseUS Todo PCTrans 或 FolderMove 这种图形化工具**，简单不易错。
   133│ 
   134│ #### 2. MATLAB 及类似大型专业软件
   135│ 对待 MATLAB，需要特别小心，下面是专门针对它的分析：
   136│ 
   137│ ---
   138│ 
   139│ ### ⚠️ 迁移 MATLAB 的关键风险和应对
   140│ 
   141│ MATLAB 不能简单地像普通软件一样复制，主要是因为三点：**许可证绑定、海量文件完整性、以及路径硬编码。**
   142│ 
   143│ **风险1：许可证失效**
   144│ MATLAB 的许可证通常与计算机的硬件信息（硬盘序列号、MAC地址等）和安装位置绑定。直接移动文件夹，它极大概率会认为你把它装到了一台“新电脑”上，启动时会要求重新激活。
   145│ *   **应对**：操作前，务必确认你手头有 MATLAB 的许可证文件（license.dat 或网络许可服务器信息）和 MathWorks 账户。万一需要重新激活，可以顺利进行。
   146│ 
   147│ **风险2：文件量大，复制易出错**
   148│ MATLAB 安装文件夹动辄几十 GB，包含海量小文件。直接用 Windows 自带的复制粘贴，不仅极慢，还很容易因为权限或路径过长而中断。
   149│ *   **应对**：迁移时，**千万不要用普通复制粘贴**。用专业迁移工具或命令行 `robocopy` 来保证完整性。
   150│ 
   151│ **风险3：系统服务可能无法启动**
   152│ 某些版本的 MATLAB 会注册系统服务（如许可证管理器服务）。直接移动会使这些服务的注册信息指向一个失效路径，导致服务无法启动，进而影响使用。
   153│ 
   154│ ---
   155│ 
   156│ ### 💡 针对 MATLAB 的稳妥迁移方案（按推荐度排序）
   157│ 
   158│ #### **方案A：使用专业迁移工具（最推荐）**
   159│ 这是处理 MATLAB 这种复杂软件最简单、成功率最高的方法。
   160│ *   **原理**：软件会自动扫描 MATLAB 所有的文件、注册表项、环境变量和系统服务，然后整体“打包移植”到新盘，并自动修改所有内部路径引用。它能最大限度地避免许可证路径错乱和注册表问题。
   161│ *   **操作**：直接使用 **EaseUS Todo PCTrans** 或 **傲梅分区助手** 的“应用迁移”功能，勾选 MATLAB 并进行迁移。这是唯一可能做到“无感迁移”的方式。
   162│ 
   163│ #### **方案B：使用符号链接 + 命令行复制（最可控的免费方案）**
   164│ 如果你希望完全免费且可控，可以用符号链接，但过程需要仔细一点。
   165│ 
   166│ 1.  **第一步：完整复制，保证文件权限和完整性**
   167│     先用 `robocopy` 命令将 MATLAB 完整复制到 D 盘。以管理员身份打开命令提示符，输入：
   168│     ```cmd
   169│     robocopy "C:\Program Files\MATLAB" "D:\Matlab" /E /COPY:DATSO /DCOPY:T /R:3 /W:10 /V /LOG:D:\Matlab_copy.log
   170│     ```
   171│     *   `/E` 复制所有子目录，包括空的。
   172│     *   `/COPY:DATSO` 复制数据、属性、时间戳、安全权限（NTFS ACL）和所有者信息，这点对 MATLAB 很重要。
   173│     *   `/DCOPY:T` 复制目录时间戳。
   174│     *   `/LOG:...` 把整个过程记录下来，万一出错好排查。
   175│     *   耐心等待命令执行完毕，确认没有大量“失败”项。
   176│ 
   177│ 2.  **第二步：重命名或删除原文件夹**
   178│     复制成功后，将 C 盘的原 MATLAB 文件夹**重命名**（比如改名为 `MATLAB_old`），这样既安全，又可以作为万一失败时的恢复备份。
   179│ 
   180│ 3.  **第三步：创建符号链接（交接点）**
   181│     执行：
   182│     ```cmd
   183│     mklink /J "C:\Program Files\MATLAB" "D:\Matlab"
   184│     ```
   185│     如果 MATLAB 安装在 `C:\Program Files (x86)` 等其他位置，请相应替换路径。**注意路径用英文双引号括起来。**
   186│ 
   187│ 4.  **第四步：测试与清理**
   188│     *   重启电脑，然后启动 MATLAB。
   189│     *   **如果一切正常**，运行几天后，就可以安全删除 C 盘那个重命名后的备份文件夹 `MATLAB_old` 了。
   190│     *   **如果启动报错或提示激活**，说明软件对路径变动比较敏感。此时只需：
   191│         ```cmd
   192│         rmdir "C:\Program Files\MATLAB"
   193│         ren "C:\Program Files\MATLAB_old" "MATLAB"
   194│         ```
   195│         这样就把一切恢复原状了，对你原来的 MATLAB 没有任何损伤。
   196│ 
   197│ ### 🔐 最关键的一步：操作前，请一定先备份激活信息
   198│ 无论用哪种方式，建议先备份以下内容，以防万一：
   199│ 1.  **许可证文件**：通常位于 `C:\Program Files\MATLAB\R20xxx\licenses\` 下，复制一份备用。
   200│ 2.  **MathWorks 账户**：确保你知道自己的 MathWorks 账户和密码。如果许可证是绑定电脑的，你可能需要通过账户来反激活/重新激活。
