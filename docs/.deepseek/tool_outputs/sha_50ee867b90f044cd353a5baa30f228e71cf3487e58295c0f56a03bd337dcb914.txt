### 容器内双Python环境现状诊断
**标签**: Docker, Python, Conda, PATH
**要点**:
1. 容器内存在两个独立Python环境：Conda的`/opt/conda/bin/python3`和系统的`/usr/bin/python3`
2. `su`命令会重置`PATH`，导致系统Python被优先找到，包列表看似不一致
**关键代码**:
```bash
which pip3   # 原始shell: /opt/conda/bin/pip3；su后: /usr/bin/pip3
```

---

### 从Conda导出包列表到系统Python重新安装
**标签**: pip, freeze, 包迁移
**要点**:
1. 在Conda环境中用`pip freeze`导出全部包名及版本到文本文件
2. 切换到系统Python后执行`pip install -r`批量安装，确保版本一致
**关键代码**:
```bash
pip3 freeze > /tmp/conda_packages.txt
pip3 install -r /tmp/conda_packages.txt
```

---

### 通过PYTHONPATH共享Conda包（不推荐）
**标签**: PYTHONPATH, 环境变量, 包共享
**要点**:
1. 设置`PYTHONPATH`指向Conda的`site-packages`可让系统Python直接导入Conda包
2. 仅当两个Python版本完全相同且所有包均为纯Python时才可行，C扩展会因动态库不兼容而失败
**关键代码**:
```bash
export PYTHONPATH=/opt/conda/lib/python3.x/site-packages:$PYTHONPATH
```

### su时保留Conda环境的正确做法
**标签**: su, Conda, 环境隔离, Dockerfile
**要点**:
1. 使用`su -`后显式source conda profile并激活环境，避免PATH被重置
2. Dockerfile中将`/opt/conda/bin`加入`PATH`设为默认，统一环境减少包重复安装
**关键代码**:
```bash
source /opt/conda/etc/profile.d/conda.sh && conda activate base
```

### 从系统Python迁移包到Conda环境
**标签**: pip, conda, 反向迁移
**要点**:
1. 在系统Python中`pip freeze`导出包列表，回到Conda环境用`pip install -r`安装
2. 也可用`conda install`安装，但需先将包名通过`cut`和`xargs`转换处理
**关键代码**:
```bash
pip3 freeze > /tmp/system_packages.txt
pip3 install -r /tmp/system_packages.txt
cat /tmp/system_packages.txt | cut -d= -f1 | xargs conda install -y
```
