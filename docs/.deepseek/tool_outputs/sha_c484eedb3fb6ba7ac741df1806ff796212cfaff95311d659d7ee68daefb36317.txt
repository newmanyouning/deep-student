   135│ ### 📂 模型文件的存储位置
   136│ 
   137│ 在 macOS 系统上，Ollama 下载的模型文件默认存储在：
   138│ 
   139│ ```bash
   140│ ~/.ollama/models/
   141│ ```
   142│ 
   143│ 这是一个隐藏目录。你可以通过以下命令来查看它的大小或直接打开它[citation:2]：
   144│ 
   145│ ```bash
   146│ # 查看模型文件夹占用的磁盘空间
   147│ du -sh ~/.ollama/models
   148│ 
   149│ # 在 Finder 中打开该文件夹
   150│ open ~/.ollama/models
   151│ ```
   152│ 
   153│ 该目录下主要包含 `blobs`（存放模型的实际数据层）和 `manifests`（存放模型的元数据）等子目录[citation:7]。
   154│ 
   155│ ### 💡 管理与自定义模型路径
   156│ 
   157│ **1. 查看当前存储路径**
   158│ 
   159│ 如果不确定模型是否保存在默认位置，可以直接检查该目录[citation:2][citation:3]：
   160│ 
   161│ ```bash
   162│ ls -la ~/.ollama/models
   163│ ```
   164│ 
   165│ **2. 修改模型下载路径（推荐）**
   166│ 
   167│ 如果默认的 `~/.ollama/models` 目录磁盘空间不足，可以通过设置 `OLLAMA_MODELS` 环境变量来更改存储位置[citation:2][citation:4][citation:7]。
   168│ 
   169│ *   **临时修改（当前终端会话有效）**：
   170│     ```bash
   171│     export OLLAMA_MODELS="/path/to/your/new/models/directory"
   172│     ```
   173│ 
   174│ *   **永久修改（推荐）**：
   175│     将上面的命令添加到你的 shell 配置文件中（例如 `~/.zshrc`，因为 macOS 默认使用 zsh），然后重新加载配置[citation:4]：
   176│     ```bash
   177│     echo 'export OLLAMA_MODELS="/path/to/your/new/models/directory"' >> ~/.zshrc
   178│     source ~/.zshrc
   179│     ```
   180│ 
   181│ 配置完成后，需要重启 Ollama 服务（可以简单地重启终端或通过 `ollama serve` 重新启动），之后再下载的模型就会保存到新路径了。
