  1747│ | **COM 加载项** | `.dll`, `.exe` | 基于COM标准，通过注册表注册，可使用C++/C#等开发[reference:1]。功能最强大，与宿主程序深度集成[reference:2]。 | 主要用于Windows平台，技术较成熟[reference:3]。 |

  1803│ 
  1804│ ### 🔑 第一步：关闭WPS沙箱保护模式
  1805│ 新版的WPS内置了沙箱环境，用于隔离代码运行。这常导致COM接口调用时报错，例如“运行时错误'70'：拒绝的权限”[reference:0][reference:1]或“异常来自 HRESULT:0x800A03EC”[reference:2]。
  1806│ 
  1807│ **操作步骤**：
  1808│ 1. 打开WPS表格，点击左上角“文件”菜单。
  1809│ 2. 选择“选项”，然后在弹出的窗口中找到并点击“全局设置”。
  1810│ 3. 在“全局设置”页面里，找到“沙箱保护模式”相关的选项。
  1811│ 4. **取消勾选**，然后点击确定，并**彻底关闭并重启WPS软件**。这是让你的COM调用获得完整系统权限的基础。
  1812│ 
  1813│ ### ⚙️ 第二步：配置信任中心（最关键）
  1814│ 这是解决VBA权限问题的核心。WPS的信任中心与Microsoft Office路径略有不同。
  1815│ 
  1816│ **详细操作路径**：
  1817│ 1. 在WPS表格中，点击左上角“**文件**” → “**选项**”。
  1818│ 2. 在弹出的“选项”对话框中，选择“**信任中心**”。
  1819│ 3. 点击右侧的“**信任中心设置**”按钮。
  1820│ 4. 在弹出的新窗口中进行以下关键设置[reference:3][reference:4]：
  1821│    - **宏设置**：
  1822│      - 为了确保功能可用，建议勾选“**启用所有宏**”(不推荐，可能有风险)[reference:5]。
  1823│      - **强烈建议**：在“宏设置”面板中，**务必勾选“信任对VBA工程对象模型的访问”**[reference:6][reference:7]。这个选项允许外部程序（如你的COM脚本）通过编程方式访问和修改VBA项目，**对实现动态注入代码至关重要**[reference:8][reference:9]。
  1824│    - **可靠发行商**：
  1825│      - 也可以在此页面勾选“**信任对于'Visual Basic项目'的访问**”，这两个选项功能是等效的[reference:10][reference:11]。
  1826│ 
  1827│ > **提示**：如果完成设置后依然报错，可以尝试在信任中心设置里将宏安全性临时调至最低，或在“受信任位置”中添加你测试用的Excel文件所在文件夹[reference:12]。
  1828│ 
  1829│ ### 🛠️ 第三步：安装VBA支持库（基础环境）
  1830│ 如果信任中心的相关选项是灰色的，或者完全找不到，说明WPS缺少VBA基础支持环境。
  1831│ 
  1832│ **确认与安装**：
  1833│ 1. **检查是否已安装**：在WPS中按 `Alt+F11` 能否打开VBA编辑器。如果打不开，就需要安装。
  1834│ 2. **安装来源**：
  1835│    - **官方渠道**：个人版通常不内置VBA。可以访问WPS官网或官方公众号，搜索并下载名为 `vba6chs.msi` 的插件进行安装[reference:13]。
  1836│    - **其他版本**：WPS企业版或超级会员用户通常已内置VBA支持库，无需额外安装[reference:14]。
  1837│ 3. **验证**：安装后重启WPS，按 `Alt+F11` 应能正常打开VBA编辑器。

  1855│         MsgBox "VBA工程对象模型访问权限已开启，请重启WPS后生效。", vbInformation
  1856│     End If
  1857│     On Error GoTo 0
  1858│ End Sub
  1859│ ```
