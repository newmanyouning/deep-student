   134│ 在 Windows 上，如果已经安装了新版本的 Python（如 3.8+），但旧版本（3.6）仍然保留，你可以通过以下步骤安全地卸载旧版本，同时保持新版本可用。
   135│ 
   136│ ---
   137│ 
   138│ ### 1. 确认当前安装的 Python 版本和路径
   139│ 
   140│ 打开命令提示符（cmd）或 PowerShell，执行：
   141│ ```cmd
   142│ where python
   143│ ```
   144│ 这会列出所有在系统环境变量 `PATH` 中能找到的 Python 可执行文件路径。通常你会看到类似：
   145│ - `C:\Users\国检中心\AppData\Local\Programs\Python\Python36\python.exe`
   146│ - `C:\Users\国检中心\AppData\Local\Programs\Python\Python310\python.exe`
   147│ 
   148│ 确认哪个路径对应 Python 3.6（通常路径中包含 `Python36`）。如果只想查看版本，可以分别运行：
   149│ ```cmd
   150│ python3.6 --version   # 或直接 python --version（可能指向默认版本）
   151│ ```
   152│ 
   153│ ---
   154│ 
   155│ ### 2. 通过“程序和功能”卸载 Python 3.6
   156│ 
   157│ - 按下 `Win + R`，输入 `appwiz.cpl` 并回车，打开“程序和功能”（或通过“设置” > “应用” > “应用和功能”）。
   158│ - 在列表中找到 **Python 3.6 (64-bit)** 或类似名称的程序。
   159│ - 右键点击并选择“卸载”，按照提示完成卸载。
   160│ 
   161│ 卸载过程中，Windows 会自动移除大部分文件，但可能残留一些用户数据或脚本目录。
   162│ 
   163│ ---
   164│ 
   165│ ### 3. 清理环境变量中的旧路径
   166│ 
   167│ Python 安装时通常会将自身路径添加到系统环境变量 `PATH` 中。卸载后，这些路径可能仍然存在，导致 `where python` 仍显示旧的路径（但文件已不存在）。你需要手动清理：
   168│ 
   169│ 1. 右键“此电脑” > “属性” > “高级系统设置” > “环境变量”。
   170│ 2. 在“系统变量”或“用户变量”中找到名为 `Path` 的变量，双击编辑。
   171│ 3. 检查并删除包含 `Python36` 或旧 Python 路径的条目（例如 `C:\Users\国检中心\AppData\Local\Programs\Python\Python36\Scripts\` 和 `C:\Users\国检中心\AppData\Local\Programs\Python\Python36\`）。
   172│ 4. 确认新 Python 的路径（如 `Python310`）仍然存在且顺序正确（通常靠前）。
   173│ 5. 点击“确定”保存所有更改。
   174│ 
   175│ **注意**：如果 `PATH` 中同时存在多个 Python 路径，命令 `python` 会调用第一个匹配到的。你可以调整顺序，或将新 Python 的路径移到旧版之上，确保默认使用新版。
   176│ 
   177│ ---
   178│ 
   179│ ### 4. 验证卸载结果
   180│ 
   181│ 重新打开命令提示符（确保是新窗口，以加载新的环境变量），运行：
   182│ ```cmd
   183│ python --version
   184│ ```
   185│ 应该显示新版本号（如 3.10.x）。  
   186│ 再运行：
   187│ ```cmd
   188│ where python
   189│ ```
   190│ 应该只显示新版本的路径。
   191│ 
   192│ ---
   193│ 
   194│ ### 5. 可选：使用 `py` 启动器管理多版本
   195│ 
   196│ Windows 上的 Python 安装包通常会附带一个 `py` 启动器（如果安装时勾选了“安装启动器”）。它可以让你通过 `py -3.6`、`py -3.10` 等命令指定运行哪个版本。如果你希望保留旧版（例如某些旧项目需要），但不想让旧版干扰系统默认的 `python` 命令，可以**不卸载**，而是调整环境变量顺序，然后通过 `py` 启动器选择版本。
   197│ 
   198│ 卸载步骤同上，但如果你决定保留旧版，只需确保环境变量中新版路径在旧版之前即可。
