    25│ ### 1. 生成多项式中的 x 是什么？
    26│ 
    27│ **简单直接的回答：x 不是待传输帧的数值，它只是一个多项式的“占位符”。**
    28│ 
    29│ **详细解释：**
    30│ 
    31│ 在CRC中，我们使用**多项式**来表示二进制串。这是一种非常巧妙的数学工具，它让我们可以用代数的方法来处理二进制除法。
    32│ 
    33│ - **二进制位与多项式项的对应关系：**
    34│     - 一个二进制数的每一位，代表多项式对应项系数是 `1` 还是 `0`。
    35│     - 二进制数的最低位（最右边）对应 x⁰ 项（即常数项）。
    36│     - 从右往左，依次是 x¹, x², x³ ... 项。
    37│ 
    38│ **举个例子：**
    39│ 假设我们有一个生成多项式为 **G(x) = x³ + x² + 1**。
    40│ - `x³` 的系数是 1
    41│ - `x²` 的系数是 1
    42│ - `x¹` 的系数是 0（因为多项式里没有 x¹ 项）
    43│ - `x⁰` 的系数是 1
    44│ 所以，这个多项式对应的二进制串就是：**1101**。
    45│ 
    46│ **结论：** 生成多项式中的 `x` 只是一个符号，用来构建一个多项式，这个多项式最终会转换成一个关键的二进制除数（比如上面的1101）。它本身不代表任何数据。

    50│ ### 2. 循环冗余校验码（CRC）的计算过程
    51│ 
    52│ CRC的计算本质上是进行一种特殊的“模2除法”。它的核心思想是：**在要发送的数据后面附加一个校验码（即余数），使得整个数据串能被一个预先约定好的除数（生成多项式）整除。**
    53│ 
    54│ 下面是详细的步骤，我们用一个例子贯穿始终：
    55│ 
    56│ - **待发送数据（帧）：** `101001`
    57│ - **生成多项式：** `G(x) = x³ + x² + 1` (对应二进制 `1101`)
    58│ 
    59│ #### 第一步：在数据帧末尾添加 n 个 “0”
    60│ 
    61│ n 是生成多项式的位数减一。我们的生成多项式是 `1101`，一共4位，所以 n = 3。
    62│ 
    63│ 原始数据： `101001`
    64│ 添加3个0后： `101001` **`000`**
    65│ 
    66│ 这个新串我们称之为 **“被扩展的数据帧”**。
    67│ 
    68│ #### 第二步：进行“模2除法”
    69│ 
    70│ 用 **生成多项式对应的二进制串** 去除 **第一步中得到的数据帧**。
    71│ 
    72│ **模2运算规则（非常重要）：**
    73│ - 加法不进位，减法不借位。**实际上，加法和减法都等同于“异或”（XOR）运算。**
    74│ - `0 XOR 0 = 0`
    75│ - `0 XOR 1 = 1`
    76│ - `1 XOR 0 = 1`
    77│ - `1 XOR 1 = 0`
    78│ 
    79│ 现在我们来除：
    80│ 除数： `1101`
