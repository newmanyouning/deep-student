  2568│ #### 3.1 直接返回值（推荐方案）
  2569│ 
  2570│ 通过`Application.Run`调用VBA函数（Function），**直接接收返回值**[reference:2]：
  2571│ 
  2572│ ```python
  2573│ # VBA中的Function
  2574│ Function MySum(x, y)
  2575│     MySum = x + y
  2576│ End Function
  2577│ 
  2578│ # Python中调用并直接获取返回值
  2579│ result = wps.Run("MySum", 3, 5)  # result = 8
  2580│ ```
  2581│ 
  2582│ **关键要点**：VBA函数名必须与赋值目标变量名一致，否则无返回值[reference:3]。即`Function MySum(x,y)`内部必须写`MySum = x + y`，而不能写`T = x + y`。
  2583│ 
  2584│ #### 3.2 写入工作表单元格（最稳定）
  2585│ 
  2586│ VBA将结果直接写入工作表的指定区域，Python再从COM读取：
  2587│ 
  2588│ ```python
  2589│ # VBA将数据写入Sheet1的A1:D10区域
  2590│ vba_code = '''
  2591│ Sub WriteResultToSheet()
  2592│     Sheet1.Range("A1:D10").Value = resultArray
  2593│ End Sub
  2594│ '''
  2595│ 
  2596│ # Python执行后读取
  2597│ wps.Run("WriteResultToSheet")
  2598│ data = wb.Sheets(1).Range("A1:D10").Value  # 返回二维元组
  2599│ ```
  2600│ 
  2601│ 这是当前项目`VBAInjector`采用的模式，也是**最稳定、最推荐**的方式。
  2602│ 
  2603│ #### 3.3 写入剪贴板
  2604│ 
  2605│ VBA通过`MSForms.DataObject`将数据写入Windows剪贴板，Python通过`pyperclip`等库读取[reference:4]：
  2606│ 
  2607│ ```vba
  2608│ ' VBA写入剪贴板
  2609│ Sub WriteToClipboard(text As String)
  2610│     Dim objData As New MSForms.DataObject
  2611│     objData.SetText text
  2612│     objData.PutInClipboard
  2613│ End Sub
  2614│ ```
  2615│ 
  2616│ 此方式适合小数据量场景，但依赖系统剪贴板，可能与其他应用冲突。
  2617│ 
  2618│ #### 3.4 写入临时文件
  2619│ 
  2620│ VBA将结果序列化为JSON/CSV文件，Python读取后解析[reference:5]。此方式适合大数据量，但涉及磁盘I/O，性能较差。
  2621│ 
  2622│ #### 3.5 创建命名区域（Named Range）
  2623│ 
  2624│ VBA将复杂结构数据以命名区域形式保存，Python通过`wb.Names("ResultRange").RefersToRange.Value`获取。
  2625│

  2628│ ### 四、WPS环境特别注意事项
  2629│ 
  2630│ #### 4.1 ProgID选择
  2631│ 
  2632│ WPS可用ProgID优先级：`Ket.Application` → `KWPS.Application` → `ET.Application`。当前测试确认`Ket.Application`在WPS 12.0版本中可用。
  2633│ 
  2634│ #### 4.2 VBA工程访问权限（关键）
  2635│ 
  2636│ 写入VBA代码**必须**在WPS信任中心启用以下两项：
  2637│ 1. **启用所有宏**
  2638│ 2. **信任对VBA工程对象模型的访问**（核心，对应COM属性`Application.VBE`）
  2639│ 
  2640│ 否则调用`workbook.VBProject`将抛出`-2146827284 (0x800A03EC)`错误。
  2641│ 
  2642│ #### 4.3 路径格式要求
  2643│ 
  2644│ 传递给WPS COM的文件路径**必须使用绝对路径**，相对路径会导致文件找不到错误[reference:6]。
  2645│ 
  2646│ #### 4.4 权限问题排查
  2647│ 
  2648│ 如果出现`VBProject`成员不存在的错误，通常由以下原因导致：
  2649│ - 未启用"信任对VBA工程对象模型的访问"
  2650│ - WPS版本过低（需2019+）
  2651│ - 工作簿文件格式不支持宏（需`.xlsm`或`.xls`格式，`.xlsx`不含VBA工程）
  2652│ 
  2653│ ---
  2654│ 
  2655│ ### 五、支持的数据类型
  2656│ 
  2657│ Python与VBA通过COM传递时，数据类型映射如下[reference:7][reference:8]：
  2658│ 
  2659│ | Python类型 | VBA类型 | 直接传递 | 备注 |
  2660│ |-----------|---------|---------|------|
  2661│ | `int`, `float` | `Long`, `Double` | ✅ | 标量类型完全兼容 |
  2662│ | `str` | `String` | ✅ | 直接传递 |
  2663│ | `list`, `tuple` | `Variant`数组 | ⚠️ | 需要转换为COM SafeArray |
  2664│ | `dict` | `Dictionary`对象 | ❌ | 不可直接传递，需序列化为JSON字符串 |
  2665│ | `bool` | `Boolean` | ✅ | 直接传递 |
  2666│ | `None` | `Empty`/`Null` | ✅ | 直接传递 |
  2667│ 
  2668│ **建议**：复杂数据结构统一序列化为JSON字符串在两端传递，这是最稳妥的方式。当前项目`VBAInjector`正是采用此方式。
