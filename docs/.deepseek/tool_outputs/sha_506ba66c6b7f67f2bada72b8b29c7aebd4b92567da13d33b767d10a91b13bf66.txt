  1927│ - **[AI 的三大核心训练阶段]** — 现代大语言模型的三个训练阶段
  1928│   - 来源文件：训练方式与分词关系.md
  1929│   - 核心解释：现代 LLM 训练分为三个阶段：① 预训练——在海量无标注文本上进行自监督学习，核心任务是"预测下一个词"，目标是让模型学习语法、知识、推理能力；② 监督微调（SFT）——使用高质量"指令-回答"对进行有监督训练，将基座模型对齐到人类意图；③ 强化学习（RLHF）——通过人类或 AI 反馈训练奖励模型，优化模型的"偏好"，使其回答更符合人类价值观。

  1931│ - **[分词器的"冻结"特性]** — 分词器在预训练前确定，后续训练无法更改
  1932│   - 来源文件：训练方式与分词关系.md
  1933│   - 核心解释：分词器（Tokenizer）通常在预训练之前就训练好，并且在后续所有训练阶段中保持不变。一旦预训练确立，分词器就被"冻结"。如果事后想通过微调让模型精通一门新语言，原有分词器对新语言的拆分效率可能极低（一个单词被拆成 7-8 个 Token），且无法直接添加新词——因为预训练阶段的 Embedding 层完全没有这些新 Token 的向量表示。

  1936│   - 来源文件：训练方式与分词关系.md
  1937│   - 核心解释：预训练的数据构成和分词器训练算法（如 BPE）决定了分词的质量。英文为主的预训练数据会使分词器高效拆分英文单词，但对中文可能退化为逐字 Token 或 UTF-8 字节回退。若预训练包含大量代码，分词器会保留空格和缩进敏感性，结构化数据的权重倒逼分词器保留更多空白符和特殊符号作为独立 Token。

  1940│   - 来源文件：训练方式与分词关系.md
  1941│   - 核心解释：微调阶段虽不改变分词器词表，但会赋予特定 Token 极其强烈的语义含义。例如：① 模板与角色扮演——引入特殊控制 Token 如 `<|user|>`、`<|assistant|>`、`<|end|>`，通过定向训练让模型学会"看到 `<|assistant|>` 必须开始生成答案，看到 `<|end|>` 必须停止"；② 思维链的固化——当微调数据使用 `\n\n` 或 `Step 1:` 等特定格式时，模型学习到在特定分词序列之后必须进行深度推理。

  1943│ - **[强化学习隐性调控分词选择偏好]** — RLHF 通过奖励机制间接塑造模型对分词边界的选择
  1944│   - 来源文件：训练方式与分词关系.md
  1945│   - 核心解释：强化学习对分词的影响最为隐性，通过奖励机制间接塑造模型行为：① Token 经济学偏见——由于训练和推理成本按 Token 数计算，RLHF 存在隐性的"简洁性偏见"，奖励模型对冗长、重复的分词序列给予较低分数，使模型倾向于输出更紧凑的词汇组合；② 安全与拒绝——在 RLHF 中，模型学习在特定"拒绝前缀"（如 `I am sorry,`）之后固化特定分词路径，某些分词组合路径被强化成了"安全阀门"。

  1948│   - 来源文件：训练方式与分词关系.md
  1949│   - 核心解释：在已预训练模型上通过微调学习新语言时，原有分词器对新语言的拆分效率极低（一个单词可能被拆成 7-8 个 Token），导致推理速度变慢；且无法直接添加新词，因为预训练阶段的 Embedding 层完全没有新 Token 的向量表示。这也解释了为什么顶级模型（如 Llama 3、GPT-4）极其重视预训练阶段的数据配比——分词器的性能上限在训练开始前就已经被锁定。

  1956│   - 来源文件：训练方式与分词关系.md
