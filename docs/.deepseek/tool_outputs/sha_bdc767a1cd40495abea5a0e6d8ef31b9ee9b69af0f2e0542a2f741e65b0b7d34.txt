    25│ ## 一、重构目标
    26│ 
    27│ 1. **固化 WPS COM 架构**：彻底移除遗留的 `xlwings`/`openpyxl` 死代码，统一到 `wps_bridge` 调度层。
    28│ 2. **解耦控制器与视图**：引入服务层 (Service Layer) 和用例层 (Use Case)，让 UI 控制器只负责协调，业务逻辑下沉。
    29│ 3. **统一异常处理与日志**：将分散的 `try-except` 收敛到框架层，确保所有错误写入 `error.log` 并友好提示。
    30│ 4. **修复 VBA 路径问题**：解决 `template.xls` 相对路径导致的提取失败。
    31│ 5. **增强 WPS 环境检测**：细化 VBA 插件检测、COM 权限检测，提供更精准的降级策略。
    32│ 6. **提高测试覆盖率**：针对服务层和桥接层编写可自动化测试的单元。

    46│ 
    47│ 采用 **分层架构**：
    48│ 
    49│ ```
    50│ ┌─────────────────────────────────────────┐
    51│ │            UI Layer (PyQt5)              │
    52│ │  MainWindow, Dialogs, Views              │
    53│ └─────────────────┬───────────────────────┘
    54│                   │ 调用
    55│ ┌─────────────────▼───────────────────────┐
    56│ │         Controller Layer                 │
    57│ │  BatchController, TemplateController ... │
    58│ └─────────────────┬───────────────────────┘
    59│                   │ 委托
    60│ ┌─────────────────▼───────────────────────┐
    61│ │          Service / UseCase Layer         │
    62│ │  ExcelAnalysisService, HeaderFooterService│
    63│ │  BatchOperationService, FormatService    │
    64│ └─────────────────┬───────────────────────┘
    65│                   │ 依赖
    66│ ┌─────────────────▼───────────────────────┐
    67│ │           WPS Bridge Facade              │
    68│ │  WPSConnector, VBAInjector, Extractor    │
    69│ └─────────────────┬───────────────────────┘
    70│                   │ COM

    72│ │           WPS Application                │
    73│ └─────────────────────────────────────────┘
    74│ ```
