    81│ ### 方法一：通过命令行参数配置（推荐，一次有效）
    82│ `duckduckgo-mcp-server` 支持直接在命令后面加选项。在 PowerShell 中你可以这样写：
    83│ 
    84│ ```powershell
    85│ uvx --index https://pypi.tuna.tsinghua.edu.cn/simple duckduckgo-mcp-server `
    86│     --safe-search strict `
    87│     --region us-en `
    88│     --fetch-backend httpx
    89│ ```
    90│ 
    91│ **常用参数说明**  
    92│ - `--safe-search`：安全搜索级别，可选 `strict`（严格）、`moderate`（中等，默认）、`off`（关闭）  
    93│ - `--region`：DuckDuckGo 区域代码，如 `us-en`（美国英语）、`cn-zh`（中国中文）、`jp-jp` 等，无特殊需求可设为 `wt-wt`（全球）或 `none`  
    94│ - `--fetch-backend`：请求后端，支持 `httpx`（默认）和 `requests`
    95│ 
    96│ 运行 `duckduckgo-mcp-server --help` 可以查看所有支持的参数。
    97│

   100│ ### 方法二：通过环境变量配置（影响当前终端会话）
   101│ 在 PowerShell 中先用 `$env:变量名` 设置环境变量，然后再运行服务器：
   102│ 
   103│ ```powershell
   104│ $env:DUCKDUCKGO_SAFE_SEARCH = "strict"
   105│ $env:DUCKDUCKGO_REGION      = "us-en"
   106│ $env:DUCKDUCKGO_FETCH_BACKEND = "httpx"
   107│ 
   108│ uvx --index https://pypi.tuna.tsinghua.edu.cn/simple duckduckgo-mcp-server
   109│ ```
   110│ 
   111│ 这样运行时会看到配置已改变，例如输出：
   112│ ```
   113│ SafeSearch: STRICT (kp=1)
   114│ Default Region: us-en
   115│ Fetch backend: httpx
   116│ ```
   117│ 
   118│ 环境变量名与值的对照关系与命令行参数一致，如果同时使用两者，命令行参数优先级更高。

   122│ ### 在 MCP 客户端（如 Claude Desktop）中永久配置
   123│ 你通常不会在终端里手动启动它，而是把它写进 MCP 客户端配置文件，并传入参数。以 Claude Desktop 的 `claude_desktop_config.json` 为例：
   124│ 
   125│ ```json
   126│ {
   127│   "mcpServers": {
   128│     "duckduckgo": {
   129│       "command": "uvx",
   130│       "args": [
   131│         "--index", "https://pypi.tuna.tsinghua.edu.cn/simple",
   132│         "duckduckgo-mcp-server",
   133│         "--safe-search", "strict",
   134│         "--region", "us-en"
   135│       ]
   136│     }
   137│   }
   138│ }
   139│ ```
   140│ 
   141│ 这样每次客户端调用时都会使用你指定的配置，无需每次手动设置环境变量。

   179│ ### 为什么不能“在线”改？
   180│ 所有配置（SafeSearch、Region、Fetch backend）都是**在服务器启动时一次性读取**的，之后运行期间没有提供任何命令或接口去热更新这些值。  
   181│ 如果你在终端里仅仅修改了环境变量，或者心里想着“我换个参数再跑一遍”，这对已经存在的进程是无效的，因为它已经固化了启动时的设置。
