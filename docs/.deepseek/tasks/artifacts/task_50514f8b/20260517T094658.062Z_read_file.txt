- **[pip show 查看 Python 包版本]** — 容器内最常用的模型版本查看方式
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：如果模型通过 pip 安装，在容器内执行 `pip show <包名>` 即可获得 Version 字段。也可用 `python -c "import <包名>; print(<包名>.__version__)"` 一行命令直接输出版本号，无需进入交互式解释器。

- **[pip list 批量列出已安装包]** — 不确定包名时先列出再筛选
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：执行 `pip list` 可列出容器内所有已安装的 Python 包及其版本。结合 `grep` 筛选关键词（如 `pip list | grep torch`）可以快速定位目标包名，然后再用 `pip show` 或 `python -c` 查看详细版本。

- **[apt/dpkg 查看系统包版本]** — 适用于通过系统包管理器安装的模型
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：如果模型是作为系统包（如 CUDA、libtorch）通过 apt 安装的，使用 `apt show <包名>` 查看包详情，或 `dpkg -l | grep <关键词>` 从已安装包列表中筛选版本信息。

- **[可执行文件 --version 参数]** — 编译型模型程序的版本查看方式
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：对于 llama.cpp 的 main 等编译好的可执行文件，尝试 `--version` 或 `-v` 参数获取版本。如果程序不在 PATH 环境变量中，需要指定完整路径如 `./main --version`。

- **[环境变量中查找版本信息]** — 通过 env 命令筛选版本相关变量
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：某些部署方式将版本号写入环境变量。使用 `env` 查看全部环境变量，或 `env | grep -i version` 筛选包含 "version" 字样的变量，适合版本信息不在包管理器记录中的场景。

- **[模型文件名/目录识别版本]** — 从文件名或 HuggingFace 缓存目录推断版本
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：列出模型存放目录（常见路径如 `/app/models`、`~/.cache/huggingface/hub/`），文件名中常包含版本标识如 `bert-base-uncased` 或 `llama-2-7b.Q4_K_M.gguf`。HuggingFace 缓存目录名还包含模型 ID 和版本哈希。

- **[find 搜索版本文件]** — 容器内查找 README 或 version 文件获取版本信息
  - 来源文件：`01_AI与编程开发/查看云服务器模型版本方法.md`
  - 核心解释：如果上述方法均无效，可用 `find / -name "*version*" 2>/dev/null` 在容器根目录递归搜索含 version 字样的文件，或直接查看项目文档如 `cat /app/README.md`，版本信息可能记录在文档或专门的版本文件中。
